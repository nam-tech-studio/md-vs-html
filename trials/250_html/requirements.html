チャットツール「Linkr」要件定義書 (Extended)

1. 概要

社内コミュニケーション用のチャットツール。本書はコア機能、コミュニケーション支援機能、および拡張機能を定義する。

2. ユーザーとワークスペース

  • ユーザーは1つのメールアドレスにつき1アカウントを保持する
  • 各ユーザーは複数のワークスペースに所属できる
  • ワークスペースはチームの単位であり、URLサブドメインで識別される (例: acme.linkr.app)
  • ワークスペース内のユーザーは表示名とアバター画像を設定できる
  • ユーザーはオンライン / 退席中 / オフラインの3種類のプレゼンス状態を持つ
  • プレゼンスは10分間操作がない場合に自動的に「退席中」となる
  • ユーザーはステータスメッセージ (絵文字 + 最大100文字のテキスト) を任意に設定できる
  • ステータスメッセージは有効期限 (1時間 / 4時間 / 終日 / カスタム / 無期限) を選択できる

3. ロールと権限

ワークスペースには以下の3つのロールが存在する。

ロールと権限
ロール権限
オーナーワークスペースの全権限。請求設定の変更、削除、他オーナーの任命が可能
管理者ユーザー招待、チャネル管理、ロール付与 (オーナー以外) が可能
メンバー通常利用権限のみ
  • ロールはオーナーまたは管理者によって変更可能 (オーナーへの昇格はオーナーのみ可能)
  • ワークスペースには常に1名以上のオーナーが存在する必要がある
  • 退会するオーナーは事前に他ユーザーへオーナー権限を委譲する必要がある

4. チャネル

  • チャネルはワークスペース内のトピック単位の会話空間である

チャネルにはパブリックとプライベートの2種類がある。

パブリック
ワークスペース全員が閲覧・参加可能
プライベート
招待されたメンバーのみ閲覧可能
  • チャネル名は半角英数字とハイフンのみ、最大80文字
  • チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ
  • ユーザーは任意のパブリックチャネルに自由参加できる
  • ユーザーは自分が参加中のチャネルから退出できる
    • ただし最後の管理者は退出前に他メンバーに管理権限を委譲する必要がある
  • チャネルにはトピック (最大250文字) と説明 (最大1500文字) を設定できる
  • チャネルはアーカイブできる
    • アーカイブ後は新規メッセージ投稿不可、ただし閲覧と検索は可能
    • アーカイブはチャネル管理者またはワークスペース管理者が実行可能
  • アーカイブされたチャネルはワークスペース管理者によって復元可能

5. メッセージ

  • ユーザーはチャネル内にテキストメッセージを投稿できる
  • メッセージは最大4000文字
  • メッセージには投稿時刻と投稿者が記録される
  • メッセージは投稿後も時系列で永続表示される
  • メッセージ本文ではMarkdown記法 (太字、斜体、コードブロック、リンク、引用、リスト) が利用可能

5.1 編集と削除

  • ユーザーは自分のメッセージを編集できる
  • 編集されたメッセージには「編集済み」の表示が付く
  • 編集履歴は保存されないが、編集日時は記録される
  • ユーザーは自分のメッセージを削除できる
  • チャネル管理者は他人のメッセージを削除できる (編集は不可)
  • 削除されたメッセージは「このメッセージは削除されました」というプレースホルダで表示される
  • メッセージ削除はソフトデリートとし、削除ログは90日間保管される

5.2 スレッド

  • 任意のメッセージに対して返信を行うことでスレッドを開始できる
  • スレッド内のメッセージはチャネルのメインタイムラインには表示されない
  • スレッド開始者および参加者は新規スレッド返信時に通知を受け取る
  • スレッド返信は「チャネルにも投稿」オプションで親チャネルへ二重投稿可能
  • スレッド内のメッセージにもメンション・編集・削除・リアクションが可能

5.3 リアクション

  • ユーザーはメッセージに対して絵文字リアクションを付与できる
  • 1メッセージあたり最大23種類の絵文字をリアクションとして表示する
  • 同一ユーザーが同一メッセージに対して同じ絵文字を複数回リアクションすることはできない
  • カスタム絵文字はワークスペース管理者がアップロードでき、全ユーザーがリアクションに使用できる
  • カスタム絵文字は1ファイル128KB以下、PNG / JPEG / GIF形式

6. メンション

  • メッセージ内で @ユーザー名 を入力するとそのユーザーをメンションできる
  • メンションされたユーザーには通知が届く (通知設定に従う)
  • @channel は同一チャネルの全メンバーをメンションする
  • @here は同一チャネルでオンラインの全メンバーをメンションする
  • @channel および @here の使用はチャネル管理者によって制限可能
  • ユーザーグループ (@team-frontend 等) を作成可能
    • ユーザーグループはオーナーまたは管理者が作成し、所属メンバーを管理する
    • ユーザーグループのメンションはグループ内全員へのメンションとして扱われる

7. ファイル共有

  • ユーザーはメッセージにファイルを添付できる
  • 1ファイルあたり最大100MB
  • 1メッセージあたり最大10ファイル添付可能
  • 対応形式: 画像 (PNG/JPEG/GIF/WebP)、PDF、動画 (MP4/MOV)、各種ドキュメント、ZIPアーカイブ
  • 画像とPDFはインラインプレビュー表示される
  • ファイルはアクセス権を持つチャネル参加者のみ閲覧可能
  • ファイルは投稿者またはチャネル管理者によって削除可能
  • 削除されたファイルは即座にダウンロード不可となるが、メタデータは90日間保持される

8. ダイレクトメッセージ (DM)

  • ユーザーは同一ワークスペース内の任意のユーザーにDMを送信できる
  • DMは1対1の会話のみをサポートする (グループDMは本書のスコープ外)
  • DM内のメッセージはチャネルメッセージと同じ仕様 (4000文字上限・Markdown対応・編集・削除・スレッド・リアクション・ファイル添付すべて利用可能)
  • DMは送信者と受信者のみが閲覧でき、ワークスペース管理者であっても他人のDMは閲覧できない

9. 検索

  • ユーザーはメッセージ本文・チャネル名・ユーザー名・ファイル名を横断検索できる
  • 検索結果はメッセージ・チャネル・ユーザー・ファイルのカテゴリ別に表示される
  • 検索対象は本人がアクセス権を持つチャネルとDMに限定される

以下の絞り込みフィルタを利用できる。

検索フィルタ一覧
フィルタ用途
from:ユーザー名投稿者で絞り込み
in:チャネル名チャネルで絞り込み
before:YYYY-MM-DD / after:YYYY-MM-DD期間で絞り込み
has:link / has:file / has:reaction添付要素で絞り込み
  • 検索結果は関連度順またはタイムスタンプ降順で並び替えできる
  • 1クエリあたり最大1000件まで返却し、それ以上はページングで取得する

10. 通知

各ユーザーは以下の3種類のチャネル通知レベルを設定できる。

  • すべてのメッセージ
  • メンション時のみ
  • なし
  • DMは常にメンション扱いで通知される
  • 通知レベルはチャネル単位、およびワークスペース全体のデフォルトとして設定可能
  • ワークスペースのデフォルトはチャネル個別設定で上書きされる
  • 「通知を一時停止」機能で30分・1時間・2時間・翌朝9時まで全通知を抑止できる
  • 通知抑止中もアプリ内バッジ表示は継続する
  • キーワード通知: ユーザーは自身のキーワードリストを設定でき、メッセージにそのキーワードが含まれた場合に通知を受け取る
  • 通知方法: ブラウザ通知、メール、モバイルプッシュ (モバイルアプリは別書で定義)
  • 通知方法の優先順位: ブラウザ未開封時はメール、5分以内に既読されない場合のみメールを送る

11. ピン留めとブックマーク

  • チャネル管理者またはチャネルメンバーは重要メッセージをチャネル内で「ピン留め」できる
  • ピン留めはチャネルあたり最大100件
  • ピン留めされたメッセージはチャネル内のサイドパネルから一覧アクセス可能
  • ユーザーは個人ブックマークとして任意のメッセージを保存できる
    • 個人ブックマークは本人のみ閲覧可能
    • 個人ブックマーク数に上限はないが、500件を超えるとフォルダ分類が推奨される

12. 下書きと送信予約

  • メッセージ入力中の内容は自動的に下書きとして保存される
  • 下書きはチャネル / DM / スレッド単位で保持される
  • 下書きはアプリ再起動後も復元される
  • ユーザーは送信予約機能で最大120日先までの任意の日時にメッセージ送信を予約できる
  • 予約済みメッセージは送信前に編集または取り消しが可能
  • 予約済みメッセージは「送信予約一覧」画面から確認可能

13. 既読管理

  • 各メッセージは閲覧者単位で既読 / 未読状態を保持する
  • チャネル一覧では未読のあるチャネルが太字で表示される
  • ユーザーは「未読をすべて既読にする」アクションでチャネル単位またはワークスペース単位で一括既読化できる
  • 既読位置 (最終既読メッセージID) はサーバーで保存され、デバイス間で同期される

14. リマインダー

  • ユーザーは任意のメッセージに対してリマインダーを設定できる
  • リマインダー時刻はプリセット (10分後 / 1時間後 / 明日朝9時 / 来週月曜朝9時) またはカスタム指定が可能
  • リマインダー時刻になるとリマインダーBotから本人にDMで通知される
  • リマインダーDMには元メッセージへのリンクが含まれる
  • ユーザーは自分のリマインダー一覧を「リマインダー」タブから確認・取り消し可能
  • 同一メッセージに対するリマインダーはユーザー1人につき1件までで、再設定すると上書きされる

15. 投票

  • ユーザーはチャネル内で投票メッセージを作成できる
  • 投票には質問文 (最大200文字) と最大10個の選択肢 (各最大80文字) を設定可能
  • 投票形式は「単一選択」または「複数選択」を選択可能
  • 投票締切は無期限・1時間後・1日後・カスタム日時から選択可能
  • 投票結果は集計のみ表示するか、各選択肢に投票したユーザー名を公開するかを作成時に選択可能
  • 投票作成者は締切前に投票を強制終了できる
  • 締切後の投票には新規投票不可だが結果は引き続き閲覧可能

16. 音声メッセージ

  • ユーザーはメッセージ入力欄から音声メッセージを録音・送信できる
  • 音声メッセージの最大長は5分
  • 録音形式はOpus (WebM コンテナ) で保存される
  • 受信側はインラインプレーヤーで再生でき、再生速度 (0.5x / 1x / 1.5x / 2x) を選択可能
  • 音声メッセージは自動文字起こしの対象となり、文字起こし結果はメッセージ下に折りたたみ表示される
  • 音声メッセージはファイル添付の対象として保存ポリシーが適用される

17. 通知ダイジェスト

  • ユーザーはメール通知ダイジェストを有効化できる
  • ダイジェスト頻度は「1日2回 (朝9時・夕方17時)」「1日1回 (朝9時)」「無効」から選択可能
  • ダイジェストには未読のメンション・キーワードヒット・DMがチャネル別に集約される
  • ダイジェストメールには各メッセージへの直接リンクが含まれる
  • ダイジェスト送信後に新規メンションがない場合はメールを送信しない
  • ダイジェスト機能はワークスペース管理者によってワークスペース単位で無効化可能

18. 監査と履歴

ワークスペース管理者は以下のイベントを監査ログで確認できる。

  • ユーザーのログイン / ログアウト
  • ロール変更
  • チャネル作成 / アーカイブ / 復元
  • ユーザー招待 / 退会
  • 監査ログは90日間保持される (拡張は別書のエンタープライズ機能で定義)
  • 通常メンバーは自身の操作履歴のみを過去30日分閲覧可能

21. セキュリティ要件

  • 通信はすべてTLS 1.2以上で暗号化される
  • パスワードはbcrypt (cost=12) でハッシュ化して保存
  • セッションは絶対24時間でタイムアウトし、再認証が必要
  • 全APIにレートリミット (IP単位とユーザー単位の両方) を適用
  • 監査ログは改竄防止のため別アカウントのS3バケットに WORM (Write Once Read Many) で保存
