要件定義書「チャットツール『Linkr』要件定義書 (Standard)」の全内容を、章立てに沿って漏れなく列挙します。

---

## 1. 概要
- 用途: 社内コミュニケーション用のチャットツール
- 本書の定義範囲: コア機能とコミュニケーション支援機能

## 2. ユーザーとワークスペース
- ユーザーは1つのメールアドレスにつき1アカウントを保持する
- 各ユーザーは複数のワークスペースに所属できる
- ワークスペースはチームの単位であり、URLサブドメインで識別される(例: `acme.linkr.app`)
- ワークスペース内のユーザーは表示名とアバター画像を設定できる
- プレゼンス状態は3種類: オンライン / 退席中 / オフライン
- プレゼンスは10分間操作がない場合に自動的に「退席中」となる

## 3. チャネル
- チャネルはワークスペース内のトピック単位の会話空間
- 種類は2種類: パブリック / プライベート
  - パブリック: ワークスペース全員が閲覧・参加可能
  - プライベート: 招待されたメンバーのみ閲覧可能
- チャネル名: 半角英数字とハイフンのみ、最大80文字
- チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ
- ユーザーは任意のパブリックチャネルに自由参加できる
- ユーザーは参加中のチャネルから退出できる
- エッジケース: 最後の管理者は退出前に他メンバーに管理権限を委譲する必要がある

## 4. メッセージ
- ユーザーはチャネル内にテキストメッセージを投稿できる
- 文字数上限: 最大4000文字
- メッセージには投稿時刻と投稿者が記録される
- 投稿後も時系列で永続表示される
- 本文でMarkdown記法が利用可能(太字、斜体、コードブロック、リンク)

## 5. メンション
- メッセージ内で `@ユーザー名` を入力するとそのユーザーをメンションできる
- メンションされたユーザーには通知が届く(通知設定に従う)
- `@channel`: 同一チャネルの全メンバーをメンション
- `@here`: 同一チャネルでオンラインの全メンバーをメンション
- `@channel` および `@here` の使用はチャネル管理者によって制限可能

## 6. ダイレクトメッセージ (DM)
- 同一ワークスペース内の任意のユーザーにDMを送信できる
- 1対1の会話のみサポート(グループDMはスコープ外)
- DM内のメッセージはチャネルメッセージと同じ仕様(4000文字上限・Markdown対応)
- DMは送信者と受信者のみが閲覧可能
- ワークスペース管理者であっても他人のDMは閲覧できない

## 7. 検索
- 横断検索対象: メッセージ本文・チャネル名・ユーザー名
- 検索結果はメッセージ・チャネル・ユーザーのカテゴリ別に表示
- 検索対象は本人がアクセス権を持つチャネルとDMに限定
- 絞り込みフィルタ:
  - `from:ユーザー名` 投稿者で絞り込み
  - `in:チャネル名` チャネルで絞り込み
  - `before:YYYY-MM-DD` / `after:YYYY-MM-DD` 期間で絞り込み
- 並び替え: 関連度順またはタイムスタンプ降順
- 1クエリあたり最大1000件まで返却、それ以上はページングで取得

## 8. 通知
- チャネル通知レベルは3種類:
  - すべてのメッセージ
  - メンション時のみ
  - なし
- DMは常にメンション扱いで通知される
- 通知レベルはチャネル単位、およびワークスペース全体のデフォルトとして設定可能
- ワークスペースのデフォルトはチャネル個別設定で上書きされる
- 「通知を一時停止」機能: 30分・1時間・2時間・翌朝9時まで全通知を抑止できる
- 通知抑止中もアプリ内バッジ表示は継続する

## 9. 非機能要件
- 同時接続ユーザー数: 1ワークスペースあたり最大500
- メッセージ送信レイテンシ: 送信から表示まで95パーセンタイルで500ms以内
- 検索応答時間: 95パーセンタイルで1秒以内(1000万メッセージ規模のワークスペース)
- データ永続性: メッセージは無期限保存(削除ポリシーは別途定義)
- 対応ブラウザ: 最新版のChrome / Safari / Firefox / Edge
- アクセシビリティ: WCAG 2.1 Level AA 準拠

## 10. 技術スタック制約
- フロントエンド: TypeScript + React 18以上、状態管理はZustand、ビルドはVite
- バックエンド: Go 1.22以上、HTTPフレームワークはchi、認証はJWT (RS256)
- データストア:
  - PostgreSQL 16: ユーザー・チャネル・メッセージ等の主データ
  - Redis 7: セッションストア、Pub/Sub(リアルタイム配信)、レートリミット
  - Elasticsearch 8: 全文検索インデックス(本文・チャネル名・ユーザー名)
- リアルタイム通信: WebSocket (RFC 6455準拠)、ハートビートは30秒間隔
- インフラ: AWS東京リージョン (ap-northeast-1)、IaCはTerraform、コンテナはECS Fargate
- CI/CD: GitHub Actions、デプロイはBlue/Greenでダウンタイム0
- 監視: Datadogで APM・メトリクス・ログを統合監視

## 11. セキュリティ要件
- 通信はすべてTLS 1.2以上で暗号化される
- パスワードはbcrypt (cost=12) でハッシュ化して保存
- セッションは絶対24時間でタイムアウトし、再認証が必要

## 用語(原文に登場した用語)
- **ワークスペース**: チームの単位、URLサブドメインで識別される
- **チャネル**: ワークスペース内のトピック単位の会話空間(パブリック/プライベート)
- **プレゼンス**: ユーザーの状態(オンライン/退席中/オフライン)
- **メンション**: `@ユーザー名` / `@channel` / `@here`
- **DM(ダイレクトメッセージ)**: 1対1の会話
- **管理者(チャネル管理者・ワークスペース管理者)**: チャネル作成者は初期管理権限を持つ。`@channel`/`@here` 制限権限はチャネル管理者
- **通知レベル**: すべて / メンション時のみ / なし

## ビジネスルール / エッジケース(原文記載のもの)
- 1メールアドレス = 1アカウント
- チャネル名は半角英数字とハイフンのみ・最大80文字
- 最後の管理者は退出前に管理権限委譲が必要
- DMは管理者でも他人のものを閲覧不可
- ワークスペースのデフォルト通知設定はチャネル個別設定で上書き
- 通知抑止中でもアプリ内バッジ表示は継続
- 検索は本人にアクセス権のあるチャネル/DMに限定
- 検索は1クエリ1000件上限、超過分はページング
- グループDMはスコープ外
- メッセージ削除ポリシーは別途定義(本書では未定義)
