要件定義書「チャットツール『Linkr』要件定義書 (Extended)」の内容を、原文に記載されている事項のみを、章立てに沿ってすべて列挙します。

---

## 1. 概要
- 社内コミュニケーション用のチャットツール
- 本書は「コア機能」「コミュニケーション支援機能」「拡張機能」を定義する

## 2. ユーザーとワークスペース
- ユーザーは1つのメールアドレスにつき1アカウントを保持する
- 各ユーザーは複数のワークスペースに所属できる
- ワークスペースはチームの単位であり、URLサブドメインで識別される(例: `acme.linkr.app`)
- ワークスペース内のユーザーは表示名とアバター画像を設定できる
- プレゼンス状態は「オンライン / 退席中 / オフライン」の3種類
- 10分間操作がない場合、自動的に「退席中」となる
- ステータスメッセージ:絵文字 + 最大100文字のテキスト(任意設定)
- ステータスメッセージの有効期限:1時間 / 4時間 / 終日 / カスタム / 無期限

## 3. ロールと権限
- 3つのロール:
  - **オーナー**:全権限。請求設定の変更、削除、他オーナー任命が可能
  - **管理者**:ユーザー招待、チャネル管理、ロール付与(オーナー以外)が可能
  - **メンバー**:通常利用権限のみ
- ロール変更はオーナーまたは管理者が可能(オーナーへの昇格はオーナーのみ)
- ワークスペースには常に1名以上のオーナーが必要
- 退会するオーナーは事前に他ユーザーへオーナー権限を委譲する必要がある

## 4. チャネル
- チャネル = ワークスペース内のトピック単位の会話空間
- 種類:パブリック(全員閲覧・参加可) / プライベート(招待者のみ閲覧可)
- チャネル名:半角英数字とハイフンのみ、最大80文字
- チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ
- ユーザーは任意のパブリックチャネルに自由参加できる
- 参加中のチャネルから退出可能
  - ただし最後の管理者は退出前に他メンバーへ管理権限を委譲する必要がある
- トピック:最大250文字、説明:最大1500文字
- アーカイブ:
  - アーカイブ後は新規メッセージ投稿不可、閲覧と検索は可能
  - 実行はチャネル管理者またはワークスペース管理者
  - 復元はワークスペース管理者が可能

## 5. メッセージ
- ユーザーはチャネル内にテキストメッセージを投稿可能
- 最大4000文字
- 投稿時刻と投稿者が記録される
- 投稿後も時系列で永続表示される
- Markdown記法(太字、斜体、コードブロック、リンク、引用、リスト)が利用可能

### 5.1 編集と削除
- 自分のメッセージは編集可能
- 編集されたメッセージには「編集済み」表示が付く
- 編集履歴は保存されないが、編集日時は記録される
- 自分のメッセージは削除可能
- チャネル管理者は他人のメッセージを削除可能(編集は不可)
- 削除されたメッセージは「このメッセージは削除されました」というプレースホルダ表示
- 削除はソフトデリート、削除ログは90日間保管

### 5.2 スレッド
- 任意のメッセージへの返信でスレッドを開始可能
- スレッド内メッセージはチャネルのメインタイムラインには表示されない
- スレッド開始者および参加者は新規スレッド返信時に通知を受ける
- 「チャネルにも投稿」オプションで親チャネルへ二重投稿可能
- スレッド内メッセージにもメンション・編集・削除・リアクションが可能

### 5.3 リアクション
- メッセージに絵文字リアクション付与可能
- 1メッセージあたり最大23種類の絵文字を表示
- 同一ユーザーが同一メッセージに同じ絵文字で複数回リアクション不可
- カスタム絵文字はワークスペース管理者がアップロード可能、全ユーザーが使用可能
- カスタム絵文字:1ファイル128KB以下、PNG / JPEG / GIF形式

## 6. メンション
- `@ユーザー名`でメンション可能
- メンションされたユーザーには通知が届く(通知設定に従う)
- `@channel`:同一チャネル全メンバーをメンション
- `@here`:同一チャネルでオンラインの全メンバーをメンション
- `@channel`および`@here`の使用はチャネル管理者によって制限可能
- ユーザーグループ(例:`@team-frontend`)を作成可能
  - 作成・所属メンバー管理はオーナーまたは管理者
  - グループメンションはグループ内全員へのメンション扱い

## 7. ファイル共有
- メッセージにファイル添付可能
- 1ファイル最大100MB
- 1メッセージあたり最大10ファイル添付可能
- 対応形式:画像(PNG/JPEG/GIF/WebP)、PDF、動画(MP4/MOV)、各種ドキュメント、ZIPアーカイブ
- 画像とPDFはインラインプレビュー表示
- アクセス権を持つチャネル参加者のみ閲覧可能
- 投稿者またはチャネル管理者が削除可能
- 削除後は即座にダウンロード不可、メタデータは90日間保持

## 8. ダイレクトメッセージ (DM)
- 同一ワークスペース内の任意ユーザーにDM送信可能
- 1対1の会話のみサポート(グループDMは本書のスコープ外)
- DM内メッセージはチャネルメッセージと同じ仕様(4000文字上限・Markdown対応・編集・削除・スレッド・リアクション・ファイル添付すべて利用可能)
- DMは送信者と受信者のみ閲覧可能、ワークスペース管理者でも他人のDMは閲覧不可

## 9. 検索
- メッセージ本文・チャネル名・ユーザー名・ファイル名を横断検索可能
- 検索結果はメッセージ・チャネル・ユーザー・ファイルのカテゴリ別に表示
- 検索対象は本人がアクセス権を持つチャネルとDMに限定
- 絞り込みフィルタ:
  - `from:ユーザー名` 投稿者で絞り込み
  - `in:チャネル名` チャネルで絞り込み
  - `before:YYYY-MM-DD` / `after:YYYY-MM-DD` 期間で絞り込み
  - `has:link` / `has:file` / `has:reaction` 添付要素で絞り込み
- 並び替え:関連度順またはタイムスタンプ降順
- 1クエリあたり最大1000件返却、それ以上はページング

## 10. 通知
- チャネル通知レベル3種類:すべてのメッセージ / メンション時のみ / なし
- DMは常にメンション扱いで通知
- 通知レベルはチャネル単位、およびワークスペース全体のデフォルトとして設定可能
- ワークスペースのデフォルトはチャネル個別設定で上書きされる
- 「通知を一時停止」:30分・1時間・2時間・翌朝9時まで全通知を抑止
- 通知抑止中もアプリ内バッジ表示は継続
- キーワード通知:ユーザーのキーワードリストに含まれた場合に通知
- 通知方法:ブラウザ通知、メール、モバイルプッシュ(モバイルアプリは別書で定義)
- 優先順位:ブラウザ未開封時はメール、5分以内に既読されない場合のみメールを送る

## 11. ピン留めとブックマーク
- チャネル管理者またはチャネルメンバーは重要メッセージをチャネル内で「ピン留め」可能
- ピン留めはチャネルあたり最大100件
- サイドパネルから一覧アクセス可能
- 個人ブックマーク:任意のメッセージを保存可能、本人のみ閲覧可能
- 個人ブックマーク数の上限はないが、500件超でフォルダ分類が推奨される

## 12. 下書きと送信予約
- 入力中の内容は自動的に下書きとして保存
- 下書きはチャネル / DM / スレッド単位で保持
- アプリ再起動後も復元
- 送信予約:最大120日先までの任意の日時に送信予約可能
- 予約済みメッセージは送信前に編集または取り消し可能
- 「送信予約一覧」画面から確認可能

## 13. 既読管理
- メッセージは閲覧者単位で既読 / 未読状態を保持
- チャネル一覧では未読のあるチャネルが太字で表示
- 「未読をすべて既読にする」アクション:チャネル単位またはワークスペース単位で一括既読化
- 既読位置(最終既読メッセージID)はサーバーで保存され、デバイス間で同期

## 14. リマインダー
- 任意のメッセージにリマインダー設定可能
- プリセット時刻:10分後 / 1時間後 / 明日朝9時 / 来週月曜朝9時、またはカスタム指定
- リマインダー時刻にリマインダーBotから本人にDMで通知
- リマインダーDMには元メッセージへのリンクが含まれる
- 「リマインダー」タブから確認・取り消し可能
- 同一メッセージへのリマインダーはユーザー1人につき1件まで(再設定で上書き)

## 15. 投票
- チャネル内で投票メッセージを作成可能
- 質問文:最大200文字、選択肢:最大10個(各最大80文字)
- 形式:単一選択 / 複数選択
- 締切:無期限・1時間後・1日後・カスタム日時
- 結果表示:集計のみ / 投票したユーザー名を公開、を作成時に選択
- 作成者は締切前に投票を強制終了可能
- 締切後は新規投票不可だが結果閲覧可能

## 16. 音声メッセージ
- メッセージ入力欄から録音・送信可能
- 最大長5分
- 録音形式:Opus(WebMコンテナ)
- 受信側はインラインプレーヤーで再生、再生速度0.5x / 1x / 1.5x / 2x選択可能
- 自動文字起こしの対象、結果はメッセージ下に折りたたみ表示
- ファイル添付の対象として保存ポリシーが適用される

## 17. 通知ダイジェスト
- メール通知ダイジェストを有効化可能
- 頻度:1日2回(朝9時・夕方17時) / 1日1回(朝9時) / 無効
- 内容:未読のメンション・キーワードヒット・DMをチャネル別に集約
- 各メッセージへの直接リンクを含む
- 新規メンションがない場合はメール送信しない
- ワークスペース管理者がワークスペース単位で無効化可能

## 18. 監査と履歴
- ワークスペース管理者が監査ログで確認できるイベント:
  - ユーザーのログイン / ログアウト
  - ロール変更
  - チャネル作成 / アーカイブ / 復元
  - ユーザー招待 / 退会
- 監査ログ保持:90日間(拡張は別書のエンタープライズ機能で定義)
- 通常メンバーは自身の操作履歴を過去30日分閲覧可能

## 19. 非機能要件
- 同時接続ユーザー数:1ワークスペースあたり最大2000
- メッセージ送信レイテンシ:送信から表示まで95パーセンタイルで500ms以内
- 検索応答時間:95パーセンタイルで1秒以内(1000万メッセージ規模のワークスペース)
- ファイルアップロード:100MBファイルが30秒以内(回線100Mbps想定)
- データ永続性:メッセージは無期限保存(削除ポリシーは別途定義)
- 対応ブラウザ:最新版のChrome / Safari / Firefox / Edge
- アクセシビリティ:WCAG 2.1 Level AA準拠
- 国際化:日本語、英語、簡体字中国語、韓国語に対応(UI翻訳)

## 20. 技術スタック制約
- **フロントエンド**:TypeScript + React 18以上、状態管理Zustand、ビルドVite、ルーティングReact Router 6
- **バックエンド**:Go 1.22以上、HTTPフレームワークchi、認証JWT(RS256)、ORMはsqlc
- **データストア**:
  - PostgreSQL 16(主データ、リードレプリカ最低2台)
  - Redis 7(セッション、Pub/Sub、レートリミット)
  - Elasticsearch 8(全文検索)
  - S3(ファイル添付、署名付きURLでアクセス制御)
- **リアルタイム通信**:WebSocket (RFC 6455)、Redis Pub/Subでサーバー間ファンアウト
- **メッセージキュー**:通知配信・監査ログ書き込みはAmazon SQSで非同期処理
- **インフラ**:AWS東京リージョン(ap-northeast-1)、IaCはTerraform、コンテナはECS Fargate
- **CI/CD**:GitHub Actions、Blue/Greenデプロイ、E2EテストはPlaywright
- **監視**:Datadog(APM/Logs)、PagerDutyでオンコール通知
- **セキュリティ**:WAFはAWS WAF、依存脆弱性スキャンはSnyk + Dependabot

## 21. セキュリティ要件
- 通信はすべてTLS 1.2以上で暗号化
- パスワードはbcrypt(cost=12)でハッシュ化して保存
- セッションは絶対24時間でタイムアウト、再認証が必要
- 全APIにレートリミット(IP単位とユーザー単位の両方)を適用
- 監査ログは改竄防止のため別アカウントのS3バケットにWORM(Write Once Read Many)で保存
