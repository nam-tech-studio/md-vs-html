# チャットツール「Linkr」要件定義書 (Extended) — 全項目列挙

## 1. 概要
- 社内コミュニケーション用のチャットツール
- 本書で定義する範囲: コア機能 / コミュニケーション支援機能 / 拡張機能

## 2. ユーザーとワークスペース
- ユーザーは1つのメールアドレスにつき1アカウント
- 各ユーザーは複数のワークスペースに所属できる
- ワークスペース = チームの単位、URLサブドメインで識別 (例: `acme.linkr.app`)
- ワークスペース内ユーザーは表示名とアバター画像を設定可能
- プレゼンス状態は3種類: オンライン / 退席中 / オフライン
- 10分間操作なしで自動的に「退席中」へ遷移
- ステータスメッセージは絵文字 + 最大100文字のテキスト、任意設定
- ステータスメッセージの有効期限は5種: 1時間 / 4時間 / 終日 / カスタム / 無期限

## 3. ロールと権限
- ロールは3種: **オーナー** / **管理者** / **メンバー**
  - **オーナー**: 全権限。請求設定変更、削除、他オーナー任命可能
  - **管理者**: ユーザー招待、チャネル管理、ロール付与 (オーナー以外) 可能
  - **メンバー**: 通常利用権限のみ
- ロール変更はオーナーまたは管理者が可能
- オーナーへの昇格はオーナーのみ可能
- ワークスペースには常に1名以上のオーナーが必要
- 退会するオーナーは事前に他ユーザーへオーナー権限を委譲する必要あり

## 4. チャネル
- チャネル = ワークスペース内のトピック単位の会話空間
- 種類は2つ: パブリック / プライベート
  - パブリック: ワークスペース全員が閲覧・参加可能
  - プライベート: 招待メンバーのみ閲覧可能
- チャネル名: 半角英数字とハイフンのみ、最大80文字
- チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ
- ユーザーは任意のパブリックチャネルに自由参加可能
- 自分が参加中のチャネルから退出可能
  - 最後の管理者は退出前に他メンバーへ管理権限を委譲する必要あり
- トピック: 最大250文字、説明: 最大1500文字
- アーカイブ機能あり
  - アーカイブ後: 新規投稿不可、閲覧と検索は可能
  - アーカイブ実行: チャネル管理者またはワークスペース管理者
- アーカイブ済みチャネルの復元: ワークスペース管理者が可能

## 5. メッセージ
- チャネル内にテキストメッセージ投稿可能
- メッセージ最大4000文字
- 投稿時刻と投稿者が記録される
- 投稿後も時系列で永続表示
- Markdown対応: 太字、斜体、コードブロック、リンク、引用、リスト

### 5.1 編集と削除
- 自分のメッセージを編集可能
- 編集後は「編集済み」表示が付与
- 編集履歴は保存されないが、編集日時は記録される
- 自分のメッセージを削除可能
- チャネル管理者は他人のメッセージを削除可能 (編集は不可)
- 削除メッセージは「このメッセージは削除されました」プレースホルダ表示
- ソフトデリート方式、削除ログは90日間保管

### 5.2 スレッド
- 任意のメッセージへの返信でスレッド開始
- スレッド内メッセージはチャネルのメインタイムラインに表示されない
- スレッド開始者および参加者は新規スレッド返信時に通知を受ける
- 「チャネルにも投稿」オプションで親チャネルへの二重投稿可能
- スレッド内メッセージにもメンション・編集・削除・リアクション可能

### 5.3 リアクション
- 絵文字リアクション付与可能
- 1メッセージあたり最大23種類の絵文字を表示
- 同一ユーザーが同一メッセージに同じ絵文字を複数回リアクション不可
- カスタム絵文字: ワークスペース管理者がアップロード、全ユーザー利用可
- カスタム絵文字制約: 1ファイル128KB以下、PNG / JPEG / GIF形式

## 6. メンション
- `@ユーザー名` でユーザーをメンション
- メンション通知は通知設定に従う
- `@channel`: 同一チャネル全メンバーへメンション
- `@here`: 同一チャネルでオンラインの全メンバーへメンション
- `@channel` および `@here` の使用はチャネル管理者によって制限可能
- ユーザーグループ作成可能 (例: `@team-frontend`)
  - ユーザーグループはオーナーまたは管理者が作成、所属メンバー管理
  - グループメンションはグループ内全員へのメンション扱い

## 7. ファイル共有
- メッセージにファイル添付可能
- 1ファイル最大100MB
- 1メッセージあたり最大10ファイル添付可能
- 対応形式: 画像 (PNG/JPEG/GIF/WebP)、PDF、動画 (MP4/MOV)、各種ドキュメント、ZIPアーカイブ
- 画像とPDFはインラインプレビュー表示
- アクセス権を持つチャネル参加者のみ閲覧可能
- 削除可能者: 投稿者またはチャネル管理者
- 削除後はダウンロード即時不可、メタデータは90日間保持

## 8. ダイレクトメッセージ (DM)
- 同一ワークスペース内の任意ユーザーへDM送信可能
- 1対1のみサポート (グループDMは本書スコープ外)
- DM内メッセージはチャネルメッセージと同仕様 (4000文字上限・Markdown・編集・削除・スレッド・リアクション・ファイル添付すべて利用可)
- 送信者と受信者のみ閲覧可、ワークスペース管理者でも他人のDMは閲覧不可

## 9. 検索
- 横断検索対象: メッセージ本文・チャネル名・ユーザー名・ファイル名
- 検索結果はカテゴリ別表示: メッセージ / チャネル / ユーザー / ファイル
- 検索対象は本人がアクセス権を持つチャネルとDMに限定
- 絞り込みフィルタ:
  - `from:ユーザー名` 投稿者で絞り込み
  - `in:チャネル名` チャネルで絞り込み
  - `before:YYYY-MM-DD` / `after:YYYY-MM-DD` 期間で絞り込み
  - `has:link` / `has:file` / `has:reaction` 添付要素で絞り込み
- 並び替え: 関連度順またはタイムスタンプ降順
- 1クエリ最大1000件、それ以上はページング

## 10. 通知
- チャネル通知レベル3種: すべてのメッセージ / メンション時のみ / なし
- DMは常にメンション扱いで通知
- 通知レベルはチャネル単位、およびワークスペース全体のデフォルトとして設定可能
- ワークスペースのデフォルトはチャネル個別設定で上書きされる
- 「通知を一時停止」: 30分・1時間・2時間・翌朝9時まで全通知抑止可能
- 通知抑止中もアプリ内バッジ表示は継続
- キーワード通知: 自分のキーワードリストを設定し、ヒット時に通知
- 通知方法: ブラウザ通知、メール、モバイルプッシュ (モバイルアプリは別書定義)
- 優先順位: ブラウザ未開封時はメール、5分以内に既読されない場合のみメール送信

## 11. ピン留めとブックマーク
- チャネル管理者またはチャネルメンバーは重要メッセージをチャネル内で「ピン留め」可能
- ピン留めはチャネルあたり最大100件
- ピン留めはチャネル内サイドパネルから一覧アクセス可能
- 個人ブックマークとして任意のメッセージを保存可能
  - 本人のみ閲覧可能
  - 上限なし、ただし500件超でフォルダ分類が推奨される

## 12. 下書きと送信予約
- メッセージ入力中の内容は自動的に下書き保存
- 下書きはチャネル / DM / スレッド単位で保持
- 下書きはアプリ再起動後も復元される
- 送信予約は最大120日先まで可能
- 予約済みメッセージは送信前に編集または取り消し可能
- 予約済みメッセージは「送信予約一覧」画面から確認可能

## 13. 既読管理
- 各メッセージは閲覧者単位で既読 / 未読状態を保持
- チャネル一覧では未読のあるチャネルが太字表示
- 「未読をすべて既読にする」: チャネル単位またはワークスペース単位で一括既読化可能
- 既読位置 (最終既読メッセージID) はサーバー保存、デバイス間同期

## 14. リマインダー
- 任意のメッセージに対してリマインダー設定可能
- プリセット時刻: 10分後 / 1時間後 / 明日朝9時 / 来週月曜朝9時、またはカスタム指定可
- 時刻になるとリマインダーBotから本人にDMで通知
- リマインダーDMには元メッセージへのリンクが含まれる
- 「リマインダー」タブから一覧確認・取り消し可能
- 同一メッセージへのリマインダーはユーザー1人につき1件、再設定で上書き

## 15. 投票
- チャネル内で投票メッセージを作成可能
- 質問文最大200文字、選択肢最大10個 (各最大80文字)
- 形式は「単一選択」または「複数選択」
- 締切: 無期限 / 1時間後 / 1日後 / カスタム日時
- 結果表示: 集計のみ、または各選択肢の投票者名を公開、を作成時に選択
- 作成者は締切前に投票を強制終了可能
- 締切後は新規投票不可、結果閲覧は可能

## 16. 音声メッセージ
- メッセージ入力欄から録音・送信可能
- 最大長5分
- 保存形式: Opus (WebMコンテナ)
- 受信側のインラインプレーヤーで再生、再生速度 0.5x / 1x / 1.5x / 2x 選択可
- 自動文字起こし対象、結果はメッセージ下に折りたたみ表示
- ファイル添付の保存ポリシーが適用される

## 17. 通知ダイジェスト
- メール通知ダイジェスト有効化可能
- 頻度: 1日2回 (朝9時・夕方17時) / 1日1回 (朝9時) / 無効
- 内容: 未読のメンション・キーワードヒット・DM をチャネル別に集約
- 各メッセージへの直接リンクを含む
- 新規メンションがない場合はメール送信しない
- ワークスペース管理者がワークスペース単位で無効化可能

## 18. 監査と履歴
- ワークスペース管理者が監査ログで確認できるイベント:
  - ユーザーのログイン / ログアウト
  - ロール変更
  - チャネル作成 / アーカイブ / 復元
  - ユーザー招待 / 退会
- 監査ログ保持期間: 90日間 (拡張は別書のエンタープライズ機能で定義)
- 通常メンバーは自身の操作履歴のみ過去30日分閲覧可能

## 19. 非機能要件
- 同時接続ユーザー数: 1ワークスペースあたり最大2000
- メッセージ送信レイテンシ: 送信から表示まで p95 で 500ms 以内
- 検索応答時間: p95 で 1秒以内 (1000万メッセージ規模のワークスペース)
- ファイルアップロード: 100MB が 30秒以内 (100Mbps回線想定)
- データ永続性: メッセージは無期限保存 (削除ポリシーは別途定義)
- 対応ブラウザ: 最新版の Chrome / Safari / Firefox / Edge
- アクセシビリティ: WCAG 2.1 Level AA 準拠
- 国際化: 日本語、英語、簡体字中国語、韓国語 (UI翻訳)

## 20. 技術スタック制約
- フロントエンド: TypeScript + React 18以上、状態管理 Zustand、ビルド Vite、ルーティング React Router 6
- バックエンド: Go 1.22以上、HTTPフレームワーク chi、認証 JWT (RS256)、ORM sqlc
- データストア:
  - PostgreSQL 16 (主データ、リードレプリカ最低2台)
  - Redis 7 (セッション、Pub/Sub、レートリミット)
  - Elasticsearch 8 (全文検索)
  - S3 (ファイル添付、署名付きURLでアクセス制御)
- リアルタイム通信: WebSocket (RFC 6455)、Redis Pub/Sub でサーバー間ファンアウト
- メッセージキュー: 通知配信・監査ログ書き込みは Amazon SQS で非同期処理
- インフラ: AWS東京リージョン (ap-northeast-1)、IaC は Terraform、コンテナは ECS Fargate
- CI/CD: GitHub Actions、Blue/Greenデプロイ、E2Eテストは Playwright
- 監視: Datadog (APM/Logs)、PagerDuty でオンコール通知
- セキュリティ: WAF は AWS WAF、依存脆弱性スキャンは Snyk + Dependabot

## 21. セキュリティ要件
- 通信はすべて TLS 1.2 以上で暗号化
- パスワードは bcrypt (cost=12) でハッシュ化保存
- セッションは絶対24時間でタイムアウト、再認証必要
- 全APIにレートリミット (IP単位とユーザー単位の両方) を適用
- 監査ログは改竄防止のため別アカウントのS3バケットに WORM (Write Once Read Many) で保存

## 横断的に出てくる主な数値・上限値の一覧
- ステータスメッセージ: 絵文字 + 最大100文字
- 自動「退席中」: 10分操作なし
- チャネル名: 最大80文字 (半角英数字とハイフン)
- チャネルトピック: 最大250文字 / 説明: 最大1500文字
- メッセージ: 最大4000文字
- 削除ログ保管: 90日間
- リアクション絵文字: 1メッセージ最大23種類
- カスタム絵文字: 1ファイル128KB以下 (PNG/JPEG/GIF)
- ファイル: 1ファイル100MB / 1メッセージ最大10ファイル
- ファイルメタデータ保持: 削除後90日間
- 検索結果: 1クエリ最大1000件
- 通知一時停止: 30分 / 1時間 / 2時間 / 翌朝9時
- メール送信判定: 5分以内に既読されない場合
- ピン留め: チャネルあたり最大100件
- 個人ブックマーク: 上限なし、500件超でフォルダ推奨
- 送信予約: 最大120日先まで
- 投票: 質問最大200文字 / 選択肢最大10個 (各最大80文字)
- 音声メッセージ: 最大5分
- 監査ログ保持: 90日間
- メンバー操作履歴閲覧: 過去30日分
- 同時接続: 1ワークスペース最大2000
- メッセージ送信レイテンシ: p95 500ms以内
- 検索応答: p95 1秒以内 (1000万メッセージ規模)
- ファイルアップロード: 100MB / 30秒以内 (100Mbps想定)
- セッションタイムアウト: 絶対24時間
- bcrypt cost: 12

## 明示されているエッジケース・ビジネスルール
- ワークスペースには常に1名以上のオーナーが必要 → 退会オーナーは事前委譲必須
- オーナー昇格はオーナーのみ可能 (管理者は不可)
- 最後の管理者がチャネル退出する場合は他メンバーに管理権限委譲が必要
- アーカイブ済みチャネルでも閲覧と検索は可能、復元はワークスペース管理者のみ
- 削除済みメッセージはプレースホルダ表示、ソフトデリート
- 編集履歴は保存されない (編集日時のみ記録)
- 同一ユーザーが同一メッセージに同じ絵文字で複数回リアクションは不可
- DM はワークスペース管理者でも他人のものは閲覧不可
- 通知抑止中もアプリ内バッジ表示は継続
- 通知優先順位: ブラウザ未開封 → メール (ただし5分以内に既読されない場合のみ)
- 締切後の投票は新規投票不可だが結果閲覧可
- 同一メッセージへのリマインダーはユーザー1人につき1件 (再設定で上書き)
- ダイジェストメールは新規メンションがなければ送信しない
- 監査ログは別アカウントのS3に WORM 保存
