## 1. 概要
- 社内コミュニケーション用のチャットツール。
- 本書はコア機能、コミュニケーション支援機能、拡張機能、エンタープライズ向け管理・統合・コンプライアンス機能を定義する。

## 2. ユーザーとアカウント
- 1メールアドレスにつき1アカウント。
- 各ユーザーは複数ワークスペースに所属可能。
- 認証はパスワード認証またはSSO。
- パスワードは最低12文字、英大文字・小文字・数字・記号を各1文字以上含む。
- パスワードリセット: 登録メールへトークン送信、有効期限60分。
- 連続5回ログイン失敗で15分ロック。
- プレゼンス状態: オンライン / 退席中 / オフラインの3種類。
- 10分間操作なしで自動「退席中」化。
- ステータスメッセージ: 絵文字 + 最大100文字、有効期限は1時間 / 4時間 / 終日 / カスタム / 無期限から選択。
- 2要素認証: TOTP / SMS / WebAuthn。
- ワークスペースオーナーは2要素認証を組織全体に強制可能。

## 3. ワークスペース
- チームの単位、URLサブドメインで識別 (例: `acme.linkr.app`)。
- 表示名・アバター画像はユーザー設定可能。
- ワークスペース名・ロゴ・アイコンはオーナーが変更可能。
- 契約プランごとに機能制限が異なる。
  - **Free**: ユーザー50名まで、メッセージ履歴は直近10000件まで閲覧可能。
  - **Pro**: ユーザー無制限、メッセージ履歴無制限。
  - **Enterprise**: SSO、SCIM、データ常駐リージョン指定、エンタープライズキー管理、監査ログ拡張。
- 削除実行から30日間は復元可能 (オーナーのみ)。
- 30日経過後は完全消去。

## 4. ロールと権限
- ロールは3種類。
  - **オーナー**: 全権限、請求設定変更・削除・他オーナー任命可。
  - **管理者**: ユーザー招待、チャネル管理、ロール付与 (オーナー以外)。
  - **メンバー**: 通常利用権限のみ。
- ロール変更はオーナーまたは管理者 (オーナーへの昇格はオーナーのみ)。
- 常に1名以上のオーナーが必要。
- 退会するオーナーは事前に他ユーザーへ権限委譲が必要。
- ゲストユーザー: 特定チャネルのみアクセス可、DM・検索・他チャネル閲覧不可、招待はオーナーまたは管理者のみ。

## 5. 招待とユーザー管理
- 招待はメールアドレス指定、管理者・オーナーのみ。
- 招待リンクの有効期限は7日間。
- 一度に最大100件まとめて招待可。
- ドメイン制限: 許可リストでメールドメインを制限可能。
- 無効化: ログイン不可、過去メッセージ・名前は残る。
- 完全削除: 投稿は「削除済みユーザー」として残るがアカウント情報は完全消去。
- SCIM v2.0でのプロビジョニング (Enterpriseプランのみ)。

## 6. SSO
- Enterpriseプランで有効化可能。
- 対応プロトコル: SAML 2.0、OIDC。
- SSO有効時にパスワード認証無効化可能 (オーナー設定)。
- SSOプロバイダ属性 (名前・メール・ユーザーグループ) を自動反映。
- JIT (ジャストインタイムプロビジョニング): SSO初回ログイン時に自動アカウント作成可。
- SCIMと併用時はユーザー作成・無効化はSCIM側を優先。

## 7. チャネル
- ワークスペース内のトピック単位会話空間。
- パブリック (全員閲覧・参加可) / プライベート (招待メンバーのみ)。
- チャネル名は半角英数字とハイフンのみ、最大80文字。
- 作成者は自動メンバーかつ初期管理権限を持つ。
- 任意のパブリックチャネルへ自由参加可能。
- 退出可能だが、最後の管理者は事前に管理権限委譲が必要。
- トピックは最大250文字、説明は最大1500文字。
- アーカイブ可: 新規投稿不可、閲覧・検索は可能、チャネル管理者またはワークスペース管理者が実行。
- アーカイブ復元はワークスペース管理者が実行可能。
- 共有チャネル (Connect): Enterpriseプラン、他ワークスペースとチャネル共有、参加者は自身のワークスペースアカウントで投稿、保存期間ポリシーはどちらを適用するか明示選択可。

## 8. メッセージ
- チャネル内テキスト投稿、最大4000文字。
- 投稿時刻と投稿者を記録、時系列で永続表示。
- Markdown対応 (太字、斜体、コードブロック、リンク、引用、リスト)。

### 8.1 編集と削除
- 自分のメッセージは編集可、編集済み表示が付く。
- 編集履歴は保存しないが編集日時は記録。
- 自分のメッセージは削除可。チャネル管理者は他人のメッセージを削除可 (編集は不可)。
- 削除メッセージは「このメッセージは削除されました」と表示。
- ソフトデリート、削除ログは90日間保管。

### 8.2 スレッド
- 任意メッセージへの返信でスレッド開始。
- スレッド内メッセージはチャネルメインタイムラインに表示されない。
- 開始者・参加者は新規返信時に通知。
- 「チャネルにも投稿」オプションで親チャネルへ二重投稿可。
- スレッド内でもメンション・編集・削除・リアクション可。

### 8.3 リアクション
- 絵文字リアクション付与可。
- 1メッセージあたり最大23種類の絵文字を表示。
- 同一ユーザーが同一メッセージに同じ絵文字を複数回リアクション不可。
- カスタム絵文字はワークスペース管理者がアップロード、全員リアクション使用可。
- カスタム絵文字: 1ファイル128KB以下、PNG/JPEG/GIF。

## 9. メンション
- `@ユーザー名` でメンション、通知設定に従って通知。
- `@channel`: 同一チャネル全員へメンション。
- `@here`: 同一チャネルのオンライン全員へメンション。
- `@channel`・`@here` 使用はチャネル管理者が制限可能。
- ユーザーグループ (例: `@team-frontend`) を作成可、オーナーまたは管理者が作成・管理、メンションはグループ全員へ。

## 10. ファイル共有
- メッセージにファイル添付可。
- 1ファイル最大100MB (Enterpriseは最大5GB)。
- 1メッセージあたり最大10ファイル。
- 対応形式: 画像 (PNG/JPEG/GIF/WebP)、PDF、動画 (MP4/MOV)、各種ドキュメント、ZIP。
- 画像・PDFはインラインプレビュー。
- ファイルはアクセス権を持つチャネル参加者のみ閲覧可。
- 投稿者またはチャネル管理者が削除可能。
- 削除後は即ダウンロード不可、メタデータは90日保持。
- ウイルススキャン: アップロード時自動、検知時は隔離。

## 11. ダイレクトメッセージ (DM)
- 同一ワークスペース内の任意ユーザーへDM送信可。
- DMは1対1のみ。
- マルチパーティDM (MPDM): 最大8名、名前なし、一時的会話。
- DM内メッセージはチャネルと同仕様。
- 送受信者のみ閲覧可、ワークスペース管理者でも他人のDMは閲覧不可。
- DMは検索対象だが本人参加のDMのみ結果に含む。

## 12. 検索
- メッセージ本文・チャネル名・ユーザー名・ファイル名を横断検索。
- 結果はメッセージ・チャネル・ユーザー・ファイル別に表示。
- 検索範囲は本人がアクセス権を持つチャネルとDMのみ。
- フィルタ: `from:ユーザー名`、`in:チャネル名`、`before:YYYY-MM-DD`/`after:YYYY-MM-DD`、`has:link`/`has:file`/`has:reaction`。
- 並び替え: 関連度順またはタイムスタンプ降順。
- 1クエリあたり最大1000件、超過分はページング。

## 13. 通知
- チャネル通知レベル3種類: すべて / メンション時のみ / なし。
- DMは常にメンション扱いで通知。
- 通知レベルはチャネル単位とワークスペース全体デフォルトを設定可、チャネル設定が上書き優先。
- 通知一時停止: 30分・1時間・2時間・翌朝9時まで全通知抑止可、ただしアプリ内バッジは継続。
- キーワード通知: 個人キーワードリストで通知可。
- 通知方法: ブラウザ通知、メール、モバイルプッシュ。
- 優先順位: ブラウザ未開封時はメール、5分以内に既読されない場合のみメール送信。

## 14. ピン留めとブックマーク
- チャネル管理者またはチャネルメンバーがピン留め可。
- ピン留めは1チャネルあたり最大100件。
- ピン留めはチャネル内サイドパネルから一覧アクセス。
- ユーザーは個人ブックマークとして任意メッセージを保存可。

## 15. 下書きと送信予約
- 入力中内容は自動下書き保存。
- 下書きはチャネル/DM/スレッド単位で保持。
- アプリ再起動後も復元。
- 送信予約は最大120日先まで。
- 予約済みメッセージは送信前の編集・取り消し可。

## 16. 既読管理
- メッセージは閲覧者単位で既読/未読を保持。
- 未読のあるチャネルは太字表示。
- 「未読をすべて既読にする」はチャネル単位またはワークスペース単位で実行可。
- 既読位置 (最終既読メッセージID) はサーバー保存、デバイス間同期。

## 17. ワークフロービルダー
- ノーコード作成機能。
- 構成: トリガー、ステップ、アクション。
- トリガー種別: 絵文字リアクション、特定キーワードのメッセージ投稿、フォーム送信、スケジュール (cron形式)、Webhook受信。
- ステップ種別: フォーム入力、条件分岐 (if/else)、メッセージ送信、外部API呼び出し、待機 (最大7日)。
- 公開単位: チャネル単位またはワークスペース単位。
- 作成・編集はオーナーまたは管理者のみ、メンバーはリクエスト送信のみ。
- 実行ログは過去30日間保存。
- 実行回数上限: 月間10000回 (Pro)、無制限 (Enterprise)。

## 18. AIアシスト機能
- 会話サマリー (チャネル/スレッド要約)。手動実行、または長時間オフラインユーザーに自動提示。
- スマートリプライ: 文脈応じた返信候補を最大3件提示。
- メンション通知へのクイックリアクション (絵文字提案、定型返信提案)。
- 検索クエリの自然言語入力対応 (例: 「先週マーケティングがリリースした件」)。
- AI生成内容には必ず「AI生成」ラベル付与。
- AI機能はワークスペース単位でオン/オフ、デフォルトはオフ。
- AIプロバイダへ送信されることをUI上で明示。
- 個人設定でオプトアウト可、オプトアウト時は本人メッセージはAI処理されない。
- AI翻訳: チャネル/スレッド単位で指定言語へ翻訳、原文・訳文併記。
- AIアクションアイテム抽出: スレッド・ハドル文字起こしから「やること」「決定事項」「期限」を構造化抽出。
- AIプロバイダ選択 (Enterprise): OpenAI / Anthropic / 自社オンプレLLMから選択可。
- AIガードレール: 機密情報 (社外秘ラベル) チャネルはAI処理を強制スキップ。
- 利用統計は管理コンソールで月次の使用量・コスト確認可。

## 19. インテグレーションと外部連携
- 連携種別: 「アプリ」と「Webhook」の2種類。
- アプリストアからインストール、オーナーまたは管理者のみ実行可、メンバーはリクエストのみ。
- 受信Webhook: 外部から投稿用URLを発行、レートリミット 60秒あたり最大60リクエスト。
- 送信Webhook: 特定キーワード投稿時に外部URLへPOST。
- スラッシュコマンド: `/コマンド名` でアプリ機能呼び出し。
- すべてOAuth 2.0で認可。
- アプリスコープ: 30種以上の細粒度スコープ (channels:read, chat:write, files:write 等) を組合せ。
- アプリ設定変更履歴は監査ログに記録。
- アプリ提出ガイドライン: アプリストア掲載前にセキュリティ審査、内製アプリは免除。
- カスタムインテグレーション: 単一ワークスペース内のものは管理者承認後に即時利用可。
- 連携先サービス例: GitHub、Jira、Asana、Notion、Google Drive、Zoom、Salesforce、PagerDuty、Datadog。

## 20. API
- REST API + リアルタイム用WebSocket API。
- 認証: BotトークンまたはユーザートークNG。
- レートリミット: 1ワークスペース毎秒60リクエスト (バースト120)。
- 超過時はHTTP 429 + `Retry-After` ヘッダ。
- API投稿メッセージはBotユーザーが投稿者として記録。
- イベントAPI: 30種類以上のイベントをWebSocketで配信 (新規メッセージ・編集・削除・リアクション追加など)。
- 公式SDK: Python / JavaScript / Go の3言語。

## 21. 監査と履歴
- ワークスペース管理者が監査ログで確認できるイベント:
  - ログイン/ログアウト (IP・User-Agent含む)
  - ロール変更
  - チャネル作成/アーカイブ/復元
  - ユーザー招待/退会/無効化
  - アプリのインストール/アンインストール
  - 設定変更 (SSO・SCIM・通知デフォルト等)
- 監査ログ保持期間: Pro 90日間、Enterprise 7年間 (法令対応)。
- 通常メンバーは自身の操作履歴のみ過去30日分閲覧可能。
- 取得方法: 管理者向けUI、CSVエクスポート、SIEMフォワーディング (Enterprise)。

## 22. データ保存とエクスポート
- データ保存リージョン: 米国・EU・アジア太平洋から選択可 (Enterprise)。
- メッセージ保存ポリシーをワークスペース全体・チャネル単位で設定可 (例: 30日/90日/1年/無期限)。
- 期限超過メッセージは自動削除。
- DMには独自ポリシー設定可。
- 標準エクスポート: パブリックチャネルのメッセージ・ファイルをZIP出力 (オーナー限定)。
- コンプライアンスエクスポート: プライベートチャネルとDMを含む全データ出力 (Enterprise限定、法的要請時のみ)。
- エクスポート完了時にオーナーに通知メール、ダウンロードリンクは7日間有効。

## 23. モバイルアプリ
- iOS 16以降 / Android 10以降のネイティブアプリ。
- 本書定義機能のうちAPI経由可能なものすべてをサポート。
- モバイル独自機能: プッシュ通知、生体認証 (Face ID / Touch ID / 指紋認証)、デバイスごとセッション管理。
- モバイルSSO認証はインアプリブラウザで実施。
- リリースから365日以上経過のバージョンは強制アップデート。
- オフライン時もキャッシュ閲覧可、復帰時に未送信メッセージ自動再送。
- 端末紛失時: 管理者または本人がリモートでセッション失効可、ローカルキャッシュも同時削除。
- プッシュ通知ペイロードは本文を含めず件名のみ (情報漏洩対策)。
- MDM: Microsoft Intune、Jamfをサポート。
- バックグラウンド更新は省電力モード (Android Doze、iOS Low Power Mode) 時に頻度を自動低下。

## 33. ヘルプとサポート
- アプリ内ヘルプセンター: ワークスペース言語で機能別ヘルプ記事表示。
- 右下のヘルプアイコンから検索可。
- お問い合わせフォーム: ヘルプセンターから直接サポートチケット起票。
- サポート時間: Pro 平日9-18時 (日本時間)、Enterprise 24/7。
- ステータスページ: 全ユーザー向け、過去90日のインシデント掲載。
- フィードバック機能: Vote付き機能要望投稿。
- リリースノート: アプリ内通知バー告知、詳細は公式ブログ。
- 開発者向けドキュメント: docs.linkr.app (APIリファレンス・SDK使用例・チュートリアル)。
- ベータプログラム: 早期アクセス、フィードバック義務あり。

## 34. 利用規約とプライバシー
- ワークスペース作成時に利用規約・プライバシーポリシー同意を要求。
- ポリシー変更時は30日前に管理者へメール通知し再同意要求。
- 利用範囲はサービス提供と統計分析に限定、第三者販売なし。
- 統計分析データは個人特定不能に匿名化。
- 個人エクスポート: JSONまたはCSVでいつでも可。
- アカウント削除時は30日以内に個人データ完全消去、バックアップは90日以内。
- Cookie: ファーストパーティCookieのみ、サードパーティトラッカー不使用。
- マーケティング通信は個別オプトイン/アウト。
- 13歳未満は利用不可、13-17歳は保護者同意ワークフロー必要。
- 法執行機関開示要求: 透明性レポートを年2回公開、件数と種別開示。
- CSAM (児童性的虐待コンテンツ): NCMEC等への自動通報、即時アカウント停止。
- バイオメトリクスデータ: サーバー送信せずデバイス内処理。
- データ常駐: 米国・EU・日本・シンガポールから選択可 (Enterprise)。
- リテンションロック: コンプラ要件で削除を防ぐべきデータは管理者でも削除不可に設定可。
- プライバシー強化: 「自分の名前で検索可能か」を個別制御可。
- 外部共有制限: 共有チャネル機能をワークスペース/チャネル単位で禁止可。
- 退職者データ: 「読み取り専用ユーザー」保持または即時無効化を選択可。
- データ持ち出し検知: 短時間に大量ダウンロード/検索発生時に管理者へ自動通知。
- 透明性レポート: 政府からの情報開示要求件数、自動コンテンツ削除件数等を半年ごとに公開。

## 24. アクセシビリティ
- WCAG 2.1 Level AA 準拠。
- スクリーンリーダー (VoiceOver / NVDA / JAWS) 対応。
- キーボードのみで全機能アクセス可。
- カラーコントラスト比 4.5:1 以上 (通常テキスト)。
- フォーカスインジケータは常時可視。

## 25. 国際化
- UI翻訳7言語: 日本語、英語、簡体字中国語、韓国語、ドイツ語、フランス語、スペイン語。
- ユーザーごとに表示言語選択可。
- ワークスペース管理者がデフォルト言語設定可。
- メッセージ本文は翻訳されない (ユーザー入力)。
- 日付・時刻はユーザータイムゾーン設定に従う。
- 文字コード: UTF-8。

## 26. セキュリティ
- 通信: TLS 1.2以上で暗号化。
- 保存時暗号化: AES-256。
- エンタープライズキー管理 (EKM): 顧客管理鍵 (CMK) によるキーラップ (Enterprise)。
- DLP: 送信時にカスタムパターン (クレジットカード番号等) を検出、ブロックまたは警告。
- セッション管理: 全デバイス同時ログイン可、一覧から個別/全セッション取り消し可。
- IPアドレス制限: 許可IPレンジ制限可 (Enterprise)。
- ペネトレーションテスト: 年2回外部ベンダー実施、結果サマリーをエンタープライズ顧客に開示。
- バグバウンティ: HackerOne経由、深刻度に応じ報奨金。
- 異常検知: 短時間の大量メッセージ削除、想定外の地理的アクセスを検知し管理者へアラート。
- パスワードポリシー: 過去5世代のパスワード再利用禁止、90日ごとの強制変更 (Enterpriseで設定可能)。
- DDoS対策: AWS Shield Advanced + Cloudflareレイヤー7保護を二重化。

## 27. 非機能要件
- 同時接続: 1ワークスペース最大10000 (Enterprise)。
- メッセージ送信レイテンシ: 95パーセンタイル500ms以内。
- 検索応答時間: 95パーセンタイル1秒以内 (1000万メッセージ規模)。
- ファイルアップロード: 100MBファイルが30秒以内 (回線100Mbps想定)。
- 可用性: SLA 99.9% (Pro) / 99.99% (Enterprise)。
- データ永続性: 保存ポリシー期間内は無損失、年間データ損失率 < 0.0001%。
- バックアップ: 全データ毎日、地理的分離3拠点、RPO 24時間 / RTO 4時間。
- 対応ブラウザ: 最新版 Chrome / Safari / Firefox / Edge。

## 29. 音声・ビデオハドル
- チャネル/DM内で音声・ビデオハドル開始可。
- 参加上限: Pro 25名 / Enterprise 50名。
- 開始時にチャネル内通知、ワンクリック参加。
- 画面共有: 全画面・特定ウィンドウ・特定ブラウザタブ選択可。
- 録画はオーナー権限ユーザーのみ開始可。
- 録画データは投稿チャネルにファイル自動添付、文字起こし付与。
- 録画はデフォルト90日保管、保存ポリシー従属。
- ホストにリアルタイム音声品質指標 (パケロス、ビットレート、遅延) 表示。
- ノイズ除去デフォルト有効、ユーザー設定で無効化可。

## 30. カスタムフォーム
- ワークフロー内またはメッセージから独立してフォーム作成可。
- フィールド種別: テキスト (短文/長文)、数値、選択 (単一/複数)、日付、日時、ファイルアップロード、ユーザー選択、チャネル選択。
- 必須項目・バリデーションルール (正規表現・最小最大値) 設定可。
- 回答結果は作成者または指定通知先チャネルへ投稿。
- CSVエクスポート可 (フォーム作成者のみ)。
- 公開リンク経由でワークスペース外からも回答可 (Enterprise)。
- 匿名回答オプション可、匿名時は回答者情報をエクスポートに含めない。

## 31. コンプライアンスとガバナンス
- リーガルホールド: 保存ポリシー超えてもデータ削除しない設定 (Enterprise)。
- リーガルホールドはコンプライアンス管理者ロールのみ設定可、本人にも通知されない。
- eDiscovery: コンプライアンスエクスポート + キーワード/期間/ユーザー絞り込み検索。
- データ主権: GDPR・CCPA・APPI等の各国法令対応、アクセス・削除リクエストフロー提供。
- 「忘れられる権利」: ユーザー要求から30日以内に匿名化または削除。
- SIEMフォワーディング: Splunk、Datadog、QRadarへリアルタイム配信 (Enterprise)。
- 準拠認証: ISO 27001、SOC 2 Type II、ISMAP、HIPAA (Enterpriseの一部プランのみ)。
- データ処理委託先一覧 (Sub-processors) を公開、変更時は事前通知。
- カスタマー鍵管理 (CKM): Enterprise契約者は自社KMSで暗号化鍵を管理可。
- カスタマーアクセスログ: 顧客自身が取得可 (Enterprise)。

## 32. オンボーディングと管理ツール
- ワークスペース新規作成時にセットアップウィザード起動 (初期チャネル・招待・通知設定をガイド)。
- 新規メンバー初回ログイン時にチュートリアル表示 (個人プロフィール・通知設定・チャネル参加)。
- 管理コンソールで全ユーザーのアクティビティ統計 (DAU/MAU、メッセージ数、チャネル別参加率) 確認可。
- 必須参加チャネルを管理者が強制設定可 (例: お試しチャネル、会社からのお知らせ)。
- セキュリティアラート: 異常ログイン (新デバイス・異常な地域) 検知時に本人と管理者へ通知。
- 健全性チェック: 「使われていないチャネル」「長期未ログインユーザー」を週次レポート。
