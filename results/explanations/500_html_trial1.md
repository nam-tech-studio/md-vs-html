# チャットツール「Linkr」要件定義書 (Enterprise) — 全内容

## 1. 概要
- 社内コミュニケーション用のチャットツール
- 本書はコア機能、コミュニケーション支援機能、拡張機能、およびエンタープライズ向けの管理・統合・コンプライアンス機能を定義する

## 2. ユーザーとアカウント
- ユーザーは1メールアドレスにつき1アカウント
- 各ユーザーは複数ワークスペースに所属可能
- 認証はパスワード認証またはSSO
- パスワードは最低12文字、英大文字・小文字・数字・記号をそれぞれ1文字以上含む
- パスワードリセットは登録メールアドレスへトークン送信、トークン有効期限60分
- 連続5回のログイン失敗で15分間ロック
- プレゼンス状態: オンライン / 退席中 / オフラインの3種類
- 10分間操作なしで自動的に「退席中」
- ステータスメッセージ: 絵文字 + 最大100文字、有効期限選択（1時間 / 4時間 / 終日 / カスタム / 無期限）
- 2要素認証: TOTP / SMS / WebAuthn
- 2要素認証はワークスペースオーナーが組織全体に強制可能

## 3. ワークスペース
- チームの単位、URLサブドメインで識別（例: `acme.linkr.app`）
- ユーザーは表示名とアバター画像を設定可能
- ワークスペース名・ロゴ・アイコンはオーナーが変更可能
- 契約プランが紐付き、機能制限が異なる
- 削除可能だが30日間は復元可能（オーナーのみ可）
- 30日経過後はメッセージ・ファイル・設定すべてが完全消去

**契約プラン比較表**
- **Free**: ユーザー50名まで、メッセージ履歴は直近10000件まで閲覧可能
- **Pro**: ユーザー無制限、メッセージ履歴無制限
- **Enterprise**: SSO、SCIM、データ常駐リージョン指定、エンタープライズキー管理、監査ログ拡張

## 4. ロールと権限
**3つのロール:**
- **オーナー**: ワークスペースの全権限。請求設定の変更、削除、他オーナーの任命が可能
- **管理者**: ユーザー招待、チャネル管理、ロール付与（オーナー以外）が可能
- **メンバー**: 通常利用権限のみ

ルール:
- ロールはオーナーまたは管理者によって変更可能（オーナーへの昇格はオーナーのみ可能）
- ワークスペースには常に1名以上のオーナーが必要
- 退会するオーナーは事前に他ユーザーへオーナー権限を委譲する必要

**ゲストユーザー:**
- 特定チャネルにのみアクセス可能な制限ロール
- DM・検索・他チャネル閲覧不可
- オーナーまたは管理者のみが招待可能

ロール階層図(SVG): オーナー > 管理者 > メンバー、別ロールとしてゲスト（特定チャネルのみ）

## 5. 招待とユーザー管理
- 管理者およびオーナーがメールアドレス指定で招待可能
- 招待リンク有効期限: 7日間
- 一度に最大100件のメールアドレスをまとめて招待可能
- ドメイン制限: 許可リストでメールドメイン制限可能
- ユーザーはオーナーまたは管理者によって無効化または完全削除が可能
  - **無効化**: ログイン不可だが過去のメッセージ・名前は残る
  - **完全削除**: 投稿メッセージは「削除済みユーザー」として残るがアカウント情報は完全消去
- SCIM v2.0 によるユーザープロビジョニング（Enterpriseプランのみ）

## 6. SSO（シングルサインオン）
- Enterpriseプランでワークスペース全体でSSOを有効化可能
- 対応プロトコル: SAML 2.0、OIDC
- SSO有効時はパスワード認証を無効化可能（オーナー設定）
- SSOプロバイダから受け取った属性（名前、メール、ユーザーグループ）は自動的に反映
- ジャストインタイムプロビジョニング（JIT）: 初回ログイン時に自動アカウント作成可能
- SCIMと併用時はユーザー作成・無効化はSCIM側を優先

## 7. チャネル
- ワークスペース内のトピック単位の会話空間
- チャネル名: 半角英数字とハイフンのみ、最大80文字
- 作成者は自動的にメンバーとなり初期管理権限を持つ
- パブリックチャネルへは自由参加可能
- 自分が参加中のチャネルから退出可能
  - 最後の管理者は退出前に他メンバーへ管理権限委譲が必要
- トピック（最大250文字）と説明（最大1500文字）を設定可能

**チャネル種別:**
- **パブリック**: ワークスペース全員が閲覧・参加可能
- **プライベート**: 招待されたメンバーのみ閲覧可能

**アーカイブ:**
- アーカイブ後は新規メッセージ投稿不可、閲覧と検索は可能
- チャネル管理者またはワークスペース管理者が実行可能
- ワークスペース管理者によって復元可能

**共有チャネル（Connect）:**
- 他のワークスペースとチャネルを共有可能（Enterpriseプラン）
- 共有先ワークスペース参加者は自身のアカウントで投稿
- どちらのワークスペースの保存期間ポリシーが適用されるか明示的に選択可能

## 8. メッセージ
- チャネル内にテキストメッセージ投稿可能
- 最大4000文字
- 投稿時刻と投稿者を記録
- 投稿後も時系列で永続表示
- Markdown記法（太字、斜体、コードブロック、リンク、引用、リスト）利用可能

### 8.1 編集と削除
- 自分のメッセージを編集可能
- 編集メッセージには「編集済み」表示
- 編集履歴は保存しないが編集日時は記録
- 自分のメッセージを削除可能
- チャネル管理者は他人のメッセージを削除可能（編集は不可）
- 削除メッセージは「このメッセージは削除されました」のプレースホルダで表示
- ソフトデリート、削除ログは90日間保管

### 8.2 スレッド
- 任意のメッセージへの返信でスレッド開始
- スレッド内メッセージはチャネルメインタイムラインには表示されない
- スレッド開始者・参加者は新規返信時に通知
- 「チャネルにも投稿」オプションで親チャネルへ二重投稿可能
- スレッド内でもメンション・編集・削除・リアクション可能

### 8.3 リアクション
- メッセージに絵文字リアクション付与可能
- 1メッセージあたり最大23種類の絵文字を表示
- 同一ユーザーが同一メッセージへ同じ絵文字を複数回リアクション不可
- カスタム絵文字: ワークスペース管理者がアップロード、全ユーザー使用可能
- カスタム絵文字: 1ファイル128KB以下、PNG / JPEG / GIF形式

## 9. メンション
- `@ユーザー名` でメンション
- メンションされたユーザーには通知（通知設定に従う）
- `@channel`: 同一チャネル全メンバーをメンション
- `@here`: 同一チャネルでオンラインの全メンバーをメンション
- `@channel`/`@here`の使用はチャネル管理者によって制限可能
- ユーザーグループ（例: `@team-frontend`）作成可能
  - オーナーまたは管理者が作成・管理
  - グループ内全員へのメンションとして扱う

## 10. ファイル共有
- メッセージにファイル添付可能
- 1ファイルあたり最大100MB（Enterpriseは最大5GB）
- 1メッセージあたり最大10ファイル添付可能
- 対応形式: 画像（PNG/JPEG/GIF/WebP）、PDF、動画（MP4/MOV）、各種ドキュメント、ZIPアーカイブ
- 画像とPDFはインラインプレビュー
- アクセス権を持つチャネル参加者のみ閲覧可能
- 投稿者またはチャネル管理者が削除可能
- 削除後即ダウンロード不可、メタデータは90日間保持
- ウイルススキャン: アップロード時に自動スキャン、検知時隔離

## 11. ダイレクトメッセージ（DM）
- 同一ワークスペース内の任意ユーザーにDM送信可能
- DMは1対1のみ
- マルチパーティDM（MPDM）: 最大8名まで、名前を持たない一時的会話
- DM内メッセージはチャネルメッセージと同仕様
- 送信者と受信者のみ閲覧、ワークスペース管理者でも他人のDMは閲覧不可
- 検索対象だが本人参加のDMのみが結果に含まれる

## 12. 検索
- メッセージ本文・チャネル名・ユーザー名・ファイル名を横断検索
- 検索結果はメッセージ・チャネル・ユーザー・ファイルのカテゴリ別表示
- 本人がアクセス権を持つチャネルとDMに限定
- 並び替え: 関連度順またはタイムスタンプ降順
- 1クエリあたり最大1000件、それ以上はページング

**絞り込みフィルタ:**
- `from:ユーザー名`: 投稿者
- `in:チャネル名`: チャネル
- `before:YYYY-MM-DD` / `after:YYYY-MM-DD`: 期間
- `has:link` / `has:file` / `has:reaction`: 添付要素

## 13. 通知
**チャネル通知レベル:**
- すべてのメッセージ
- メンション時のみ
- なし

- DMは常にメンション扱いで通知
- チャネル単位、およびワークスペース全体のデフォルト設定可能
- ワークスペースデフォルトはチャネル個別設定で上書き
- 通知一時停止: 30分・1時間・2時間・翌朝9時まで全通知抑止
- 抑止中もアプリ内バッジ表示は継続
- キーワード通知: ユーザーのキーワードリスト設定可能
- 通知方法: ブラウザ通知、メール、モバイルプッシュ
- 優先順位: ブラウザ未開封時はメール、5分以内に既読されない場合のみメール送信

## 14. ピン留めとブックマーク
- チャネル管理者またはチャネルメンバーがピン留め可能
- 1チャネルあたり最大100件
- ピン留めはサイドパネルから一覧アクセス可能
- 個人ブックマークとして任意のメッセージを保存可能

## 15. 下書きと送信予約
- 入力中の内容は自動的に下書き保存
- チャネル / DM / スレッド単位で保持
- アプリ再起動後も復元
- 送信予約: 最大120日先までの任意の日時を予約可能
- 予約済みメッセージは送信前に編集または取り消し可能

## 16. 既読管理
- 各メッセージは閲覧者単位で既読/未読状態を保持
- 未読のあるチャネルは太字表示
- チャネル単位またはワークスペース単位で一括既読化可能
- 既読位置（最終既読メッセージID）はサーバー保存、デバイス間同期

## 17. ワークフロービルダー
- ノーコードのワークフロー作成機能
- トリガー、ステップ、アクションの組み合わせで定義
- チャネル単位またはワークスペース単位で公開
- 作成・編集はオーナーまたは管理者のみ、メンバーはリクエスト送信のみ
- 実行ログは過去30日間保存
- 実行回数: Pro 月間10000回まで、Enterprise 無制限

**トリガー種別:** 絵文字リアクション、特定キーワードのメッセージ投稿、フォーム送信、スケジュール（cron形式）、Webhook受信
**ステップ種別:** フォーム入力、条件分岐（if/else）、メッセージ送信、外部API呼び出し、待機（最大7日）

構造図(SVG): トリガー → ステップ → アクション

## 18. AIアシスト機能
- 「会話サマリー」機能（チャネル/スレッドの要約）
- 手動実行または長時間オフラインだったユーザーへ自動生成提示
- スマートリプライ: 文脈に応じた返信候補を3件まで提示
- メンション通知へのクイックリアクション（絵文字提案、定型返信提案）
- 検索クエリの自然言語入力対応
- AI生成内容には必ず「AI生成」ラベル
- ワークスペース単位でオン/オフ切替、デフォルトはオフ
- AI機能利用時のメッセージ内容は外部AIプロバイダに送信されることをUI上で明示
- ユーザーは個人設定でオプトアウト可能（オプトアウト時は本人メッセージはAI処理されない）
- AI翻訳: チャネル/スレッド単位で指定言語へ翻訳、原文と訳文を併記表示
- AIアクションアイテム抽出: スレッド・ハドル文字起こしから「やること」「決定事項」「期限」を構造化抽出
- AIプロバイダ選択（Enterprise）: 自社契約のOpenAI / Anthropic / 自社オンプレLLMから選択可能
- AIガードレール: 機密情報（社外秘ラベル）チャネルのメッセージはAI処理を強制スキップ
- AI利用統計は管理コンソールで確認可能、月次の使用量とコスト表示

## 19. インテグレーションと外部連携
- 外部連携は「アプリ」と「Webhook」の2種類
- アプリストアからインストール型アプリ追加可能
- インストールはオーナーまたは管理者のみ（メンバーはリクエストのみ）
- 受信Webhook: 外部からチャネルへ投稿用URL発行可能
  - レートリミット: 60秒あたり最大60リクエスト
- 送信Webhook: 特定キーワード投稿時に外部URLへPOST送信
- スラッシュコマンド: `/コマンド名` で機能呼出
- 全連携はOAuth 2.0で認可
- アプリスコープ: 細粒度のスコープ（channels:read, chat:write, files:write など30種以上）の組み合わせ
- アプリ設定の変更履歴は監査ログに記録
- アプリ提出ガイドライン: アプリストア掲載前にセキュリティ審査（内製アプリは免除）
- カスタムインテグレーション: 単一ワークスペース内利用は管理者承認後に即時利用可能
- 連携先サービス例: GitHub、Jira、Asana、Notion、Google Drive、Zoom、Salesforce、PagerDuty、Datadog

## 20. API
- REST APIおよびリアルタイムイベント受信用WebSocket APIを提供
- 認証: Botトークンまたはユーザートークン
- レートリミット: 1ワークスペースあたり毎秒60リクエスト（バースト120）
- レートリミット超過時: HTTP 429 + `Retry-After`ヘッダ
- API経由投稿はBotユーザーが投稿者として記録
- イベントAPIは新規メッセージ・編集・削除・リアクション追加など30種類以上のイベントをWebSocket配信
- API SDK: Python / JavaScript / Go の3言語

## 21. 監査と履歴
監査ログで確認可能なイベント:
- ログイン / ログアウト（IPアドレス・User-Agent含む）
- ロール変更
- チャネル作成 / アーカイブ / 復元
- ユーザー招待 / 退会 / 無効化
- アプリのインストール / アンインストール
- 設定変更（SSO・SCIM・通知デフォルト等）

**保持期間:**
- Pro: 90日間
- Enterprise: 7年間（法令対応）

- 通常メンバーは自身の操作履歴のみ過去30日分閲覧可能
- 監査ログは管理者向けUI、CSVエクスポート、SIEMフォワーディング（Enterprise）で取得可能

## 22. データ保存とエクスポート
- データ保存リージョン: 米国・EU・アジア太平洋から選択可能（Enterprise）
- メッセージ保存期間ポリシー: ワークスペース全体およびチャネル単位で設定可能
  - 例: 30日 / 90日 / 1年 / 無期限
  - 期限超過は自動削除
- DMには独自の保存ポリシー設定可能

**データエクスポート:**
- **標準エクスポート**: パブリックチャネルのメッセージとファイルをZIP出力（オーナー限定）
- **コンプライアンスエクスポート**: プライベートチャネルおよびDMを含む全データ出力（Enterprise限定、法的要請時のみ）
- 完了時オーナーに通知メール、ダウンロードリンクは7日間有効

## 23. モバイルアプリ
- iOS（iOS 16以降）およびAndroid（Android 10以降）のネイティブアプリ
- API経由で提供可能な全機能をサポート
- モバイル独自機能: プッシュ通知、生体認証（Face ID / Touch ID / 指紋認証）、デバイスごとのセッション管理
- SSO認証はインアプリブラウザで実施
- リリースから365日以上経過したアプリは強制アップデート要求
- オフライン時もキャッシュされたメッセージ閲覧可能、復帰時に未送信メッセージ自動再送
- 紛失時: 管理者または本人がリモートからセッション失効可能、失効と同時にローカルキャッシュ削除
- プッシュ通知ペイロードはメッセージ本文を含めず、件名のみ送信（情報漏洩対策）
- MDM対応: Microsoft Intune および Jamf による配布・ポリシー強制
- 省電力モード時はバックグラウンド更新の頻度を自動的に下げる（Android Doze、iOS Low Power Mode）

## 24. アクセシビリティ
- WCAG 2.1 Level AA 準拠
- スクリーンリーダー（VoiceOver / NVDA / JAWS）対応
- キーボードのみで全機能アクセス可能
- カラーコントラスト比 4.5:1 以上（通常テキスト）
- フォーカスインジケータは常に可視

## 25. 国際化
- UI翻訳: 日本語、英語、簡体字中国語、韓国語、ドイツ語、フランス語、スペイン語の7言語
- ユーザーごとに表示言語を選択可能
- ワークスペース管理者がデフォルト言語設定可能
- メッセージ本文は翻訳されない
- 日付・時刻はユーザータイムゾーン設定に従う
- 文字コード: UTF-8

## 26. セキュリティ
- 通信: TLS 1.2以上で暗号化
- 保存時暗号化: AES-256でメッセージ・ファイル
- エンタープライズキー管理（EKM）: 顧客管理鍵（CMK）によるキーラップ（Enterprise）
- データ漏洩防止（DLP）: カスタムパターン（クレジットカード番号等）を検出してブロック/警告
- セッション管理: 全デバイスで同時ログイン可能、個別または全セッション取り消し可能
- IPアドレス制限: 許可IPレンジに制限可能（Enterprise）
- ペネトレーションテスト: 年2回外部ベンダーが実施、結果サマリーをエンタープライズ顧客に開示
- バグバウンティ: HackerOne経由、深刻度に応じて報奨金
- 異常検知: 短時間の大量メッセージ削除、想定外の地理的アクセスパターンを検知して管理者にアラート
- パスワードポリシー: 過去5世代のパスワード再利用禁止、90日ごとの強制変更（Enterpriseで設定可能）
- DDoS対策: AWS Shield Advanced および Cloudflareのレイヤー7保護を二重化

## 27. 非機能要件
| 項目 | 要件 |
|---|---|
| 同時接続ユーザー数 | 1ワークスペースあたり最大10000（Enterprise） |
| メッセージ送信レイテンシ | 送信から表示まで95パーセンタイルで500ms以内 |
| 検索応答時間 | 95パーセンタイルで1秒以内（1000万メッセージ規模） |
| ファイルアップロード | 100MBファイルが30秒以内（回線100Mbps想定） |
| 可用性 | SLA 99.9%（Pro）/ 99.99%（Enterprise） |

### 28.6 監視と運用
- APM: Datadog
- ログ集約: Datadog Logs
- アラート: PagerDuty連携、オンコールローテーションは1週間交代
- SLO: メッセージ送信成功率99.95%、APIレスポンスp95 500ms以内
- インシデント対応: PagerDuty呼出から5分以内の一次応答、30分以内のステータスページ更新
- カオスエンジニアリング: 月1回 Gremlin、本番に近いステージング環境で障害注入
- パフォーマンステスト: 主要エンドポイントの k6 負荷テストをCIで毎晩実行
- リリースバジェット（Error Budget）: SLO違反時はリリース自動停止、根本原因分析完了まで再開不可
- 設定変更履歴: Terraform State の差分をGit記録、本番環境変更は2名承認制
- DBマイグレーション: pg-online-schema-change によるロックフリーマイグレーション
- セキュリティパッチ: クリティカル24時間以内、ハイ7日以内、ミディアム30日以内
- バックアップリストア演習: 四半期に1回、別環境で本番バックアップから復旧リハーサル
- カスタマーアドバイザリ: 重大インシデント発生時は1時間以内ステータスページ更新、24時間以内に詳細レポート公開

## 29. 音声・ビデオハドル
- チャネルおよびDM内で音声・ビデオハドル開始可能
- 最大25名（Pro）/ 50名（Enterprise）
- 開始時にチャネル内通知、ワンクリック参加
- 画面共有: 全画面・特定ウィンドウ・特定ブラウザタブ選択可能
- 録画はオーナー権限のみ開始可能
- 録画データは投稿チャネルにファイルとして自動添付、文字起こし付与
- 録画はデフォルト90日保管（保存ポリシーに従う）
- 音声品質指標（パケロス・ビットレート・遅延）はリアルタイムでホストに表示
- バックグラウンドノイズ除去はデフォルト有効、ユーザー設定で無効化可能

## 30. カスタムフォーム
- ワークフロー内またはメッセージから独立したフォーム作成可能
- フィールド種別: テキスト（短文/長文）、数値、選択（単一/複数）、日付、日時、ファイルアップロード、ユーザー選択、チャネル選択
- 必須項目・バリデーションルール（正規表現・最小最大値）設定可能
- 回答結果は作成者または指定通知先チャネルに投稿
- 回答はCSVエクスポート可能（フォーム作成者のみ）
- 公開リンク経由でワークスペース外ユーザーからも回答可能（Enterprise）
- 匿名回答オプション、匿名時は回答者情報をエクスポートに含めない

## 31. コンプライアンスとガバナンス
- リーガルホールド: 特定ユーザー/チャネルのデータを保存ポリシーに関わらず削除しない設定（Enterprise）
- リーガルホールドはコンプライアンス管理者ロールのみ設定可能、設定中は本人にも通知されない
- eDiscovery: コンプライアンスエクスポート + 特定キーワード・期間・ユーザーで絞り込み検索
- データ主権: GDPR・CCPA・APPI等に対応したデータアクセス・削除リクエストフロー
- 「忘れられる権利」: ユーザー要求から30日以内に該当データの匿名化または削除完了
- SIEMフォワーディング: Splunk・Datadog・QRadarへリアルタイム配信（Enterprise）
- 準拠: ISO 27001、SOC 2 Type II、ISMAP、HIPAA（Enterpriseの一部プランのみ）
- データ処理委託先一覧（Sub-processors）は公開、変更時は事前通知
- カスタマー鍵管理（CKM）: Enterprise契約者は自社KMSで暗号化鍵を管理可能
- カスタマーアクセスログ機能を提供（Enterprise）

## 34. 利用規約とプライバシー
- ワークスペース作成時に利用規約・プライバシーポリシー同意要求
- ポリシー変更時は30日前に全ワークスペース管理者へメール通知し、再同意要求
- ユーザーデータ利用範囲はサービス提供と統計分析に限定、第三者販売なし
- 統計分析データは個人特定不能な形に匿名化後利用
- 個人エクスポート: ユーザーは自身のデータをJSONまたはCSVでいつでもエクスポート可能
- アカウント削除時: 個人データを30日以内に完全消去（バックアップからの削除は90日以内）
- Cookie: ファーストパーティCookieのみ、サードパーティトラッカー不使用
- マーケティング通信: ニュースレター・新機能アナウンスは個別オプトイン/アウト
- 未成年ユーザー: 13歳未満は利用不可、13-17歳は保護者同意ワークフロー通過時のみ
- 法執行機関開示要求: 透明性レポートを年2回公開、件数と種別を開示
- 児童性的虐待コンテンツ（CSAM）: NCMEC等への自動通報および即座のアカウント停止
- バイオメトリクスデータ: 顔認識・指紋等はサーバー送信せず、デバイス内処理
- データ常駐: 顧客は保管国を米国・EU・日本・シンガポールから選択可能（Enterprise）
- リテンションロック: 削除を防ぐべきデータは管理者でも削除不可に設定可能
- プライバシー強化: 「自分の名前で検索可能か」をユーザー個別制御可能
- 外部共有制限: 共有チャネル機能の利用をワークスペース/チャネル単位で禁止可能
- 退職者データ: 「読み取り専用ユーザー」として保持か、即時無効化を選択可能
- データ持ち出し検知: 短時間の大量メッセージダウンロード/検索発生時に管理者へ自動通知
- 透明性レポート: 政府からの情報開示要求件数、自動コンテンツ削除件数等を半年ごとに公開
