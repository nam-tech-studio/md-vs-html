## 1. 概要
- 社内コミュニケーション用チャットツール。
- 本書はコア機能、コミュニケーション支援機能、拡張機能、エンタープライズ向けの管理・統合・コンプライアンス機能を定義する。

## 2. ユーザーとアカウント
- 1メールアドレスにつき1アカウント。
- 各ユーザーは複数ワークスペースに所属可。
- 認証はパスワード認証またはSSO。
- パスワード: 最低12文字、英大文字・小文字・数字・記号それぞれ1文字以上必須。
- パスワードリセット: 登録メールへトークン送信、有効期限60分。
- 連続5回ログイン失敗で15分間ロック。
- プレゼンスは「オンライン / 退席中 / オフライン」の3種類。
- 10分間操作なしで自動的に「退席中」。
- ステータスメッセージ: 絵文字+最大100文字、有効期限は1時間/4時間/終日/カスタム/無期限から選択。
- 2要素認証: TOTP / SMS / WebAuthn。
- ワークスペースオーナーは2要素認証を組織全体に強制可能。

## 3. ワークスペース
- チームの単位、URLサブドメインで識別 (例: `acme.linkr.app`)。
- ユーザーは表示名とアバター画像を設定可能。
- ワークスペース名・ロゴ・アイコンはオーナーが変更可能。
- 契約プラン:
  - **Free**: ユーザー50名まで、メッセージ履歴は直近10000件まで閲覧可能。
  - **Pro**: ユーザー無制限、メッセージ履歴無制限。
  - **Enterprise**: SSO、SCIM、データ常駐リージョン指定、エンタープライズキー管理、監査ログ拡張。
- 削除可能だが30日間は復元可能 (オーナーのみ可)。
- 30日経過後にメッセージ・ファイル・設定すべて完全消去。

## 4. ロールと権限
- 3ロール:
  - **オーナー**: 全権限。請求設定変更、削除、他オーナーの任命。
  - **管理者**: ユーザー招待、チャネル管理、ロール付与 (オーナー以外)。
  - **メンバー**: 通常利用権限のみ。
- ロール変更はオーナーまたは管理者が可能 (オーナー昇格はオーナーのみ)。
- 常に1名以上のオーナーが必要。
- 退会するオーナーは事前に他者へオーナー権限委譲が必要。
- ゲストユーザー: 特定チャネルのみアクセス可能。
  - DM・検索・他チャネル閲覧不可。
  - 招待はオーナーまたは管理者のみ。

## 5. 招待とユーザー管理
- 管理者・オーナーがメールアドレス指定で招待可能。
- 招待リンクの有効期限は7日間。
- 一度に最大100件のメールアドレスをまとめて招待可能。
- ドメイン制限: 許可リストでメールドメインを制限可能。
- 無効化または完全削除がオーナー・管理者により可能。
  - 無効化: ログイン不可、過去メッセージ・名前は残る。
  - 完全削除: 投稿は「削除済みユーザー」として残るがアカウント情報は完全消去。
- SCIM v2.0 によるユーザープロビジョニング (Enterpriseのみ)。

## 6. SSO (シングルサインオン)
- Enterpriseプランでワークスペース全体にSSO有効化可能。
- 対応プロトコル: SAML 2.0、OIDC。
- SSO有効時はパスワード認証を無効化可能 (オーナー設定)。
- SSOプロバイダから受け取った属性 (名前、メール、ユーザーグループ) を自動反映。
- JIT (ジャストインタイムプロビジョニング): SSO初回ログイン時に自動アカウント作成可能。
- SCIM併用時はSCIM側のユーザー作成・無効化を優先。

## 7. チャネル
- ワークスペース内のトピック単位の会話空間。
- パブリックとプライベートの2種。
  - パブリック: 全員閲覧・参加可能。
  - プライベート: 招待者のみ。
- チャネル名は半角英数字とハイフンのみ、最大80文字。
- 作成者は自動メンバー化、初期管理権限を持つ。
- 任意のパブリックチャネルへ自由参加可。
- 自分が参加中のチャネルから退出可。
  - 最後の管理者は退出前に管理権限委譲が必要。
- トピック最大250文字、説明最大1500文字。
- アーカイブ可能。
  - アーカイブ後は新規投稿不可、閲覧と検索は可能。
  - 実行はチャネル管理者またはワークスペース管理者。
- アーカイブ済みチャネルはワークスペース管理者が復元可能。
- 共有チャネル (Connect): Enterpriseで他ワークスペースとチャネル共有可能。
  - 共有先参加者は自身のワークスペースのアカウントで投稿。
  - メッセージにどちらの保存期間ポリシーを適用するか明示選択可能。

## 8. メッセージ
- チャネル内テキスト投稿可能。
- 最大4000文字。
- 投稿時刻と投稿者を記録。
- 投稿後は時系列で永続表示。
- Markdown記法 (太字、斜体、コードブロック、リンク、引用、リスト) 利用可。

### 8.1 編集と削除
- 自分のメッセージ編集可。
- 編集メッセージには「編集済み」表示。
- 編集履歴は保存されないが、編集日時は記録。
- 自分のメッセージ削除可。
- チャネル管理者は他人のメッセージ削除可 (編集は不可)。
- 削除メッセージは「このメッセージは削除されました」プレースホルダ表示。
- ソフトデリート、削除ログは90日間保管。

### 8.2 スレッド
- 任意メッセージへの返信でスレッド開始。
- スレッド内メッセージはメインタイムラインに非表示。
- 開始者と参加者は新規返信時に通知。
- 「チャネルにも投稿」オプションで親チャネルへ二重投稿可能。
- スレッド内でもメンション・編集・削除・リアクション可能。

### 8.3 リアクション
- 絵文字リアクション付与可。
- 1メッセージあたり最大23種類の絵文字を表示。
- 同一ユーザーが同一メッセージに同じ絵文字で複数回リアクション不可。
- カスタム絵文字はワークスペース管理者がアップロード、全ユーザーが使用可。
- カスタム絵文字: 1ファイル128KB以下、PNG/JPEG/GIF。

## 9. メンション
- `@ユーザー名` でメンション。
- メンションされたユーザーに通知 (通知設定に従う)。
- `@channel` で同一チャネル全メンバー、`@here` でオンライン全メンバー。
- `@channel`/`@here` の使用はチャネル管理者が制限可能。
- ユーザーグループ (`@team-frontend` 等) 作成可能。
  - オーナーまたは管理者が作成・メンバー管理。
  - グループ内全員へのメンションとして扱う。

## 10. ファイル共有
- メッセージにファイル添付可。
- 1ファイル最大100MB (Enterpriseは最大5GB)。
- 1メッセージあたり最大10ファイル添付可。
- 対応形式: 画像 (PNG/JPEG/GIF/WebP)、PDF、動画 (MP4/MOV)、各種ドキュメント、ZIPアーカイブ。
- 画像とPDFはインラインプレビュー。
- アクセス権を持つチャネル参加者のみ閲覧可。
- 投稿者またはチャネル管理者が削除可。
- 削除後は即ダウンロード不可、メタデータは90日間保持。
- ウイルススキャン: アップロード時に自動、検知時隔離。

## 11. ダイレクトメッセージ (DM)
- 同一ワークスペース内任意ユーザーへDM送信可。
- 1対1のみ。
- マルチパーティDM (MPDM): 最大8名、名前なしの一時的な会話。
- DMはチャネルメッセージと同じ仕様。
- 送信者と受信者のみ閲覧、ワークスペース管理者でも他人のDM閲覧不可。
- 検索対象だが本人参加のDMのみ結果に含まれる。

## 12. 検索
- メッセージ本文・チャネル名・ユーザー名・ファイル名を横断検索。
- カテゴリ別 (メッセージ・チャネル・ユーザー・ファイル) 表示。
- アクセス権を持つチャネルとDMに限定。
- フィルタ: `from:ユーザー名`、`in:チャネル名`、`before:YYYY-MM-DD`/`after:YYYY-MM-DD`、`has:link`/`has:file`/`has:reaction`。
- 関連度順またはタイムスタンプ降順で並べ替え可能。
- 1クエリ最大1000件、それ以上はページング。

## 13. 通知
- 3種のチャネル通知レベル: すべてのメッセージ / メンション時のみ / なし。
- DMは常にメンション扱い。
- チャネル単位とワークスペースデフォルトで設定可能。
- ワークスペースデフォルトはチャネル個別設定で上書き。
- 通知一時停止: 30分・1時間・2時間・翌朝9時まで。
- 抑止中もアプリ内バッジは継続。
- キーワード通知: 自身のキーワードリストにマッチで通知。
- 通知方法: ブラウザ通知、メール、モバイルプッシュ。
- 優先順位: ブラウザ未開封時はメール、5分以内未既読のみメール送信。

## 14. ピン留めとブックマーク
- チャネル管理者またはチャネルメンバーがチャネル内ピン留め可能。
- チャネルあたり最大100件。
- サイドパネルから一覧アクセス可能。
- 個人ブックマークとして任意メッセージ保存可能。

## 15. 下書きと送信予約
- 入力中の内容を自動下書き保存。
- チャネル/DM/スレッド単位で保持。
- アプリ再起動後も復元。
- 送信予約: 最大120日先まで。
- 予約済みは送信前に編集または取り消し可能。

## 16. 既読管理
- 閲覧者単位で既読/未読保持。
- 未読のあるチャネルは太字表示。
- チャネル単位/ワークスペース単位で一括既読化可能。
- 既読位置 (最終既読メッセージID) はサーバーで保存、デバイス間同期。

## 17. ワークフロービルダー
- ノーコードのワークフロー作成。
- トリガー、ステップ、アクションの組み合わせで定義。
- トリガー種別: 絵文字リアクション、特定キーワードのメッセージ投稿、フォーム送信、スケジュール (cron)、Webhook受信。
- ステップ種別: フォーム入力、条件分岐 (if/else)、メッセージ送信、外部API呼び出し、待機 (最大7日)。
- チャネル単位またはワークスペース単位で公開可能。
- 作成・編集はオーナー/管理者のみ、メンバーはリクエスト送信のみ。
- 実行ログは過去30日間保存。
- 実行回数: Pro 月間10000回まで、Enterprise 無制限。

## 18. AIアシスト機能
- 「会話サマリー」機能 (チャネル/スレッド要約)。
- 手動実行または長時間オフラインだったユーザーへ自動生成。
- スマートリプライ: 返信候補を3件まで提示。
- メンション通知へのクイックリアクション (絵文字提案、定型返信提案)。
- 検索クエリの自然言語入力対応 (例: 「先週マーケティングがリリースした件」)。
- AI生成内容には必ず「AI生成」ラベル付与。
- AI機能はワークスペース単位でオン/オフ、デフォルトはオフ。
- AI機能利用時、メッセージが外部AIプロバイダに送信されることをUI上で明示。
- ユーザーは個人設定でオプトアウト可能 (本人のメッセージはAI処理されない)。
- AI翻訳: チャネル/スレッド単位で指定言語へ翻訳、原文と訳文併記。
- AIアクションアイテム抽出: 「やること」「決定事項」「期限」を構造化抽出 (スレッド・ハドル文字起こしから)。
- AIプロバイダ選択 (Enterprise): OpenAI / Anthropic / 自社オンプレLLM から選択可能。
- AIガードレール: 機密情報 (社外秘ラベル) チャネルはAI処理を強制スキップ。
- AI利用統計は管理コンソールで月次の使用量とコスト確認可能。

## 19. インテグレーションと外部連携
- 「アプリ」と「Webhook」の2種類。
- アプリストアからインストール型アプリ追加可能。
- インストールはオーナー/管理者のみ、メンバーはリクエスト送信のみ。
- 受信Webhook: 外部からチャネル投稿用URL発行、レートリミット 60秒あたり最大60リクエスト。
- 送信Webhook: 特定キーワード投稿時に外部URLへPOST。
- スラッシュコマンド (`/コマンド名`) でアプリ機能呼び出し。
- 全連携はOAuth 2.0で認可。
- アプリスコープ: 30種以上の細粒度スコープ (channels:read, chat:write, files:write 等)。
- アプリ設定変更履歴は監査ログに記録。
- アプリ提出ガイドライン: アプリストア掲載前にセキュリティ審査 (内製アプリは免除)。
- カスタムインテグレーション: 単一ワークスペース内利用は管理者承認後に即時利用可。
- 連携先サービス例: GitHub、Jira、Asana、Notion、Google Drive、Zoom、Salesforce、PagerDuty、Datadog。

## 20. API
- REST API + WebSocket APIを提供。
- 認証はBotトークンまたはユーザートークン。
- レートリミット: 1ワークスペースあたり毎秒60リクエスト (バースト120)。
- 超過時はHTTP 429 + `Retry-After` ヘッダ。
- API投稿メッセージはBotユーザー記録。
- イベントAPI: 30種以上のイベントをWebSocket配信 (新規メッセージ・編集・削除・リアクション追加など)。
- SDK: Python / JavaScript / Go の3言語。

## 21. 監査と履歴
- 監査ログ対象イベント:
  - ログイン/ログアウト (IPアドレス・User-Agent含む)
  - ロール変更
  - チャネル作成/アーカイブ/復元
  - ユーザー招待/退会/無効化
  - アプリのインストール/アンインストール
  - 設定変更 (SSO・SCIM・通知デフォルト等)
- 保持期間: Pro 90日、Enterprise 7年 (法令対応)。
- 通常メンバーは自身の操作履歴を過去30日分閲覧可能。
- 取得手段: 管理者向けUI、CSVエクスポート、SIEMフォワーディング (Enterprise)。

## 22. データ保存とエクスポート
- データ保存リージョン: 米国・EU・アジア太平洋から選択 (Enterprise)。
- メッセージ保存期間ポリシー: ワークスペース全体・チャネル単位 (例 30日/90日/1年/無期限)。
- 期限超過は自動削除。
- DMは独自の保存ポリシー設定可能。
- エクスポート:
  - **標準**: パブリックチャネルのメッセージとファイルをZIP出力 (オーナー限定)。
  - **コンプライアンス**: プライベートチャネルとDM含む全データ出力 (Enterprise限定、法的要請時のみ)。
- 完了時オーナーへ通知メール、ダウンロードリンクは7日間有効。

## 23. モバイルアプリ
- iOS 16以降、Android 10以降のネイティブアプリ。
- API経由で提供可能な全機能をサポート。
- モバイル独自: プッシュ通知、生体認証 (Face ID/Touch ID/指紋認証)、デバイスごとのセッション管理。
- SSO認証はインアプリブラウザ。
- リリースから365日経過の古いバージョンは強制アップデート。
- オフライン時もキャッシュメッセージ閲覧可、復帰時に未送信を自動再送。
- 紛失時: 管理者または本人がリモートからセッション失効可能、同時にローカルキャッシュも削除。
- プッシュ通知ペイロードはメッセージ本文を含めず件名のみ (情報漏洩対策)。
- MDM: Microsoft Intune、Jamf による配布・ポリシー強制をサポート。
- バックグラウンド更新は省電力モード (Android Doze、iOS Low Power Mode) で頻度を下げる。

## 33. ヘルプとサポート
- アプリ内ヘルプセンター (機能別ヘルプ記事をワークスペース言語で表示)。
- 右下のヘルプアイコンから検索可能。
- お問い合わせフォーム (ヘルプセンターからチケット起票)。
- サポート対応時間: Pro 平日9-18時 (日本時間)、Enterprise 24/7。
- ステータスページ: 全ユーザー向け、過去90日のインシデント掲載。
- フィードバック機能: Vote付きで他ユーザーが賛同表明。
- リリースノート: アプリ内通知バーで主要更新告知、詳細は公式ブログ。
- 開発者向けドキュメント: docs.linkr.app に公開 (APIリファレンス・SDK使用例・チュートリアル)。
- ベータプログラム: 希望ワークスペースは早期アクセス可、フィードバック義務あり。

## 34. 利用規約とプライバシー
- ワークスペース作成時に利用規約・プライバシーポリシー同意要求。
- ポリシー変更時は30日前に管理者へメール通知、再同意要求。
- データ利用範囲はサービス提供と統計分析に限定、第三者販売なし。
- 統計分析データは個人特定不能な匿名化後に利用。
- 個人エクスポート: JSON/CSV形式でいつでも可能。
- アカウント削除時は30日以内に個人データ完全消去 (バックアップからは90日以内)。
- Cookie: ファーストパーティのみ、サードパーティトラッカー不使用。
- マーケティング通信: 個別にオプトイン/アウト可能。
- 未成年: 13歳未満不可、13-17歳は保護者同意ワークフロー通過時のみ可。
- 法執行機関からの開示要求: 透明性レポートを年2回公開、件数と種別を開示。
- CSAM (児童性的虐待コンテンツ): NCMEC等への自動通報および即座のアカウント停止。
- バイオメトリクスデータ: サーバー送信せずデバイス内処理 (顔認識・指紋等)。
- データ常駐: 米国・EU・日本・シンガポールから保管国選択 (Enterprise)。
- リテンションロック: コンプライアンス要件により管理者でも削除不可に設定可能。
- プライバシー強化: 「自分の名前で検索可能か」を個別制御可能。
- 外部共有制限: 共有チャネル機能をワークスペース/チャネル単位で禁止可能。
- 退職者データ: 「読み取り専用ユーザー」として保持 or 即時無効化を選択。
- データ持ち出し検知: 短時間の大量ダウンロード/検索を管理者へ自動通知。
- 透明性レポート: 政府からの情報開示要求件数、自動コンテンツ削除件数等を半年ごと公開。

## 24. アクセシビリティ
- WCAG 2.1 Level AA 準拠。
- スクリーンリーダー (VoiceOver / NVDA / JAWS) 対応。
- キーボードのみで全機能アクセス可能。
- カラーコントラスト比 4.5:1 以上 (通常テキスト)。
- フォーカスインジケータ常時可視。

## 25. 国際化
- UI翻訳: 日本語、英語、簡体字中国語、韓国語、ドイツ語、フランス語、スペイン語の7言語。
- ユーザーごとに表示言語選択可能。
- 管理者がワークスペースのデフォルト言語を設定可能。
- メッセージ本文は翻訳されない (ユーザー入力)。
- 日付・時刻はユーザータイムゾーン設定に従う。
- 文字コード: UTF-8。

## 26. セキュリティ
- 通信は全てTLS 1.2以上で暗号化。
- 保存時暗号化: AES-256でメッセージ・ファイル。
- EKM (エンタープライズキー管理): CMK (顧客管理鍵) によるキーラップ (Enterprise)。
- DLP: メッセージ送信時にカスタムパターン (クレジットカード番号等) を検出、ブロックまたは警告。
- セッション管理: 全デバイスで同時ログイン可、個別/全セッション取り消し可能。
- IPアドレス制限: 許可IPレンジに制限可能 (Enterprise)。
- ペネトレーションテスト: 年2回外部ベンダー実施、結果サマリーをエンタープライズ顧客に開示。
- バグバウンティ: HackerOne経由、深刻度に応じて報奨金。
- 異常検知: 短時間の大量メッセージ削除、想定外の地理的アクセスパターンを検知し管理者アラート。
- パスワードポリシー: 過去5世代の再利用禁止、90日ごと強制変更 (Enterpriseで設定可能)。
- DDoS対策: AWS Shield Advanced + Cloudflareレイヤー7保護を二重化。

## 27. 非機能要件
- 同時接続: 1ワークスペースあたり最大10000 (Enterprise)。
- メッセージ送信レイテンシ: 95パーセンタイルで500ms以内。
- 検索応答時間: 95パーセンタイルで1秒以内 (1000万メッセージ規模)。
- ファイルアップロード: 100MBを30秒以内 (回線100Mbps想定)。
- 可用性: SLA 99.9% (Pro) / 99.99% (Enterprise)。
- データ永続性: 保存ポリシー期間内は無損失、年間データ損失率 < 0.0001%。
- バックアップ: 毎日全データ、地理的に分離した3拠点 (RPO 24時間 / RTO 4時間)。
- 対応ブラウザ: 最新版のChrome / Safari / Firefox / Edge。

### 28.5 CI/CD と開発フロー

### 28.6 監視と運用
- APM: Datadog。
- ログ集約: Datadog Logs。
- アラート: PagerDuty、オンコールローテーションは1週間交代。
- SLO: メッセージ送信成功率99.95%、APIレスポンス p95 500ms以内。
- インシデント: PagerDuty呼び出しから5分以内一次応答、30分以内ステータスページ更新。
- カオスエンジニアリング: 月1回 Gremlin で本番に近いステージング環境で障害注入。
- パフォーマンステスト: 主要エンドポイントの k6 負荷テストをCIで毎晩実行。
- リリースバジェット (Error Budget): SLO違反時はリリース自動停止、根本原因分析完了まで再開不可。
- 設定変更履歴: Terraform State 差分をGitに記録、本番変更は2名承認制。
- DBマイグレーション: pg-online-schema-change によるロックフリーマイグレーション。
- セキュリティパッチ: クリティカル24時間以内、ハイ7日以内、ミディアム30日以内。
- バックアップリストア演習: 四半期に1回、別環境で本番バックアップから復旧リハーサル。
- カスタマーアドバイザリ: 重大インシデントは1時間以内にステータスページ更新、24時間以内に詳細レポート公開。

## 30. カスタムフォーム
- ワークフロー内またはメッセージから独立したフォーム作成可能。
- フィールド種別: テキスト (短文/長文)、数値、選択 (単一/複数)、日付、日時、ファイルアップロード、ユーザー選択、チャネル選択。
- 必須項目・バリデーション (正規表現・最小最大値) 設定可能。
- 回答結果は作成者または指定通知先チャネルに投稿。
- 回答はCSVエクスポート可能 (作成者のみ)。
- 公開リンク経由でワークスペース外からも回答可能 (Enterprise)。
- 匿名回答オプション設定可能、匿名時はエクスポートに回答者情報を含めない。

## 31. コンプライアンスとガバナンス
- リーガルホールド (Legal Hold): 保存ポリシーに関わらずデータ削除しない (Enterprise)。
- 設定はコンプライアンス管理者ロールのみ、設定中は本人にも通知されない。
- eDiscovery: コンプライアンスエクスポートに加え、キーワード・期間・ユーザーで絞り込み検索可能。
- データ主権: GDPR・CCPA・APPI等のアクセス・削除リクエストフローを提供。
- 「忘れられる権利」: ユーザー要求から30日以内に匿名化または削除。
- SIEMフォワーディング: Splunk・Datadog・QRadarへリアルタイム配信 (Enterprise)。
- ISO 27001、SOC 2 Type II、ISMAP、HIPAA (Enterpriseの一部プランのみ) 準拠。
- データ処理委託先一覧 (Sub-processors) を公開、変更時は事前通知。
- カスタマー鍵管理 (CKM): Enterprise契約者は自社KMSで暗号化鍵管理可能。
- カスタマーアクセスログ機能: 顧客自身がアクセスログ取得可能 (Enterprise)。

## 32. オンボーディングと管理ツール
- 新規ワークスペース作成時にセットアップウィザード (初期チャネル・招待・通知設定をガイド)。
- 新規メンバー初回ログイン時にプロフィール設定・通知設定・チャネル参加チュートリアル表示。
- 管理コンソールで全ユーザーのアクティビティ統計 (DAU/MAU・メッセージ数・チャネル別参加率) 確認可能。
- 管理者が必須参加チャネル (例「お試しチャネル」「会社からのお知らせ」) を強制設定可能。
- セキュリティアラート: 異常ログイン (新デバイス・異常な地域) を検知、本人と管理者へ通知。
- 健全性チェック: 「使われていないチャネル」「長期間ログインしていないユーザー」を週次レポート。
