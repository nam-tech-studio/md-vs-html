# チャットツール「Linkr」要件定義書 (Enterprise) — 全要件徹底解説

## 1. 概要
- 社内コミュニケーション用のチャットツール
- 本書はコア機能、コミュニケーション支援機能、拡張機能、およびエンタープライズ向けの管理・統合・コンプライアンス機能を定義する

## 2. ユーザーとアカウント
- 1メールアドレスにつき1アカウント
- 各ユーザーは複数のワークスペースに所属可能
- 認証はパスワード認証またはSSO
- パスワード要件: 最低12文字、英大文字・小文字・数字・記号をそれぞれ1文字以上
- パスワードリセットは登録メールアドレスへのトークン送信、トークン有効期限60分
- 連続5回のログイン失敗で15分間ロック
- プレゼンス3種類: オンライン / 退席中 / オフライン
- 10分間操作なしで自動的に「退席中」
- ステータスメッセージ: 絵文字+最大100文字、有効期限選択可 (1時間/4時間/終日/カスタム/無期限)
- 2要素認証: TOTP / SMS / WebAuthn
- 2要素認証はワークスペースオーナーが組織全体に強制可能

## 3. ワークスペース
- チームの単位、URLサブドメインで識別 (例: `acme.linkr.app`)
- 表示名・アバター画像をユーザーが設定可能
- ワークスペース名・ロゴ・アイコンはオーナーが変更可能
- 契約プランが紐付き、プランで機能制限が異なる
- 削除実行から30日間は復元可能 (オーナーのみ)
- 30日経過後はメッセージ・ファイル・設定すべて完全消去

### 契約プラン比較
| プラン | 主な制限・機能 |
|---|---|
| Free | ユーザー50名まで、メッセージ履歴は直近10000件まで閲覧可能 |
| Pro | ユーザー無制限、メッセージ履歴無制限 |
| Enterprise | SSO、SCIM、データ常駐リージョン指定、エンタープライズキー管理、監査ログ拡張 |

## 4. ロールと権限
- **オーナー**: 全権限。請求設定変更、削除、他オーナー任命可
- **管理者**: ユーザー招待、チャネル管理、ロール付与 (オーナー以外) 可
- **メンバー**: 通常利用権限のみ
- ロール変更はオーナーまたは管理者 (オーナー昇格はオーナーのみ)
- 常に1名以上のオーナーが必要
- 退会するオーナーは事前にオーナー権限を委譲する必要
- ロール階層図: オーナー > 管理者 > メンバー、別ロールとしてゲスト

### ゲストユーザー
- 特定チャネルにのみアクセス可能な制限ロール
- DM・検索・他チャネル閲覧不可
- オーナーまたは管理者のみが招待可能

## 5. 招待とユーザー管理
- 管理者・オーナーがメールアドレス指定で招待
- 招待リンクの有効期限: 7日間
- 一度に最大100件のメールアドレスを一括招待可能
- ドメイン制限: 許可リストで参加可能ドメインを制限可能
- ユーザーは無効化または完全削除可能
  - **無効化**: ログイン不可、過去メッセージ・名前は残る
  - **完全削除**: 投稿は「削除済みユーザー」として残るがアカウント情報は完全消去
- SCIM v2.0によるユーザープロビジョニング (Enterpriseのみ)

## 6. SSO
- Enterpriseでワークスペース全体でSSO有効化可能
- 対応プロトコル: SAML 2.0、OIDC
- SSO有効時はパスワード認証を無効化可能 (オーナー設定)
- SSOプロバイダ属性 (名前、メール、ユーザーグループ) を自動反映
- JIT (ジャストインタイムプロビジョニング): SSO初回ログイン時に自動アカウント作成可能
- SCIM併用時はユーザー作成・無効化はSCIM側を優先

## 7. チャネル
- ワークスペース内のトピック単位の会話空間
- チャネル名: 半角英数字とハイフンのみ、最大80文字
- 作成者は自動的にメンバー、初期管理権限を持つ
- パブリックチャネルへは自由参加可能
- 退出可能だが最後の管理者は他メンバーに権限委譲が必要
- トピック最大250文字、説明最大1500文字

### 種別
- **パブリック**: ワークスペース全員が閲覧・参加可能
- **プライベート**: 招待されたメンバーのみ閲覧可能

### アーカイブ
- アーカイブ後は新規投稿不可、閲覧・検索は可能
- チャネル管理者またはワークスペース管理者が実行
- ワークスペース管理者が復元可能

### 共有チャネル (Connect)
- 他ワークスペースとチャネル共有可能 (Enterprise)
- 共有先参加者は自身のワークスペースのアカウントで投稿
- どちらの保存期間ポリシーを適用するか明示的に選択可能

## 8. メッセージ
- テキストメッセージを投稿可能
- 最大4000文字
- 投稿時刻と投稿者を記録
- 時系列で永続表示
- Markdown対応 (太字、斜体、コードブロック、リンク、引用、リスト)

### 8.1 編集と削除
- 自分のメッセージを編集可能
- 編集には「編集済み」表示
- 編集履歴は保存しないが編集日時は記録
- 自分のメッセージを削除可能
- チャネル管理者は他人のメッセージを削除可能 (編集は不可)
- 削除時は「このメッセージは削除されました」プレースホルダ表示
- ソフトデリート、削除ログは90日間保管

### 8.2 スレッド
- 任意メッセージへの返信でスレッド開始
- スレッド内メッセージはメインタイムラインに非表示
- スレッド開始者・参加者は新規返信時に通知
- 「チャネルにも投稿」オプションで親チャネルへ二重投稿可能
- スレッド内でもメンション・編集・削除・リアクション可

### 8.3 リアクション
- 絵文字リアクション付与可能
- 1メッセージあたり最大23種類の絵文字を表示
- 同一ユーザー・同一メッセージ・同一絵文字を複数回付与不可
- カスタム絵文字: ワークスペース管理者がアップロード、全ユーザー使用可
- カスタム絵文字: 1ファイル128KB以下、PNG/JPEG/GIF

## 9. メンション
- `@ユーザー名` でメンション
- メンションされたユーザーへ通知 (通知設定に従う)
- `@channel` は同一チャネルの全メンバー
- `@here` は同一チャネルでオンラインの全メンバー
- `@channel`/`@here` の使用はチャネル管理者が制限可能
- ユーザーグループ (例: `@team-frontend`) を作成可能
  - オーナーまたは管理者が作成・所属管理
  - メンションはグループ全員へのメンションとして扱う

## 10. ファイル共有
- メッセージにファイル添付可能
- 1ファイル最大100MB (Enterpriseは最大5GB)
- 1メッセージあたり最大10ファイル
- 対応形式: 画像 (PNG/JPEG/GIF/WebP)、PDF、動画 (MP4/MOV)、各種ドキュメント、ZIP
- 画像とPDFはインラインプレビュー
- アクセス権を持つチャネル参加者のみ閲覧可能
- 投稿者またはチャネル管理者が削除可能
- 削除直後にダウンロード不可、メタデータは90日間保持
- アップロード時のウイルススキャン、検知時は隔離

## 11. ダイレクトメッセージ (DM)
- 同一ワークスペース内の任意のユーザーへ送信
- DMは1対1のみ
- マルチパーティDM (MPDM): 最大8名のグループDM、名前を持たない一時的会話
- DM内メッセージ仕様はチャネルメッセージと同じ
- 送信者と受信者のみ閲覧、ワークスペース管理者でも他人のDMは閲覧不可
- 検索対象だが本人参加のDMのみ検索結果に含む

## 12. 検索
- 横断検索: メッセージ本文・チャネル名・ユーザー名・ファイル名
- カテゴリ別 (メッセージ・チャネル・ユーザー・ファイル) に表示
- 検索対象は本人がアクセス権を持つチャネル・DMに限定
- 関連度順またはタイムスタンプ降順で並び替え可
- 1クエリ最大1000件、それ以上はページング

### フィルタ
| フィルタ | 説明 |
|---|---|
| `from:ユーザー名` | 投稿者で絞り込み |
| `in:チャネル名` | チャネルで絞り込み |
| `before:YYYY-MM-DD` / `after:YYYY-MM-DD` | 期間で絞り込み |
| `has:link` / `has:file` / `has:reaction` | 添付要素で絞り込み |

## 13. 通知
- チャネル通知レベル3種類: すべてのメッセージ / メンション時のみ / なし
- DMは常にメンション扱いで通知
- チャネル単位とワークスペース全体デフォルトを設定可、チャネル個別が上書き
- 通知一時停止: 30分・1時間・2時間・翌朝9時まで
- 通知抑止中もアプリ内バッジ表示は継続
- キーワード通知: ユーザー個別のキーワードリスト
- 通知方法: ブラウザ通知、メール、モバイルプッシュ
- 優先順位: ブラウザ未開封時はメール、5分以内に既読されない場合のみメール送信

## 14. ピン留めとブックマーク
- チャネル管理者またはチャネルメンバーがピン留め可能
- ピン留めはチャネルあたり最大100件
- サイドパネルから一覧アクセス可能
- 個人ブックマーク: 任意のメッセージを保存可能

## 15. 下書きと送信予約
- 入力中の内容は自動的に下書き保存
- 下書きはチャネル/DM/スレッド単位
- アプリ再起動後も復元
- 送信予約: 最大120日先まで
- 予約済みメッセージは送信前に編集・取り消し可能

## 16. 既読管理
- 各メッセージは閲覧者単位で既読/未読を保持
- 未読のあるチャネルは太字表示
- 「未読をすべて既読にする」アクション (チャネル単位/ワークスペース単位)
- 既読位置 (最終既読メッセージID) はサーバー保存、デバイス間同期

## 17. ワークフロービルダー
- ノーコードのワークフロー作成機能
- トリガー、ステップ、アクションの組み合わせで定義
- チャネル単位またはワークスペース単位で公開可能
- 作成・編集はオーナーまたは管理者のみ、メンバーはリクエスト送信のみ
- 実行ログは過去30日間保存
- 実行回数: 月間10000回 (Pro) / 無制限 (Enterprise)

### トリガー / ステップ種別
| 区分 | 種別 |
|---|---|
| トリガー | 絵文字リアクション、特定キーワードのメッセージ投稿、フォーム送信、スケジュール (cron形式)、Webhook受信 |
| ステップ | フォーム入力、条件分岐 (if/else)、メッセージ送信、外部API呼び出し、待機 (最大7日) |

- 構造: トリガー → ステップ → アクション

## 18. AIアシスト機能
- 会話サマリー (チャネル・スレッドのメッセージ要約)
- 会話サマリーは手動実行または長時間オフラインだったユーザーに自動生成提示
- スマートリプライ: 文脈に応じた返信候補を3件まで提示
- メンション通知に対するクイックリアクション (絵文字提案、定型返信提案)
- 自然言語検索 (例: 「先週マーケティングがリリースした件」)
- AI生成内容には必ず「AI生成」ラベル
- ワークスペース単位でオン/オフ切替、デフォルトはオフ
- AI利用時メッセージ内容が外部AIプロバイダに送信されることをUI上で明示
- ユーザーは個人設定でオプトアウト可 (本人のメッセージはAI処理されない)
- AI翻訳: チャネル/スレッド単位、原文と訳文を併記
- AIアクションアイテム抽出: スレッド・ハドル文字起こしから「やること」「決定事項」「期限」を構造化抽出
- AIプロバイダ選択 (Enterprise): 自社契約のOpenAI / Anthropic / 自社オンプレLLM
- AIガードレール: 「社外秘ラベル」付きチャネルはAI処理を強制スキップ
- 管理コンソールで月次の使用量とコスト確認可能

## 19. インテグレーションと外部連携
- 「アプリ」と「Webhook」の2種類でサポート
- アプリストアからインストール可能
- インストールはオーナーまたは管理者のみ (メンバーはリクエスト送信のみ)
- 受信Webhook: 外部からチャネルへ投稿、レートリミット60秒あたり最大60リクエスト
- 送信Webhook: 特定キーワード投稿時に外部URLへPOST
- スラッシュコマンド: `/コマンド名` でアプリ機能呼び出し
- すべての連携はOAuth 2.0で認可
- アプリスコープ: 細粒度スコープ (channels:read, chat:write, files:write など30種以上)
- アプリ設定変更履歴は監査ログに記録
- アプリ提出ガイドライン: アプリストア掲載前にセキュリティ審査 (内製アプリは免除)
- カスタムインテグレーション: 単一ワークスペース内、管理者承認後即時利用可
- 連携先例: GitHub、Jira、Asana、Notion、Google Drive、Zoom、Salesforce、PagerDuty、Datadog

## 20. API
- REST APIおよびWebSocket API (リアルタイムイベント受信)
- 認証はBotトークンまたはユーザートークン
- レートリミット: 1ワークスペース毎秒60リクエスト (バースト120)
- 超過時はHTTP 429、`Retry-After` ヘッダで通知
- API経由の投稿はBotユーザーが投稿者として記録
- イベントAPI: 新規メッセージ・編集・削除・リアクション追加など30種類以上をWebSocket配信
- SDK: Python / JavaScript / Go の3言語

## 21. 監査と履歴
- 監査ログで管理者が確認可能なイベント:
  - ログイン/ログアウト (IPアドレス・User-Agent含む)
  - ロール変更
  - チャネル作成/アーカイブ/復元
  - ユーザー招待/退会/無効化
  - アプリのインストール/アンインストール
  - 設定変更 (SSO・SCIM・通知デフォルト等)

### 保持期間
| プラン | 保持期間 |
|---|---|
| Pro | 90日間 |
| Enterprise | 7年間 (法令対応) |

- 通常メンバーは自身の操作履歴のみ過去30日分閲覧可
- 管理者向けUI、CSVエクスポート、SIEMフォワーディング (Enterprise) で取得

## 22. データ保存とエクスポート
- データ保存リージョン: 米国・EU・アジア太平洋から選択 (Enterprise)
- メッセージ保存期間ポリシー (例: 30日/90日/1年/無期限)、ワークスペース全体・チャネル単位
- 期限超過のメッセージは自動削除
- DMには独自の保存ポリシー設定可能

### データエクスポート
- **標準エクスポート**: パブリックチャネルのメッセージとファイルをZIP出力 (オーナー限定)
- **コンプライアンスエクスポート**: プライベートチャネル・DMを含む全データ (Enterprise限定、法的要請時のみ)
- 完了時にオーナーへ通知メール、ダウンロードリンクは7日間有効

## 23. モバイルアプリ
- iOS (iOS 16以降) / Android (Android 10以降) のネイティブアプリ
- API経由で提供可能な全機能をサポート
- モバイル独自機能: プッシュ通知、生体認証 (Face ID / Touch ID / 指紋認証)、デバイスごとのセッション管理
- モバイルからのSSOはインアプリブラウザ
- リリースから365日以上経過したアプリは強制アップデート
- オフライン時もキャッシュ閲覧可能、復帰時に未送信メッセージを自動再送
- 紛失時: 管理者または本人がリモートからセッション失効可能、同時にローカルキャッシュ削除
- プッシュ通知ペイロードはメッセージ本文を含めず件名のみ (情報漏洩対策)
- MDM: Microsoft Intune および Jamf
- バックグラウンド更新は省電力モード時に頻度低減 (Android Doze、iOS Low Power Mode)

## 24. アクセシビリティ
- WCAG 2.1 Level AA 準拠
- スクリーンリーダー対応 (VoiceOver / NVDA / JAWS)
- キーボードのみで全機能アクセス可能
- カラーコントラスト比 4.5:1 以上 (通常テキスト)
- フォーカスインジケータは常に可視

## 25. 国際化
- UI翻訳7言語: 日本語、英語、簡体字中国語、韓国語、ドイツ語、フランス語、スペイン語
- ユーザーごとに表示言語選択可能
- ワークスペース管理者がデフォルト言語設定可能
- メッセージ本文は翻訳しない (ユーザー入力)
- 日付・時刻はユーザータイムゾーン設定に従う
- 文字コード: UTF-8

## 26. セキュリティ
- 通信はTLS 1.2以上
- 保存時暗号化: AES-256
- EKM (エンタープライズキー管理): 顧客管理鍵 (CMK) によるキーラップ (Enterprise)
- DLP (データ漏洩防止): カスタムパターン (クレジットカード番号等) 検出、ブロック/警告
- セッション管理: 全デバイス同時ログイン可、個別または全セッション取り消し可能
- IPアドレス制限 (Enterprise)
- ペネトレーションテスト: 年2回外部ベンダー実施、結果サマリーをエンタープライズ顧客に開示
- バグバウンティ: HackerOne、深刻度に応じて報奨金
- 異常検知: 短時間の大量メッセージ削除、想定外の地理的アクセスパターン
- パスワードポリシー: 過去5世代の再利用禁止、90日ごとの強制変更 (Enterpriseで設定可)
- DDoS対策: AWS Shield Advanced + Cloudflare レイヤー7保護の二重化

## 27. 非機能要件
| 項目 | 要件 |
|---|---|
| 同時接続ユーザー数 | 1ワークスペースあたり最大10000 (Enterprise) |
| メッセージ送信レイテンシ | 送信から表示まで95パーセンタイルで500ms以内 |
| 検索応答時間 | 95パーセンタイルで1秒以内 (1000万メッセージ規模) |
| ファイルアップロード | 100MBファイルが30秒以内 (回線100Mbps想定) |
| 可用性 | SLA 99.9% (Pro) / 99.99% (Enterprise) |
| データ永続性 | 保存ポリシー期間内は無損失 (年間データ損失率 < 0.0001%) |
| バックアップ | 全データを毎日バックアップ、地理的に分離した3拠点に保管 (RPO 24時間 / RTO 4時間) |
| 対応ブラウザ | 最新版のChrome / Safari / Firefox / Edge |

### 28.2 バックエンド
- 言語: Go 1.22以上
- HTTPフレームワーク: chi
- ORM/SQLジェネレータ: sqlc
- 認証: JWT (RS256、鍵ローテーションは月次)
- リアルタイム: WebSocket (RFC 6455)、サーバー間ファンアウトはRedis Pub/Sub
- 非同期処理: Amazon SQS + Lambda (通知配信、監査ログ、エクスポート)

### 28.3 データストア
- PostgreSQL 16: 主データ。Aurora PostgreSQL Serverless v2、リードレプリカ最低3台
- Redis 7: セッション、Pub/Sub、レートリミット、ホットキャッシュ。ElastiCache for Redis (クラスタモード有効)
- Elasticsearch 8: 全文検索。OpenSearch Service
- S3: ファイル添付。署名付きURL、ライフサイクルポリシー
- Glacier: 長期保存データ (Enterprise保存期間ポリシー)

### 28.4 インフラ
- クラウド: AWS、リージョンは契約に応じて選択 (米国・EU・APAC)
- IaC: Terraform (terragruntでマルチ環境管理)
- コンテナオーケストレーション: ECS Fargate
- CDN: CloudFront
- WAF: AWS WAF + Cloudflareの二段構成
- DNS: Route 53
- 秘匿情報管理: AWS Secrets Manager + 顧客管理鍵 (CMK)

### 28.5 CI/CD と開発フロー
- バージョン管理: GitHub
- CI/CD: GitHub Actions (PR毎にlint・test・E2E、mainマージ時にBlue/Greenデプロイ)
- ブランチ戦略: トランクベース開発 + Feature Flag
- フィーチャーフラグ: LaunchDarkly
- 依存脆弱性スキャン: Snyk + Dependabot
- コードレビュー: 全PRに最低2名、CODEOWNERSで自動アサイン
- リリース頻度: 週2回 (火・木)、ホットフィックスは随時

### 28.6 監視と運用
- APM: Datadog
- ログ集約: Datadog Logs
- アラート: PagerDuty連携、オンコールローテーション1週間交代
- SLO: メッセージ送信成功率99.95%、APIレスポンスp95 500ms以内
- インシデント対応: PagerDuty呼び出しから5分以内一次応答、30分以内ステータスページ更新
- カオスエンジニアリング: 月1回 Gremlin で障害注入 (本番に近いステージング)
- パフォーマンステスト: k6 負荷テストをCIで毎晩
- リリースバジェット: SLO違反時はリリース自動停止、根本原因分析完了まで再開不可
- 設定変更履歴: Terraform State 差分をGitに記録、本番環境変更は2名承認制
- DBマイグレーション: pg-online-schema-change によるロックフリー
- セキュリティパッチ運用: クリティカル24時間以内、ハイ7日以内、ミディアム30日以内
- バックアップリストア演習: 四半期に1回、別環境で本番バックアップから復旧
- カスタマーアドバイザリ: 重大インシデント発生時、1時間以内ステータス更新、24時間以内に詳細レポート

## 29. 音声・ビデオハドル
- チャネル・DM内で音声・ビデオハドル開始可能
- 最大25名 (Pro) / 50名 (Enterprise)
- 開始時にチャネル内通知、ワンクリック参加
- 画面共有: 全画面・特定ウィンドウ・特定ブラウザタブ
- 録画はオーナー権限を持つユーザーのみ開始可能
- 録画は投稿チャネルにファイルとして自動添付、文字起こし付与
- 録画はデフォルト90日間保管 (保存ポリシーに従う)
- 音声品質指標 (パケロス・ビットレート・遅延) はリアルタイムでホストに表示
- バックグラウンドノイズ除去をデフォルト有効、ユーザー設定で無効化可能

## 30. カスタムフォーム
- ワークフロー内またはメッセージから独立して作成可能
- フィールド種別: テキスト (短文/長文)、数値、選択 (単一/複数)、日付、日時、ファイルアップロード、ユーザー選択、チャネル選択
- 必須項目・バリデーションルール (正規表現・最小最大値) 設定可能
- 回答結果は作成者または指定通知先チャネルに投稿
- CSVエクスポート可能 (フォーム作成者のみ)
- 公開リンク経由でワークスペース外からも回答可能 (Enterprise)
- 匿名回答オプション、匿名時はエクスポートに回答者情報含めない

## 31. コンプライアンスとガバナンス
- リーガルホールド: 特定ユーザー/チャネルのデータを保存ポリシーに関わらず削除しない (Enterprise)
- リーガルホールドはコンプライアンス管理者ロールのみ設定可、本人にも通知されない
- eDiscovery: コンプライアンスエクスポートに加え、特定キーワード・期間・ユーザーで絞り込み検索
- データ主権: GDPR・CCPA・APPI等への対応フロー
- 「忘れられる権利」: ユーザー要求から30日以内に匿名化または削除完了
- SIEMフォワーディング: Splunk・Datadog・QRadarへリアルタイム配信 (Enterprise)
- 準拠: ISO 27001、SOC 2 Type II、ISMAP、HIPAA (Enterpriseの一部プランのみ)
- データ処理委託先一覧 (Sub-processors) は公開、変更時は事前通知
- カスタマー鍵管理 (CKM): Enterprise契約者は自社KMSで暗号化鍵管理可能
- カスタマーアクセスログ機能 (Enterprise)

## 32. オンボーディングと管理ツール
- セットアップウィザード: 新規ワークスペース作成時、初期チャネル・招待・通知設定をガイド
- 新規メンバー初回ログイン時: プロフィール・通知・チャネル参加のチュートリアル
- 管理コンソールで全ユーザーアクティビティ統計 (DAU/MAU・メッセージ数・チャネル別参加率) 確認可能
- 必須参加チャネル強制設定可能 (例: 「お試しチャネル」「会社からのお知らせ」)
- セキュリティアラート: 異常ログイン (新デバイス・異常な地域) を本人と管理者へ通知
- 健全性チェック: 「使われていないチャネル」「長期間ログインしていないユーザー」を週次レポート

## 34. 利用規約とプライバシー
- ワークスペース作成時に利用規約・プライバシーポリシーへの同意要求
- ポリシー変更時は30日前に全ワークスペース管理者へメール通知、再同意要求
- ユーザーデータ利用範囲はサービス提供と統計分析に限定、第三者販売なし
- 統計分析データは匿名化後に利用
- ユーザーは自身のデータをJSON/CSVでエクスポート可能 (個人エクスポート)
- アカウント削除時は個人データを30日以内に完全消去 (バックアップからは90日以内)
- Cookie: ファーストパーティCookieのみ、サードパーティトラッカー不使用
- マーケティング通信: ニュースレター・新機能アナウンスは個別オプトイン/アウト
- 未成年: 13歳未満は利用不可、13-17歳は保護者同意ワークフロー必須
- 法執行機関からの開示要求: 透明性レポートを年2回公開
- CSAM: NCMEC等への自動通報、即座のアカウント停止
- バイオメトリクスデータ: サーバーに送信せずデバイス内処理
- データ常駐: 米国・EU・日本・シンガポールから選択 (Enterprise)
- リテンションロック: コンプライアンス要件で削除を防ぐべきデータは管理者でも削除不可に設定可能
- プライバシー強化: 「自分の名前で検索可能か」を個別制御可能
- 外部共有制限: 共有チャネル機能をワークスペース/チャネル単位で禁止可能
- 退職者データ: 「読み取り専用ユーザー」保持または即時無効化を選択
- データ持ち出し検知: 短時間に大量のメッセージダウンロードや検索発生時に管理者通知
- 透明性レポート: 政府開示要求件数、自動コンテンツ削除件数等を半年ごと公開
