## 1. 概要

- 社内コミュニケーション用のチャットツール。
- 本書はコア機能のみを定義する。

## 2. ユーザーとワークスペース

機能要件（`data-category="functional"`）:
- ユーザーは1つのメールアドレスにつき1アカウントを保持する。
- 各ユーザーは複数のワークスペースに所属できる。
- ワークスペースはチームの単位であり、URLサブドメインで識別される（例: `acme.linkr.app`）。
- ワークスペース内のユーザーは表示名とアバター画像を設定できる。

図（SVG diagram）:
- 図のラベル: 「ユーザー・ワークスペース・サブドメインの関係図」
- タイトルテキスト: 「ユーザーは複数ワークスペースに所属し、ワークスペースはサブドメインで識別される」
- 図中の要素: 「ユーザー」「ワークスペースA」「ワークスペースB」「acme.linkr.app」「サブドメイン」、およびそれらを結ぶ線。
- キャプション: 「ユーザーと複数ワークスペースの所属関係 (サブドメインで識別)」

## 3. チャネル

機能要件:
- チャネルはワークスペース内のトピック単位の会話空間である。
- チャネルにはパブリックとプライベートの2種類がある。

### 3.1 チャネル種別（表「チャネル種別と閲覧・参加範囲」）

| 種別 | 閲覧・参加範囲 |
|---|---|
| パブリック | ワークスペース全員が閲覧・参加可能 |
| プライベート | 招待されたメンバーのみ閲覧可能 |

両行とも `data-category="functional"`。

### 3.2 チャネル運用ルール

機能要件:
- チャネル名は半角英数字とハイフンのみ、最大80文字。
- チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ。
- ユーザーは任意のパブリックチャネルに自由参加できる。
- ユーザーは自分が参加中のチャネルから退出できる。
  - 警告（`data-category="warning"`）: ただし最後の管理者は退出前に他メンバーに管理権限を委譲する必要がある。

## 4. メッセージ

機能要件:
- ユーザーはチャネル内にテキストメッセージを投稿できる。
- メッセージは最大4000文字。
- メッセージには投稿時刻と投稿者が記録される。
- メッセージは投稿後も時系列で永続表示される。
- メッセージ本文ではMarkdown記法（太字、斜体、コードブロック、リンク）が利用可能。

## 5. ダイレクトメッセージ (DM)

機能要件:
- ユーザーは同一ワークスペース内の任意のユーザーにDMを送信できる。
- DMは1対1の会話のみをサポートする（グループDMは本書のスコープ外）。
- DM内のメッセージはチャネルメッセージと同じ仕様（4000文字上限・Markdown対応）。

セキュリティ要件（`data-category="security"`）:
- DMは送信者と受信者のみが閲覧でき、ワークスペース管理者であっても他人のDMは閲覧できない。

## 6. 非機能要件（表「非機能要件一覧」、全行 `data-category="nfr"`）

| 項目 | 要件 |
|---|---|
| 同時接続ユーザー数 | 1ワークスペースあたり最大500 |
| メッセージ送信レイテンシ | 送信から表示まで95%tileで500ms以内 |
| データ永続性 | メッセージは無期限保存（削除ポリシーは別途定義） |
| 対応ブラウザ | 最新版のChrome / Safari / Firefox / Edge |

## 7. 技術スタック制約（`data-category="tech"`）

- フロントエンド: TypeScript + React（バージョン18以上）
- バックエンド: Go（1.22以上）、HTTPフレームワークはchi
- データストア: PostgreSQL 16（主データ）+ Redis 7（セッション・Pub/Sub）
- リアルタイム通信: WebSocket（RFC 6455準拠）
- インフラ: AWS東京リージョン（ap-northeast-1）で稼働、IaCはTerraformで管理
