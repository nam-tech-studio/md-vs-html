## 1. 概要
- 社内コミュニケーション用のチャットツール。
- 本書は **コア機能、コミュニケーション支援機能、および拡張機能** を定義する。

---

## 2. ユーザーとワークスペース
- ユーザーは **1つのメールアドレスにつき1アカウント** を保持する。
- 各ユーザーは複数のワークスペースに所属できる。
- ワークスペースはチームの単位であり、**URLサブドメイン** で識別される（例: `acme.linkr.app`）。
- ワークスペース内のユーザーは **表示名とアバター画像** を設定できる。
- プレゼンス状態は **オンライン / 退席中 / オフライン** の3種類。
- **10分間操作がない** 場合、自動的に「退席中」となる。
- ステータスメッセージ: **絵文字 + 最大100文字のテキスト** を任意設定可能。
- ステータスメッセージの有効期限は **1時間 / 4時間 / 終日 / カスタム / 無期限** から選択可能。
- 図: プレゼンス状態遷移図（オンライン → 退席中 → オフライン、10分無操作で自動遷移）。

---

## 3. ロールと権限
ワークスペースには3つのロールが存在する。

| ロール | 権限 |
|---|---|
| オーナー | ワークスペースの全権限。**請求設定の変更、削除、他オーナーの任命** が可能 |
| 管理者 | ユーザー招待、チャネル管理、**ロール付与（オーナー以外）** が可能 |
| メンバー | 通常利用権限のみ |

- ロールはオーナーまたは管理者によって変更可能（**オーナーへの昇格はオーナーのみ**）。
- ワークスペースには **常に1名以上のオーナー** が必要。
- 退会するオーナーは **事前に他ユーザーへオーナー権限を委譲** する必要がある。
- 図: ロール階層図（オーナー > 管理者 > メンバー）。

---

## 4. チャネル
- チャネルはワークスペース内のトピック単位の会話空間。
- 種類: **パブリック**（ワークスペース全員が閲覧・参加可能）、**プライベート**（招待されたメンバーのみ閲覧可能）。
- チャネル名は **半角英数字とハイフンのみ、最大80文字**。
- 作成者は自動的にメンバーとなり、**初期管理権限** を持つ。
- ユーザーは任意のパブリックチャネルに **自由参加** できる。
- 自分が参加中のチャネルから退出可能。ただし **最後の管理者は退出前に他メンバーに管理権限を委譲** する必要がある。
- チャネルには **トピック（最大250文字）** と **説明（最大1500文字）** を設定可能。
- アーカイブ: 新規メッセージ投稿不可、ただし **閲覧と検索は可能**。実行可能者は **チャネル管理者またはワークスペース管理者**。
- アーカイブされたチャネルは **ワークスペース管理者によって復元可能**。

---

## 5. メッセージ
- ユーザーはチャネル内にテキストメッセージを投稿できる。
- メッセージは **最大4000文字**。
- 投稿時刻と投稿者が記録される。
- 投稿後も時系列で永続表示される。
- **Markdown記法**（太字、斜体、コードブロック、リンク、引用、リスト）が利用可能。

### 5.1 編集と削除
- ユーザーは自分のメッセージを編集できる。
- 編集されたメッセージには **「編集済み」表示** が付く。
- **編集履歴は保存されない**（編集日時のみ記録）。
- ユーザーは自分のメッセージを削除できる。
- **チャネル管理者は他人のメッセージを削除できる（編集は不可）**。
- 削除されたメッセージは「**このメッセージは削除されました**」というプレースホルダで表示。
- 削除は **ソフトデリート**、削除ログは **90日間保管**。

### 5.2 スレッド
- 任意のメッセージへの返信でスレッドを開始可能。
- スレッド内メッセージは **チャネルのメインタイムラインには表示されない**。
- スレッド開始者および参加者は新規スレッド返信時に通知を受ける。
- 「**チャネルにも投稿**」オプションで親チャネルへ二重投稿可能。
- スレッド内でも **メンション・編集・削除・リアクション** が可能。

### 5.3 リアクション
- 絵文字リアクションを付与できる。
- **1メッセージあたり最大23種類の絵文字** を表示。
- 同一ユーザーが同一メッセージに同じ絵文字を **複数回リアクション不可**。
- カスタム絵文字は **ワークスペース管理者がアップロード**、全ユーザーが使用可能。
- カスタム絵文字: **1ファイル128KB以下、PNG / JPEG / GIF**。

---

## 6. メンション
- `@ユーザー名` でメンション可能。
- メンションされたユーザーには通知が届く（通知設定に従う）。
- `@channel`: 同一チャネルの **全メンバー** をメンション。
- `@here`: 同一チャネルで **オンラインの全メンバー** をメンション。
- `@channel` および `@here` の使用は **チャネル管理者によって制限可能**。
- ユーザーグループ（例: `@team-frontend`）を作成可能。
  - 作成は **オーナーまたは管理者**、所属メンバーを管理。
  - グループメンションはグループ内全員へのメンション扱い。

---

## 7. ファイル共有
- メッセージにファイルを添付可能。
- **1ファイルあたり最大100MB**。
- **1メッセージあたり最大10ファイル**。
- 対応形式: **画像（PNG/JPEG/GIF/WebP）、PDF、動画（MP4/MOV）、各種ドキュメント、ZIPアーカイブ**。
- **画像とPDFはインラインプレビュー** 表示。
- ファイルは **アクセス権を持つチャネル参加者のみ閲覧可能**。
- 削除可能者: **投稿者またはチャネル管理者**。
- 削除されたファイルは即座にダウンロード不可。**メタデータは90日間保持**。

---

## 8. ダイレクトメッセージ (DM)
- 同一ワークスペース内の任意のユーザーにDMを送信可能。
- **1対1の会話のみサポート**（グループDMはスコープ外）。
- DM内メッセージはチャネルメッセージと同じ仕様（**4000文字上限・Markdown対応・編集・削除・スレッド・リアクション・ファイル添付すべて利用可能**）。
- **ワークスペース管理者であっても他人のDMは閲覧不可**。

---

## 9. 検索
- 検索対象: **メッセージ本文・チャネル名・ユーザー名・ファイル名** を横断。
- 検索結果は **メッセージ・チャネル・ユーザー・ファイル** のカテゴリ別に表示。
- 検索範囲は **本人がアクセス権を持つチャネルとDMに限定**。

絞り込みフィルタ:
| フィルタ | 用途 |
|---|---|
| `from:ユーザー名` | 投稿者で絞り込み |
| `in:チャネル名` | チャネルで絞り込み |
| `before:YYYY-MM-DD` / `after:YYYY-MM-DD` | 期間で絞り込み |
| `has:link` / `has:file` / `has:reaction` | 添付要素で絞り込み |

- 並び替え: **関連度順またはタイムスタンプ降順**。
- **1クエリあたり最大1000件**、超過分はページングで取得。

---

## 10. 通知
チャネル通知レベル（3種類）:
- すべてのメッセージ
- メンション時のみ
- なし

- **DMは常にメンション扱い** で通知される。
- 通知レベルは **チャネル単位、およびワークスペース全体のデフォルト** として設定可能。
- ワークスペースのデフォルトは **チャネル個別設定で上書き** される。
- 「通知を一時停止」: **30分・1時間・2時間・翌朝9時まで** の全通知抑止。
- 通知抑止中も **アプリ内バッジ表示は継続**。
- **キーワード通知**: 自身のキーワードリスト設定で、メッセージにキーワード含有時に通知。
- 通知方法: **ブラウザ通知、メール、モバイルプッシュ**（モバイルアプリは別書で定義）。
- 優先順位: **ブラウザ未開封時はメール、5分以内に既読されない場合のみメールを送る**。

---

## 11. ピン留めとブックマーク
- **チャネル管理者またはチャネルメンバー** が重要メッセージを「ピン留め」可能。
- ピン留めは **チャネルあたり最大100件**。
- ピン留めはチャネル内のサイドパネルから一覧アクセス可能。
- 個人ブックマークは任意のメッセージを保存可能。
  - **本人のみ閲覧可能**。
  - 上限なし。ただし **500件を超えるとフォルダ分類が推奨**。

---

## 12. 下書きと送信予約
- 入力中の内容は **自動的に下書き保存**。
- 下書きは **チャネル / DM / スレッド単位** で保持。
- アプリ再起動後も復元される。
- 送信予約: **最大120日先までの任意の日時** を予約可能。
- 予約済みメッセージは **送信前に編集または取り消しが可能**。
- 「**送信予約一覧**」画面から確認可能。

---

## 13. 既読管理
- 各メッセージは **閲覧者単位** で既読 / 未読状態を保持。
- チャネル一覧では **未読のあるチャネルが太字** で表示。
- 「未読をすべて既読にする」アクションでチャネル単位またはワークスペース単位の一括既読化。
- 既読位置（**最終既読メッセージID**）はサーバーで保存され、**デバイス間で同期**。

---

## 14. リマインダー
- 任意のメッセージにリマインダーを設定可能。
- プリセット: **10分後 / 1時間後 / 明日朝9時 / 来週月曜朝9時** またはカスタム。
- 時刻になると **リマインダーBotから本人にDM** で通知。
- DMには **元メッセージへのリンク** が含まれる。
- 自分のリマインダー一覧は「**リマインダー**」タブから確認・取り消し可能。
- **同一メッセージに対するリマインダーはユーザー1人につき1件まで**。再設定すると上書き。

---

## 15. 投票
- チャネル内で投票メッセージを作成可能。
- **質問文は最大200文字**、**選択肢は最大10個（各最大80文字）**。
- 形式: **単一選択 / 複数選択**。
- 締切: **無期限・1時間後・1日後・カスタム日時**。
- 表示形式: **集計のみ** または **各選択肢に投票したユーザー名を公開** から作成時に選択可能。
- 作成者は締切前に **強制終了可能**。
- 締切後は **新規投票不可だが結果は閲覧可能**。

---

## 16. 音声メッセージ
- メッセージ入力欄から録音・送信可能。
- **最大長は5分**。
- 録音形式: **Opus（WebM コンテナ）**。
- インラインプレーヤーで再生、再生速度 **0.5x / 1x / 1.5x / 2x** を選択可能。
- **自動文字起こし** の対象。結果はメッセージ下に **折りたたみ表示**。
- ファイル添付の保存ポリシーが適用される。

---

## 17. 通知ダイジェスト
- メール通知ダイジェストを有効化可能。
- 頻度: **「1日2回（朝9時・夕方17時）」「1日1回（朝9時）」「無効」**。
- 内容: **未読のメンション・キーワードヒット・DM** をチャネル別に集約。
- メールには **各メッセージへの直接リンク** を含む。
- **新規メンションがない場合はメールを送信しない**。
- ワークスペース管理者によって **ワークスペース単位で無効化可能**。

---

## 18. 監査と履歴
ワークスペース管理者が監査ログで確認できるイベント:
- ユーザーのログイン / ログアウト
- ロール変更
- チャネル作成 / アーカイブ / 復元
- ユーザー招待 / 退会

- 監査ログは **90日間保持**（拡張は別書のエンタープライズ機能で定義）。
- **通常メンバーは自身の操作履歴のみ過去30日分閲覧可能**。

---

## 19. 非機能要件

| 項目 | 要件 |
|---|---|
| 同時接続ユーザー数 | 1ワークスペースあたり最大2000 |
| メッセージ送信レイテンシ | 送信から表示まで **95パーセンタイルで500ms以内** |
| 検索応答時間 | 95パーセンタイルで1秒以内（**1000万メッセージ規模のワークスペース**） |
| ファイルアップロード | 100MBファイルが30秒以内に完了（**回線100Mbps想定**） |
| データ永続性 | メッセージは **無期限保存**（削除ポリシーは別途定義） |
| 対応ブラウザ | 最新版のChrome / Safari / Firefox / Edge |
| アクセシビリティ | **WCAG 2.1 Level AA 準拠** |
| 国際化 | 日本語、英語、簡体字中国語、韓国語に対応（UI翻訳） |

## 20. 技術スタック制約

| レイヤー | 構成 |
|---|---|
| フロントエンド | TypeScript + **React 18以上**、状態管理は **Zustand**、ビルドは **Vite**、ルーティングは **React Router 6** |
| バックエンド | **Go 1.22以上**、HTTPフレームワークは **chi**、認証は **JWT (RS256)**、ORMは **sqlc** |
| リアルタイム通信 | **WebSocket (RFC 6455)**、**Redis Pub/Sub** でサーバー間ファンアウト |
| メッセージキュー | 通知配信・監査ログ書き込みは **Amazon SQS** で非同期処理 |
| インフラ | **AWS東京リージョン (ap-northeast-1)**、IaCは **Terraform**、コンテナは **ECS Fargate** |
| CI/CD | **GitHub Actions**、**Blue/Greenデプロイ**、E2Eテストは **Playwright** |
| 監視 | **Datadog (APM/Logs)**、**PagerDuty** でオンコール通知 |
| セキュリティ | WAFは **AWS WAF**、依存脆弱性スキャンは **Snyk + Dependabot** |

データストア:
- **PostgreSQL 16**: 主データ、**リードレプリカ最低2台**。
- **Redis 7**: セッション、Pub/Sub、レートリミット。
- **Elasticsearch 8**: 全文検索。
- **S3**: ファイル添付、**署名付きURL** でアクセス制御。

---

## 21. セキュリティ要件
- 通信はすべて **TLS 1.2以上** で暗号化。
- パスワードは **bcrypt（cost=12）** でハッシュ化して保存。
- セッションは **絶対24時間でタイムアウト**、再認証が必要。
- 全APIに **レートリミット（IP単位とユーザー単位の両方）** を適用。
- 監査ログは改竄防止のため **別アカウントのS3バケットに WORM（Write Once Read Many）** で保存。
