チャットツール「Linkr」要件定義書 (Standard) の内容を漏らさず列挙します。

---

## 1. 概要

- 社内コミュニケーション用のチャットツールである
- 本書は「コア機能」と「コミュニケーション支援機能」を定義する

## 2. ユーザーとワークスペース

- ユーザーは1つのメールアドレスにつき1アカウントを保持する
- 各ユーザーは複数のワークスペースに所属できる
- ワークスペースはチームの単位であり、URLサブドメインで識別される(例: `acme.linkr.app`)
- ワークスペース内のユーザーは表示名とアバター画像を設定できる
- ユーザーはプレゼンス状態を3種類持つ: **オンライン / 退席中 / オフライン**
- プレゼンスは10分間操作がない場合に自動的に「退席中」となる

## 3. チャネル

- チャネルはワークスペース内のトピック単位の会話空間である
- チャネルには2種類ある
  - **パブリック**: ワークスペース全員が閲覧・参加可能
  - **プライベート**: 招待されたメンバーのみ閲覧可能
- チャネル名は半角英数字とハイフンのみ、最大80文字
- チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ
- ユーザーは任意のパブリックチャネルに自由参加できる
- ユーザーは自分が参加中のチャネルから退出できる
- **エッジケース**: 最後の管理者は退出前に他メンバーに管理権限を委譲する必要がある

## 4. メッセージ

- ユーザーはチャネル内にテキストメッセージを投稿できる
- メッセージは最大4000文字
- メッセージには投稿時刻と投稿者が記録される
- メッセージは投稿後も時系列で永続表示される
- メッセージ本文ではMarkdown記法が利用可能(**太字、斜体、コードブロック、リンク**)

## 5. メンション

- メッセージ内で `@ユーザー名` を入力するとそのユーザーをメンションできる
- メンションされたユーザーには通知が届く(通知設定に従う)
- `@channel` は同一チャネルの全メンバーをメンションする
- `@here` は同一チャネルでオンラインの全メンバーをメンションする
- `@channel` および `@here` の使用はチャネル管理者によって制限可能

## 6. ダイレクトメッセージ (DM)

- ユーザーは同一ワークスペース内の任意のユーザーにDMを送信できる
- DMは1対1の会話のみをサポートする(**グループDMは本書のスコープ外**)
- DM内のメッセージはチャネルメッセージと同じ仕様(4000文字上限・Markdown対応)
- DMは送信者と受信者のみが閲覧でき、**ワークスペース管理者であっても他人のDMは閲覧できない**

## 7. 検索

- ユーザーはメッセージ本文・チャネル名・ユーザー名を横断検索できる
- 検索結果はメッセージ・チャネル・ユーザーのカテゴリ別に表示される
- 検索対象は本人がアクセス権を持つチャネルとDMに限定される
- 利用できる絞り込みフィルタ:
  - `from:ユーザー名` — 投稿者で絞り込み
  - `in:チャネル名` — チャネルで絞り込み
  - `before:YYYY-MM-DD` / `after:YYYY-MM-DD` — 期間で絞り込み
- 検索結果は **関連度順** または **タイムスタンプ降順** で並び替えできる
- 1クエリあたり最大1000件まで返却し、それ以上はページングで取得する

## 8. 通知

- チャネル通知レベルは3種類設定可能:
  - すべてのメッセージ
  - メンション時のみ
  - なし
- DMは常にメンション扱いで通知される
- 通知レベルはチャネル単位、およびワークスペース全体のデフォルトとして設定可能
- ワークスペースのデフォルトはチャネル個別設定で上書きされる
- 「通知を一時停止」機能で全通知を抑止できる、抑止期間: **30分 / 1時間 / 2時間 / 翌朝9時まで**
- 通知抑止中もアプリ内バッジ表示は継続する

## 9. 非機能要件

- **同時接続ユーザー数**: 1ワークスペースあたり最大500
- **メッセージ送信レイテンシ**: 送信から表示まで95パーセンタイルで500ms以内
- **検索応答時間**: 95パーセンタイルで1秒以内(1000万メッセージ規模のワークスペース)
- **データ永続性**: メッセージは無期限保存(削除ポリシーは別途定義)
- **対応ブラウザ**: 最新版のChrome / Safari / Firefox / Edge
- **アクセシビリティ**: WCAG 2.1 Level AA 準拠

## 10. 技術スタック制約

- **フロントエンド**: TypeScript + React 18以上、状態管理は Zustand、ビルドは Vite
- **バックエンド**: Go 1.22以上、HTTPフレームワークは chi、認証は JWT (RS256)
- **データストア**:
  - PostgreSQL 16: ユーザー・チャネル・メッセージ等の主データ
  - Redis 7: セッションストア、Pub/Sub(リアルタイム配信)、レートリミット
  - Elasticsearch 8: 全文検索インデックス(本文・チャネル名・ユーザー名)
- **リアルタイム通信**: WebSocket (RFC 6455準拠)、ハートビートは30秒間隔
- **インフラ**: AWS東京リージョン (ap-northeast-1)、IaC は Terraform、コンテナは ECS Fargate
- **CI/CD**: GitHub Actions、デプロイは Blue/Green でダウンタイム0
- **監視**: Datadog で APM・メトリクス・ログを統合監視

## 11. セキュリティ要件

- 通信はすべて TLS 1.2以上で暗号化される
- パスワードは bcrypt (cost=12) でハッシュ化して保存
- セッションは絶対24時間でタイムアウトし、再認証が必要

## 用語

- **Linkr** — チャットツールの名称
- **Standard** — 本要件定義書のグレード/版
- **ワークスペース** — チームの単位、URLサブドメインで識別
- **チャネル** — ワークスペース内のトピック単位の会話空間(パブリック/プライベート)
- **DM** — ダイレクトメッセージ(1対1)
- **プレゼンス** — オンライン/退席中/オフライン
- **メンション** — `@ユーザー名` / `@channel` / `@here`
