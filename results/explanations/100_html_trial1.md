# 文書タイトル
**チャットツール「Linkr」要件定義書 (Standard)**

`<html lang="ja">`、文字コード `UTF-8`、ビューポート設定あり、外部スタイル `./styles.css` をリンク。

---

## 1. 概要
- 社内コミュニケーション用のチャットツールである。
- 本書は **コア機能** と **コミュニケーション支援機能** を定義する。

---

## 2. ユーザーとワークスペース

リスト記載要件:
- ユーザーは **1つのメールアドレスにつき1アカウント** を保持する。
- 各ユーザーは **複数のワークスペース** に所属できる。
- ワークスペースは **チームの単位** であり、**URLサブドメイン** で識別される(例: `acme.linkr.app`)。
- ワークスペース内のユーザーは **表示名** と **アバター画像** を設定できる。
- ユーザーは **オンライン / 退席中 / オフライン** の **3種類のプレゼンス状態** を持つ。
- プレゼンスは **10分間操作がない場合に自動的に「退席中」** となる。

図(SVG):「プレゼンス状態遷移図」
- 3つの状態ボックス: オンライン(緑系) → 退席中(黄系) → オフライン(灰系)。
- オンライン→退席中の遷移に「10分無操作」の注記、矢印は実線。
- 退席中→オフラインの遷移は破線(`stroke-dasharray="4 3"`)。
- 図キャプション: 「プレゼンス状態(オンライン / 退席中 / オフライン)と『10分間無操作で退席中』の自動遷移」。

---

## 3. チャネル

導入リスト:
- チャネルは **ワークスペース内のトピック単位の会話空間** である。
- チャネルには **パブリック** と **プライベート** の2種類がある。

### 3.1 チャネル種別 (表「チャネル種別と閲覧範囲」)
| 種別 | 閲覧・参加範囲 |
|---|---|
| パブリック | ワークスペース全員が閲覧・参加可能 |
| プライベート | 招待されたメンバーのみ閲覧可能 |

### 3.2 命名・運用ルール
- チャネル名は **半角英数字とハイフンのみ**、**最大80文字**。
- チャネル作成者は **自動的にメンバー** となり、**初期管理権限** を持つ。
- ユーザーは **任意のパブリックチャネルに自由参加** できる。
- ユーザーは自分が **参加中のチャネルから退出** できる。
  - **エッジケース(警告):** ただし **最後の管理者は退出前に他メンバーに管理権限を委譲する必要** がある。

---

## 4. メッセージ
- ユーザーはチャネル内に **テキストメッセージ** を投稿できる。
- メッセージは **最大4000文字**。
- メッセージには **投稿時刻** と **投稿者** が記録される。
- メッセージは投稿後も **時系列で永続表示** される。
- メッセージ本文では **Markdown記法(太字、斜体、コードブロック、リンク)** が利用可能。

---

## 5. メンション
- メッセージ内で `@ユーザー名` を入力するとそのユーザーをメンションできる。
- メンションされたユーザーには **通知** が届く(**通知設定に従う**)。

### 5.1 特殊メンション (定義リスト)
- `@channel` … 同一チャネルの **全メンバー** をメンションする。
- `@here` … 同一チャネルで **オンラインの全メンバー** をメンションする。

警告事項:
- `@channel` および `@here` の使用は **チャネル管理者によって制限可能**。

---

## 6. ダイレクトメッセージ (DM)
- ユーザーは **同一ワークスペース内の任意のユーザー** にDMを送信できる。
- DMは **1対1の会話のみ** をサポート(**グループDMは本書のスコープ外**)。
- DM内のメッセージはチャネルメッセージと **同じ仕様(4000文字上限・Markdown対応)**。
- セキュリティ: DMは **送信者と受信者のみが閲覧** でき、**ワークスペース管理者であっても他人のDMは閲覧できない**。

---

## 7. 検索

導入要件:
- ユーザーは **メッセージ本文・チャネル名・ユーザー名** を横断検索できる。
- 検索結果は **メッセージ・チャネル・ユーザー** のカテゴリ別に表示される。
- 検索対象は **本人がアクセス権を持つチャネルとDMに限定** される。

### 7.1 絞り込みフィルタ (表「利用可能な検索フィルタ」)
| フィルタ | 機能 |
|---|---|
| `from:ユーザー名` | 投稿者で絞り込み |
| `in:チャネル名` | チャネルで絞り込み |
| `before:YYYY-MM-DD` / `after:YYYY-MM-DD` | 期間で絞り込み |

### 7.2 並び替え・ページング
- 検索結果は **関連度順** または **タイムスタンプ降順** で並び替えできる。
- **1クエリあたり最大1000件** まで返却し、それ以上は **ページングで取得**。

---

## 8. 通知

### 8.1 チャネル通知レベル
各ユーザーは以下の **3種類** を設定できる(表「チャネル通知レベル」):
- すべてのメッセージ
- メンション時のみ
- なし

### 8.2 設定の優先順位とDM
- **DMは常にメンション扱いで通知** される。
- 通知レベルは **チャネル単位**、および **ワークスペース全体のデフォルト** として設定可能。
- **ワークスペースのデフォルトはチャネル個別設定で上書きされる**。

### 8.3 通知の一時停止
- 「通知を一時停止」機能で **30分・1時間・2時間・翌朝9時まで** 全通知を抑止できる。
- 通知抑止中も **アプリ内バッジ表示は継続** する。

## 9. 非機能要件 (表「非機能要件一覧」)

| 項目 | 要件 |
|---|---|
| 同時接続ユーザー数 | 1ワークスペースあたり **最大500** |
| メッセージ送信レイテンシ | 送信から表示まで **95パーセンタイルで500ms以内** |
| 検索応答時間 | **95パーセンタイルで1秒以内**(**1000万メッセージ規模のワークスペース**) |
| データ永続性 | メッセージは **無期限保存**(**削除ポリシーは別途定義**) |
| 対応ブラウザ | 最新版の **Chrome / Safari / Firefox / Edge** |
| アクセシビリティ | **WCAG 2.1 Level AA 準拠** |

## 10. 技術スタック制約

### 10.1 アプリケーション層 (定義リスト)
- **フロントエンド:** TypeScript + **React 18以上**、状態管理は **Zustand**、ビルドは **Vite**。
- **バックエンド:** **Go 1.22以上**、HTTPフレームワークは **chi**、認証は **JWT (RS256)**。

### 10.2 データストア (表「データストア構成」)
| プロダクト | 用途 |
|---|---|
| PostgreSQL 16 | ユーザー・チャネル・メッセージ等の主データ |
| Redis 7 | セッションストア、Pub/Sub(リアルタイム配信)、レートリミット |
| Elasticsearch 8 | 全文検索インデックス(本文・チャネル名・ユーザー名) |

### 10.3 通信・インフラ・運用 (定義リスト)
- **リアルタイム通信:** **WebSocket(RFC 6455準拠)**、**ハートビートは30秒間隔**。
- **インフラ:** **AWS東京リージョン(ap-northeast-1)**、IaCは **Terraform**、コンテナは **ECS Fargate**。
- **CI/CD:** **GitHub Actions**、デプロイは **Blue/Green** で **ダウンタイム0**。
- **監視:** **Datadog** で **APM・メトリクス・ログを統合監視**。

---

## 11. セキュリティ要件
- 通信はすべて **TLS 1.2以上** で暗号化される。
- パスワードは **bcrypt (cost=12)** でハッシュ化して保存。
- セッションは **絶対24時間でタイムアウト** し、**再認証が必要**。
