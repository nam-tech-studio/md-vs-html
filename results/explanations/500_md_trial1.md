要件定義書「チャットツール『Linkr』要件定義書 (Enterprise)」の内容を、章立てに従って一切漏らさず列挙します。原文に書かれていることのみを述べます。

## 1. 概要

- 社内コミュニケーション用のチャットツール
- 本書はコア機能、コミュニケーション支援機能、拡張機能、エンタープライズ向けの管理・統合・コンプライアンス機能を定義する

## 2. ユーザーとアカウント

- 1メールアドレスにつき1アカウント
- ユーザーは複数ワークスペースに所属可能
- パスワード認証またはSSOで認証
- パスワード: 最低12文字、英大文字・小文字・数字・記号を各1文字以上
- パスワードリセット: 登録メールへトークン送信、トークン有効期限60分
- 連続5回ログイン失敗で15分ロック
- プレゼンス3種: オンライン / 退席中 / オフライン
- 10分間操作なしで自動「退席中」
- ステータスメッセージ: 絵文字 + 最大100文字、有効期限 (1時間 / 4時間 / 終日 / カスタム / 無期限)
- 2要素認証: TOTP / SMS / WebAuthn
- 2要素認証はワークスペースオーナーが組織全体に強制可能

## 3. ワークスペース

- チームの単位、URLサブドメインで識別 (例: `acme.linkr.app`)
- 表示名・アバター画像を設定可能
- ワークスペース名・ロゴ・アイコンはオーナーが変更可能
- プラン別機能制限:
  - **Free**: ユーザー50名まで、メッセージ履歴は直近10000件まで閲覧可能
  - **Pro**: ユーザー無制限、メッセージ履歴無制限
  - **Enterprise**: SSO、SCIM、データ常駐リージョン指定、エンタープライズキー管理、監査ログ拡張
- 削除実行から30日間は復元可能 (オーナーのみ)
- 30日経過後はメッセージ・ファイル・設定すべてが完全消去

## 4. ロールと権限

- 3つのロール
  - **オーナー**: 全権限、請求設定変更・削除・他オーナー任命可
  - **管理者**: ユーザー招待、チャネル管理、ロール付与 (オーナー以外) 可
  - **メンバー**: 通常利用権限のみ
- ロールはオーナーまたは管理者によって変更可 (オーナーへの昇格はオーナーのみ)
- 常に1名以上のオーナーが必要
- 退会するオーナーは事前に他ユーザーへ権限を委譲する必要
- ゲストユーザー: 特定チャネルにのみアクセス可能な制限ロール
  - DM・検索・他チャネル閲覧不可
  - オーナーまたは管理者のみが招待可能

## 5. 招待とユーザー管理

- 管理者・オーナーがメールアドレス指定で招待
- 招待リンク有効期限7日間
- 一度に最大100件まで一括招待可能
- ドメイン制限: 許可リストでメールドメイン制限可能
- 無効化: ログイン不可だが過去のメッセージ・名前は残る
- 完全削除: 投稿は「削除済みユーザー」として残るがアカウント情報は完全消去
- SCIM v2.0によるプロビジョニング (Enterpriseプランのみ)

## 6. SSO

- Enterpriseプランで全ワークスペースSSO有効化可
- 対応プロトコル: SAML 2.0、OIDC
- SSO有効時はパスワード認証を無効化可能 (オーナー設定)
- SSOプロバイダから受け取った属性 (名前、メール、ユーザーグループ) は自動反映
- ジャストインタイムプロビジョニング (JIT): 初回ログイン時に自動アカウント作成可能
- SCIMと併用時はユーザー作成・無効化はSCIM側を優先

## 7. チャネル

- ワークスペース内のトピック単位の会話空間
- パブリック (全員閲覧・参加可) とプライベート (招待者のみ閲覧可) の2種
- チャネル名: 半角英数字とハイフンのみ、最大80文字
- 作成者は自動メンバー、初期管理権限あり
- パブリックチャネルは自由参加可
- 退出可能、ただし最後の管理者は退出前に他メンバーへ管理権限を委譲する必要
- トピック最大250文字、説明最大1500文字
- アーカイブ後は新規投稿不可、閲覧と検索は可能
- アーカイブはチャネル管理者またはワークスペース管理者が実行可能
- アーカイブ復元はワークスペース管理者
- 共有チャネル (Connect): 他ワークスペースとチャネル共有可能 (Enterprise)
  - 共有先参加者は自身のワークスペースのアカウントで投稿
  - どちらのワークスペースの保存期間ポリシーが適用されるかを明示的に選択可能

## 8. メッセージ

- メッセージは最大4000文字
- 投稿時刻と投稿者を記録
- 時系列で永続表示
- Markdown記法 (太字、斜体、コードブロック、リンク、引用、リスト) 利用可

### 8.1 編集と削除

- 自分のメッセージを編集可
- 編集メッセージには「編集済み」表示
- 編集履歴は保存しない、編集日時のみ記録
- 自分のメッセージを削除可
- チャネル管理者は他人のメッセージを削除可 (編集は不可)
- 削除メッセージは「このメッセージは削除されました」とプレースホルダ表示
- ソフトデリート、削除ログは90日間保管

### 8.2 スレッド

- 任意メッセージへの返信でスレッド開始
- スレッド内メッセージはメインタイムラインに表示されない
- スレッド開始者・参加者は新規返信時に通知
- 「チャネルにも投稿」オプションで親チャネルへ二重投稿可能
- スレッド内でもメンション・編集・削除・リアクション可

### 8.3 リアクション

- 絵文字リアクション付与可
- 1メッセージあたり最大23種類の絵文字を表示
- 同一ユーザーが同一メッセージへ同じ絵文字を複数回つけることは不可
- カスタム絵文字はワークスペース管理者がアップロード、全ユーザーが使用可
- カスタム絵文字: 1ファイル128KB以下、PNG / JPEG / GIF形式

## 9. メンション

- `@ユーザー名` でメンション
- メンションされたユーザーには通知 (通知設定に従う)
- `@channel`: チャネル全メンバーをメンション
- `@here`: チャネル内オンラインの全メンバーをメンション
- `@channel`・`@here`の使用はチャネル管理者が制限可能
- ユーザーグループ (`@team-frontend` 等) を作成可能
  - オーナーまたは管理者が作成・所属メンバー管理
  - グループメンションはグループ全員へのメンションとして扱う

## 10. ファイル共有

- 1ファイル最大100MB (Enterprise最大5GB)
- 1メッセージ最大10ファイル
- 対応形式: 画像 (PNG/JPEG/GIF/WebP)、PDF、動画 (MP4/MOV)、各種ドキュメント、ZIPアーカイブ
- 画像とPDFはインラインプレビュー
- アクセス権を持つチャネル参加者のみ閲覧可
- 投稿者またはチャネル管理者が削除可
- 削除後は即座にダウンロード不可、メタデータは90日間保持
- ウイルススキャン: アップロード時に自動実行、検知時は隔離

## 11. ダイレクトメッセージ (DM)

- 同一ワークスペース内任意ユーザーへDM送信可
- DMは1対1のみサポート
- マルチパーティDM (MPDM): 最大8名のグループDM、名前なしの一時的会話
- DM内メッセージはチャネルメッセージと同じ仕様
- 送信者と受信者のみ閲覧、ワークスペース管理者でも他人のDMは閲覧不可
- 検索対象は本人参加のDMのみ

## 12. 検索

- メッセージ本文・チャネル名・ユーザー名・ファイル名を横断検索
- カテゴリ別 (メッセージ・チャネル・ユーザー・ファイル) 表示
- 検索範囲は本人がアクセス権を持つチャネルとDMに限定
- 絞り込みフィルタ:
  - `from:ユーザー名`
  - `in:チャネル名`
  - `before:YYYY-MM-DD` / `after:YYYY-MM-DD`
  - `has:link` / `has:file` / `has:reaction`
- 関連度順またはタイムスタンプ降順
- 1クエリ最大1000件、それ以上はページング

## 13. 通知

- チャネル通知レベル3種: すべてのメッセージ / メンション時のみ / なし
- DMは常にメンション扱いで通知
- チャネル単位、ワークスペース全体デフォルトとして設定可能
- ワークスペースデフォルトはチャネル個別設定で上書き
- 通知一時停止: 30分・1時間・2時間・翌朝9時まで
- 通知抑止中もアプリ内バッジ表示は継続
- キーワード通知: 自身のキーワードリストに該当時に通知
- 通知方法: ブラウザ通知、メール、モバイルプッシュ
- 優先順位: ブラウザ未開封時はメール、5分以内に既読されない場合のみメールを送る

## 14. ピン留めとブックマーク

- チャネル管理者またはチャネルメンバーがチャネル内ピン留め可能
- ピン留めはチャネルあたり最大100件
- サイドパネルから一覧アクセス可
- 個人ブックマークとして任意メッセージを保存可

## 15. 下書きと送信予約

- 入力中内容は自動下書き保存
- チャネル / DM / スレッド単位で保持
- アプリ再起動後も復元
- 送信予約は最大120日先まで
- 予約済みメッセージは送信前に編集または取り消し可

## 16. 既読管理

- 各メッセージは閲覧者単位で既読/未読を保持
- チャネル一覧では未読のあるチャネルが太字表示
- 「未読をすべて既読にする」でチャネル単位またはワークスペース単位の一括既読化
- 最終既読メッセージIDはサーバー保存、デバイス間で同期

## 17. ワークフロービルダー

- ノーコードのワークフロー作成
- トリガー、ステップ、アクションの組み合わせ
- トリガー種別: 絵文字リアクション、特定キーワードのメッセージ投稿、フォーム送信、スケジュール (cron形式)、Webhook受信
- ステップ種別: フォーム入力、条件分岐 (if/else)、メッセージ送信、外部API呼び出し、待機 (最大7日)
- チャネル単位またはワークスペース単位で公開可能
- 作成・編集はオーナーまたは管理者のみ、メンバーはリクエスト送信のみ
- 実行ログは過去30日間保存
- 実行回数: Pro 月10000回まで、Enterprise 無制限

## 18. AIアシスト機能

- 会話サマリー機能 (チャネル・スレッド要約)
- 手動実行または長時間オフラインだったユーザーへ自動生成提示
- スマートリプライ: 返信候補を3件まで提示
- メンション通知へのクイックリアクション (絵文字提案、定型返信提案)
- 検索クエリの自然言語入力対応 (例:「先週マーケティングがリリースした件」)
- AI生成コンテンツには必ず「AI生成」ラベル
- ワークスペース単位でオン/オフ切替可能、デフォルトはオフ
- AI機能利用時、メッセージ内容が外部AIプロバイダに送信されることをUI上で明示
- ユーザーは個人設定でオプトアウト可能 (本人のメッセージはAI処理されない)
- AI翻訳: チャネル/スレッド単位、原文と訳文を併記
- AIアクションアイテム抽出: スレッド・ハドル文字起こしから「やること」「決定事項」「期限」を構造化抽出
- AIプロバイダ選択 (Enterprise): OpenAI / Anthropic / 自社オンプレLLMから選択
- AIガードレール: 機密情報 (社外秘ラベル) チャネルはAI処理を強制スキップ
- AI利用統計を管理コンソールで確認可能、月次の使用量とコスト表示

## 19. インテグレーションと外部連携

- 「アプリ」と「Webhook」の2種類で外部連携
- アプリストアからインストール
- インストールはオーナーまたは管理者のみ、メンバーはリクエスト送信のみ
- 受信Webhook: 外部サービスからチャネルへ投稿用URL発行可
  - レートリミット: 60秒あたり最大60リクエスト
- 送信Webhook: 特定キーワード投稿時に外部URLへPOST
- スラッシュコマンド: `/コマンド名` でアプリ機能呼び出し
- すべての連携はOAuth 2.0で認可
- アプリスコープ: channels:read, chat:write, files:write など30種以上
- アプリ設定変更履歴は監査ログに記録
- アプリ提出ガイドライン: アプリストア掲載前にセキュリティ審査 (内製アプリは免除)
- カスタムインテグレーション: 単一ワークスペース内利用は管理者承認後に即時利用可能
- 連携先サービス例: GitHub、Jira、Asana、Notion、Google Drive、Zoom、Salesforce、PagerDuty、Datadog

## 20. API

- REST APIおよびWebSocket APIを提供
- 認証はBotトークンまたはユーザートークン
- レートリミット: 1ワークスペースあたり毎秒60リクエスト (バースト120)
- 超過時はHTTP 429を返却、`Retry-After`ヘッダで待機時間通知
- API投稿メッセージはBotユーザーが投稿者として記録
- イベントAPI: 新規メッセージ・編集・削除・リアクション追加など30種類以上のイベントをWebSocketで配信
- API SDK: Python / JavaScript / Goの3言語

## 21. 監査と履歴

- 監査ログ対象イベント:
  - ログイン/ログアウト (IPアドレス・User-Agent含む)
  - ロール変更
  - チャネル作成 / アーカイブ / 復元
  - ユーザー招待 / 退会 / 無効化
  - アプリのインストール / アンインストール
  - 設定変更 (SSO・SCIM・通知デフォルト等)
- 保持期間: Pro 90日間、Enterprise 7年間 (法令対応)
- 通常メンバーは自身の操作履歴のみを過去30日分閲覧可能
- 取得方法: 管理者向けUI、CSVエクスポート、SIEMフォワーディング (Enterprise)

## 22. データ保存とエクスポート

- データ保存リージョン: 米国・EU・アジア太平洋から選択可 (Enterprise)
- メッセージ保存期間ポリシー: ワークスペース全体およびチャネル単位で設定可
  - 例: 30日 / 90日 / 1年 / 無期限
  - 期限超過のメッセージは自動削除
- DMには独自の保存ポリシーを設定可
- データエクスポート:
  - **標準エクスポート**: パブリックチャネルのメッセージとファイルをZIP出力 (オーナー限定)
  - **コンプライアンスエクスポート**: プライベートチャネルおよびDMを含む全データ出力 (Enterprise限定、法的要請時のみ)
- 完了時オーナーに通知メール、ダウンロードリンクは7日間有効

## 23. モバイルアプリ

- iOS (iOS 16以降) およびAndroid (Android 10以降) のネイティブアプリ
- API経由で提供可能な全機能をサポート
- モバイル独自機能: プッシュ通知、生体認証 (Face ID / Touch ID / 指紋認証)、デバイスごとのセッション管理
- モバイルからのSSO認証はインアプリブラウザ
- リリースから365日以上経過のバージョンは強制アップデート
- オフライン時もキャッシュメッセージは閲覧可、復帰時に未送信メッセージを自動再送
- 端末紛失時: 管理者または本人がリモートでセッション失効可、失効と同時にローカルキャッシュも削除
- プッシュ通知ペイロードはメッセージ本文を含めず件名のみ送信 (情報漏洩対策)
- MDM: Microsoft Intune および Jamf による配布・ポリシー強制サポート
- 省電力モード時はバックグラウンド更新頻度を自動低下 (Android Doze、iOS Low Power Mode)

## 33. ヘルプとサポート

- アプリ内ヘルプセンター: 機能別ヘルプ記事をワークスペース言語で表示
- 右下のヘルプアイコンから検索可能
- お問い合わせフォーム: ヘルプセンターから直接サポートチケット起票可
- サポート対応時間: Pro 平日9-18時 (日本時間)、Enterprise 24/7
- ステータスページ: サービス稼働状況公開、過去90日のインシデント掲載
- フィードバック機能: 機能要望投稿可、Vote付きで他ユーザーが賛同表明可
- リリースノート: アプリ内通知バーで主要更新告知、詳細は公式ブログ
- 開発者向けドキュメント: docs.linkr.app に APIリファレンス・SDK使用例・チュートリアル
- ベータプログラム: 希望ワークスペースは新機能早期アクセス、フィードバック義務あり

## 34. 利用規約とプライバシー

- ワークスペース作成時に利用規約・プライバシーポリシー同意要求
- ポリシー変更時は30日前に全ワークスペース管理者へメール通知、再同意要求
- ユーザーデータの利用範囲はサービス提供と統計分析に限定、第三者販売しない
- 統計分析データは個人特定不能に匿名化後に利用
- ユーザーはJSONまたはCSV形式で個人エクスポート可能
- アカウント削除時、個人データを30日以内に完全消去 (バックアップからは90日以内)
- Cookie: ファーストパーティCookieのみ、サードパーティトラッカー不使用
- マーケティング通信: 個別にオプトイン/アウト可能
- 13歳未満は利用不可、13-17歳は保護者同意ワークフロー通過時のみ利用可
- 法執行機関からの開示要求: 透明性レポートを年2回公開、件数と種別を開示
- CSAM (児童性的虐待コンテンツ): NCMEC等への自動通報および即座のアカウント停止
- バイオメトリクスデータ: サーバー送信せず、デバイス内で処理
- データ常駐: 米国・EU・日本・シンガポールから選択可 (Enterprise)
- リテンションロック: コンプライアンス要件で削除を防ぐべきデータは管理者でも削除不可に設定可
- プライバシー強化機能: 「自分の名前で検索可能か」をユーザーが個別に制御可
- 外部共有制限: 共有チャネル機能の利用をワークスペース単位またはチャネル単位で禁止可
- 退職者データ: 「読み取り専用ユーザー」として保持か、即時無効化を選択可
- データ持ち出し検知: 短時間に大量のメッセージダウンロードや検索が発生した場合に管理者へ自動通知
- 透明性レポート: 政府からの情報開示要求件数、自動コンテンツ削除件数等を半年ごとに公開

## 24. アクセシビリティ

- WCAG 2.1 Level AA 準拠
- スクリーンリーダー (VoiceOver / NVDA / JAWS) 対応
- キーボードのみで全機能アクセス可能
- カラーコントラスト比 4.5:1 以上 (通常テキスト)
- フォーカスインジケータは常に可視

## 25. 国際化

- UI翻訳: 日本語、英語、簡体字中国語、韓国語、ドイツ語、フランス語、スペイン語の7言語
- ユーザーごとに表示言語選択可
- ワークスペースのデフォルト言語を管理者が設定可
- メッセージ本文は翻訳されない
- 日付・時刻はユーザーのタイムゾーンに従って表示
- 文字コード: UTF-8

## 26. セキュリティ

- 通信はTLS 1.2以上で暗号化
- 保存時暗号化: AES-256でメッセージ・ファイルを暗号化
- エンタープライズキー管理 (EKM): 顧客管理鍵 (CMK) によるキーラップ (Enterprise)
- データ漏洩防止 (DLP): カスタムパターン (クレジットカード番号等) 検出、ブロックまたは警告
- セッション管理: 全デバイス同時ログイン可、セッション一覧から個別または全セッション取消可能
- IPアドレス制限: ワークスペースアクセスを許可IPレンジに制限可 (Enterprise)
- ペネトレーションテスト: 年2回外部ベンダー実施、結果サマリーをエンタープライズ顧客に開示
- バグバウンティ: HackerOne経由、深刻度に応じて報奨金支払い
- 異常検知: 短時間の大量メッセージ削除、想定外の地理的アクセスパターンを検知し管理者へアラート
- パスワードポリシー: 過去5世代のパスワード再利用禁止、90日ごとの強制変更 (Enterpriseで設定可)
- DDoS対策: AWS Shield Advanced および Cloudflare レイヤー7保護を二重化

## 27. 非機能要件

- 同時接続ユーザー数: 1ワークスペース最大10000 (Enterprise)
- メッセージ送信レイテンシ: 95パーセンタイル500ms以内
- 検索応答時間: 95パーセンタイル1秒以内 (1000万メッセージ規模)
- ファイルアップロード: 100MBファイルが30秒以内 (回線100Mbps想定)
- 可用性: SLA 99.9% (Pro) / 99.99% (Enterprise)
- データ永続性: 年間データ損失率 < 0.0001%
- バックアップ: 全データ毎日、地理的に分離した3拠点保管 (RPO 24時間 / RTO 4時間)
- 対応ブラウザ: 最新版のChrome / Safari / Firefox / Edge

## 31. コンプライアンスとガバナンス

- リーガルホールド: 特定ユーザー・チャネルデータを保存ポリシーに関わらず削除しない (Enterprise)
- リーガルホールドはコンプライアンス管理者ロールのみ設定可、本人にも通知されない
- eDiscovery: コンプライアンスエクスポートに加え、特定キーワード・期間・ユーザーで絞り込み検索
- データ主権: GDPR・CCPA・APPI等に対応したアクセス・削除リクエストフロー
- 「忘れられる権利」: ユーザー要求から30日以内に該当データの匿名化または削除完了
- 監査ログのSIEMフォワーディング: Splunk・Datadog・QRadarへリアルタイム配信 (Enterprise)
- ISO 27001、SOC 2 Type II、ISMAP、HIPAA (Enterpriseの一部プランのみ) 準拠
- データ処理委託先一覧 (Sub-processors) は公開ページで開示、変更時は事前通知
- カスタマー鍵管理 (CKM): Enterprise契約者は自社KMSで暗号化鍵を管理可
- カスタマーアクセスログ機能 (Enterprise): 顧客自身がアクセスログを取得可

## 32. オンボーディングと管理ツール

- 新規ワークスペース作成時にセットアップウィザード (初期チャネル・招待・通知設定をガイド)
- 新規メンバー初回ログイン時に個人プロフィール設定・通知設定・チャネル参加のチュートリアル表示
- 管理コンソール: アクティビティ統計 (DAU/MAU・メッセージ数・チャネル別参加率) 確認可
- 管理者は「お試しチャネル」「会社からのお知らせ」等の必須参加チャネルを強制設定可
- セキュリティアラート: 異常ログイン (新デバイス・異常な地域) を検知し本人と管理者へ通知
- 健全性チェック: 「使われていないチャネル」「長期間ログインしていないユーザー」を週次レポート
