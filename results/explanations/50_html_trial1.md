## 2. ユーザーとワークスペース

機能要件 (`data-category="functional"`):

- ユーザーは **1つのメールアドレスにつき1アカウント** を保持する。
- 各ユーザーは **複数のワークスペース** に所属できる。
- ワークスペースは **チームの単位** であり、**URLサブドメインで識別** される (例: `acme.linkr.app`)。
- ワークスペース内のユーザーは **表示名** と **アバター画像** を設定できる。

図 (SVGダイアグラム, `aria-label="ユーザー・ワークスペース・サブドメインの関係図"`):

- タイトル: 「ユーザーは複数ワークスペースに所属し、ワークスペースはサブドメインで識別される」
- 構成要素: 「ユーザー」ボックス、「ワークスペースA」「ワークスペースB」ボックス、「acme.linkr.app」「サブドメイン」ボックス
- ユーザーから2つのワークスペースに線が引かれ、各ワークスペースからそれぞれサブドメイン要素に線が結ばれる。
- figcaption: 「ユーザーと複数ワークスペースの所属関係 (サブドメインで識別)」

## 3. チャネル

機能要件:

- チャネルは **ワークスペース内のトピック単位の会話空間** である。
- チャネルには **パブリック** と **プライベート** の2種類がある。

### 3.1 チャネル種別 (表)

表キャプション: 「チャネル種別と閲覧・参加範囲」

| 種別 | 閲覧・参加範囲 |
|---|---|
| パブリック | ワークスペース全員が閲覧・参加可能 |
| プライベート | 招待されたメンバーのみ閲覧可能 |

### 3.2 チャネル運用ルール

機能要件:

- チャネル名は **半角英数字とハイフンのみ**、**最大80文字**。
- チャネル作成者は **自動的にメンバーとなり、初期管理権限を持つ**。
- ユーザーは **任意のパブリックチャネルに自由参加** できる。
- ユーザーは自分が参加中のチャネルから **退出できる**。
  - 警告 (`data-category="warning"`): **最後の管理者は退出前に他メンバーに管理権限を委譲する必要がある**。

---

## 4. メッセージ

機能要件:

- ユーザーはチャネル内に **テキストメッセージ** を投稿できる。
- メッセージは **最大4000文字**。
- メッセージには **投稿時刻** と **投稿者** が記録される。
- メッセージは投稿後も **時系列で永続表示** される。
- メッセージ本文では **Markdown記法** が利用可能 (太字、斜体、コードブロック、リンク)。

---

## 5. ダイレクトメッセージ (DM)

機能要件:

- ユーザーは **同一ワークスペース内** の任意のユーザーにDMを送信できる。
- DMは **1対1の会話のみ** をサポートする (グループDMは本書のスコープ外)。
- DM内のメッセージはチャネルメッセージと **同じ仕様** (4000文字上限・Markdown対応)。

セキュリティ要件 (`data-category="security"`):

- DMは **送信者と受信者のみ** が閲覧でき、**ワークスペース管理者であっても他人のDMは閲覧できない**。

---

## 6. 非機能要件 (`data-category="nfr"`)

表キャプション: 「非機能要件一覧」

| 項目 | 要件 |
|---|---|
| 同時接続ユーザー数 | 1ワークスペースあたり最大500 |
| メッセージ送信レイテンシ | 送信から表示まで95%tileで500ms以内 |
| データ永続性 | メッセージは無期限保存 (削除ポリシーは別途定義) |
| 対応ブラウザ | 最新版のChrome / Safari / Firefox / Edge |

## 7. 技術スタック制約 (`data-category="tech"`)

定義リスト形式:

- **フロントエンド**: TypeScript + React (バージョン18以上)
- **バックエンド**: Go (1.22以上)、HTTPフレームワークは chi
- **データストア**: PostgreSQL 16 (主データ) + Redis 7 (セッション・Pub/Sub)
- **リアルタイム通信**: WebSocket (RFC 6455準拠)
- **インフラ**: AWS東京リージョン (`ap-northeast-1`) で稼働、IaCは Terraform で管理
