## 1. 概要

## 2. ユーザーとワークスペース

### ユーザー関連
- ユーザーは1つのメールアドレスにつき1アカウントを保持する。
- 各ユーザーは複数のワークスペースに所属できる。
- ワークスペース内のユーザーは表示名とアバター画像を設定できる。

### ワークスペース関連
- ワークスペースはチームの単位であり、URLサブドメインで識別される（例: `acme.linkr.app`）。

### プレゼンス
- ユーザーはオンライン / 退席中 / オフラインの3種類のプレゼンス状態を持つ。
- プレゼンスは10分間操作がない場合に自動的に「退席中」となる。

### プレゼンス状態遷移図（SVG図）
- 状態遷移を3つの矩形で図示:
  - 「オンライン」（緑系: `#e6f4ea` 塗り、`#34a853` 枠）
  - 「退席中」（黄系: `#fef7e0` 塗り、`#fbbc04` 枠）
  - 「オフライン」（灰系: `#f1f3f4` 塗り、`#9aa0a6` 枠）
- 矢印:
  - オンライン → 退席中: 実線、ラベル「10分無操作」
  - 退席中 → オフライン: 破線（`stroke-dasharray="4 3"`）
- 図のキャプション: 「プレゼンス状態 (オンライン / 退席中 / オフライン) と「10分間無操作で退席中」の自動遷移」

## 3. チャネル

### 基本
- チャネルはワークスペース内のトピック単位の会話空間である。
- チャネルにはパブリックとプライベートの2種類がある。

### 3.1 チャネル種別（表）
| 種別 | 閲覧・参加範囲 |
|---|---|
| パブリック | ワークスペース全員が閲覧・参加可能 |
| プライベート | 招待されたメンバーのみ閲覧可能 |

### 3.2 命名・運用ルール
- チャネル名は半角英数字とハイフンのみ、最大80文字。
- チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ。
- ユーザーは任意のパブリックチャネルに自由参加できる。
- ユーザーは自分が参加中のチャネルから退出できる。
  - **警告事項**: ただし最後の管理者は退出前に他メンバーに管理権限を委譲する必要がある。

---

## 4. メッセージ
- ユーザーはチャネル内にテキストメッセージを投稿できる。
- メッセージは最大4000文字。
- メッセージには投稿時刻と投稿者が記録される。
- メッセージは投稿後も時系列で永続表示される。
- メッセージ本文ではMarkdown記法（太字、斜体、コードブロック、リンク）が利用可能。

---

## 5. メンション
- メッセージ内で `@ユーザー名` を入力するとそのユーザーをメンションできる。
- メンションされたユーザーには通知が届く（通知設定に従う）。

### 5.1 特殊メンション
- `@channel`: 同一チャネルの全メンバーをメンションする。
- `@here`: 同一チャネルでオンラインの全メンバーをメンションする。
- **警告事項**: `@channel` および `@here` の使用はチャネル管理者によって制限可能。

---

## 6. ダイレクトメッセージ (DM)
- ユーザーは同一ワークスペース内の任意のユーザーにDMを送信できる。
- DMは1対1の会話のみをサポートする（**グループDMは本書のスコープ外**）。
- DM内のメッセージはチャネルメッセージと同じ仕様（4000文字上限・Markdown対応）。
- **セキュリティ**: DMは送信者と受信者のみが閲覧でき、ワークスペース管理者であっても他人のDMは閲覧できない。

---

## 7. 検索
- ユーザーはメッセージ本文・チャネル名・ユーザー名を横断検索できる。
- 検索結果はメッセージ・チャネル・ユーザーのカテゴリ別に表示される。
- 検索対象は本人がアクセス権を持つチャネルとDMに限定される。

### 7.1 絞り込みフィルタ
| フィルタ | 機能 |
|---|---|
| `from:ユーザー名` | 投稿者で絞り込み |
| `in:チャネル名` | チャネルで絞り込み |
| `before:YYYY-MM-DD` / `after:YYYY-MM-DD` | 期間で絞り込み |

### 7.2 並び替え・ページング
- 検索結果は関連度順またはタイムスタンプ降順で並び替えできる。
- 1クエリあたり最大1000件まで返却し、それ以上はページングで取得する。

---

## 8. 通知

### 8.1 チャネル通知レベル
各ユーザーは以下の3種類のチャネル通知レベルを設定できる:
- すべてのメッセージ
- メンション時のみ
- なし

### 8.2 設定の優先順位とDM
- DMは常にメンション扱いで通知される。
- 通知レベルはチャネル単位、およびワークスペース全体のデフォルトとして設定可能。
- ワークスペースのデフォルトはチャネル個別設定で上書きされる。

### 8.3 通知の一時停止
- 「通知を一時停止」機能で **30分・1時間・2時間・翌朝9時まで** 全通知を抑止できる。
- 通知抑止中もアプリ内バッジ表示は継続する。

## 9. 非機能要件
| 項目 | 要件 |
|---|---|
| 同時接続ユーザー数 | 1ワークスペースあたり最大500 |
| メッセージ送信レイテンシ | 送信から表示まで95パーセンタイルで500ms以内 |
| 検索応答時間 | 95パーセンタイルで1秒以内（1000万メッセージ規模のワークスペース） |
| データ永続性 | メッセージは無期限保存（**削除ポリシーは別途定義**） |
| 対応ブラウザ | 最新版のChrome / Safari / Firefox / Edge |
| アクセシビリティ | WCAG 2.1 Level AA 準拠 |

## 10. 技術スタック制約

### 10.1 アプリケーション層
- **フロントエンド**: TypeScript + React 18以上、状態管理はZustand、ビルドはVite。
- **バックエンド**: Go 1.22以上、HTTPフレームワークはchi、認証はJWT (RS256)。

### 10.2 データストア
| プロダクト | 用途 |
|---|---|
| PostgreSQL 16 | ユーザー・チャネル・メッセージ等の主データ |
| Redis 7 | セッションストア、Pub/Sub（リアルタイム配信）、レートリミット |
| Elasticsearch 8 | 全文検索インデックス（本文・チャネル名・ユーザー名） |

### 10.3 通信・インフラ・運用
- **リアルタイム通信**: WebSocket (RFC 6455準拠)、ハートビートは30秒間隔。
- **インフラ**: AWS東京リージョン (ap-northeast-1)、IaCはTerraform、コンテナはECS Fargate。
- **CI/CD**: GitHub Actions、デプロイはBlue/Greenでダウンタイム0。
- **監視**: DatadogでAPM・メトリクス・ログを統合監視。

---

## 11. セキュリティ要件
- 通信はすべてTLS 1.2以上で暗号化される。
- パスワードはbcrypt (cost=12) でハッシュ化して保存。
- セッションは絶対24時間でタイムアウトし、再認証が必要。

各要件項目には以下のカテゴリ属性が付与されている:
- `user` … ユーザー関連要件
- `workspace` … ワークスペース関連
- `presence` … プレゼンス関連
- `channel` … チャネル関連
- `message` … メッセージ関連
- `mention` … メンション関連
- `dm` … DM関連
- `search` … 検索関連
- `notification` … 通知関連
- `nfr` … 非機能要件
- `tech` … 技術スタック
- `security` … セキュリティ
- `warning` … 注意事項（最後の管理者の退出制約、`@channel`/`@here` の制限）

1. チャネル種別と閲覧範囲
2. 利用可能な検索フィルタ
