# チャットツール「Linkr」要件定義書 (Enterprise)

## 1. 概要
- 社内コミュニケーション用のチャットツール
- 本書はコア機能、コミュニケーション支援機能、拡張機能、およびエンタープライズ向けの管理・統合・コンプライアンス機能を定義する

## 2. ユーザーとアカウント
- ユーザーは1つのメールアドレスにつき1アカウントを保持する
- 各ユーザーは複数のワークスペースに所属できる
- ユーザーアカウントはパスワード認証またはSSOで認証される
- パスワードは最低12文字、英大文字・小文字・数字・記号をそれぞれ1文字以上含む必要がある
- パスワードのリセットは登録メールアドレスへのトークン送信で行われ、トークン有効期限は60分
- アカウントは連続5回のログイン失敗で15分間ロックされる
- ユーザーはオンライン / 退席中 / オフラインの3種類のプレゼンス状態を持つ
- プレゼンスは10分間操作がない場合に自動的に「退席中」となる
- ステータスメッセージ: 絵文字 + 最大100文字、有効期限は1時間 / 4時間 / 終日 / カスタム / 無期限から選択可能
- 2要素認証はTOTP / SMS / WebAuthnに対応
- 2要素認証はワークスペースオーナーが組織全体に強制可能

## 3. ワークスペース
- ワークスペースはチームの単位で、URLサブドメインで識別される(例: `acme.linkr.app`)
- ユーザーはワークスペース内で表示名とアバター画像を設定できる
- ワークスペース名・ロゴ・アイコンはオーナーが変更可能
- ワークスペースには契約プランが紐付き、プランによって機能制限が異なる
- ワークスペースは削除可能だが、削除実行から30日間は復元可能(オーナーのみ)
- 30日経過後はメッセージ・ファイル・設定すべてが完全消去される

**契約プラン比較**
- **Free**: ユーザー50名まで、メッセージ履歴は直近10000件まで閲覧可能
- **Pro**: ユーザー無制限、メッセージ履歴無制限
- **Enterprise**: SSO、SCIM、データ常駐リージョン指定、エンタープライズキー管理、監査ログ拡張

## 4. ロールと権限
- **オーナー**: ワークスペースの全権限。請求設定の変更、削除、他オーナーの任命が可能
- **管理者**: ユーザー招待、チャネル管理、ロール付与(オーナー以外)が可能
- **メンバー**: 通常利用権限のみ
- ロールはオーナーまたは管理者によって変更可能(オーナーへの昇格はオーナーのみ可能)
- ワークスペースには常に1名以上のオーナーが存在する必要がある
- 退会するオーナーは事前に他ユーザーへオーナー権限を委譲する必要がある

**ゲストユーザー**
- 特定チャネルにのみアクセス可能な制限ロール
- ゲストはDM・検索・他チャネル閲覧不可
- ゲストアカウントはオーナーまたは管理者のみが招待可能

(SVG図: オーナー > 管理者 > メンバー、別ロールとしてゲスト「特定チャネルのみ」)

## 5. 招待とユーザー管理
- 管理者およびオーナーはメールアドレス指定でユーザーを招待できる
- 招待リンクの有効期限は7日間
- 一度に最大100件のメールアドレスをまとめて招待可能
- ドメイン制限: ワークスペースに参加可能なメールドメインを許可リストで制限可能
- ユーザーはオーナーまたは管理者によって無効化または完全削除が可能
- **無効化**: ログイン不可だが過去のメッセージ・名前は残る
- **完全削除**: 投稿メッセージは「削除済みユーザー」として残るがアカウント情報は完全消去
- SCIM v2.0 (System for Cross-domain Identity Management) によるプロビジョニング (Enterpriseのみ)

## 6. SSO (シングルサインオン)
- Enterpriseプランでワークスペース全体でSSOを有効化できる
- 対応プロトコル: SAML 2.0、OIDC (OpenID Connect)
- SSO有効時はパスワード認証を無効化可能(オーナー設定)
- SSOプロバイダから受け取った属性(名前、メール、ユーザーグループ)は自動的にLinkrアカウントに反映される
- ジャストインタイムプロビジョニング(JIT): SSO初回ログイン時に自動的にアカウント作成可能
- SCIMと併用時はユーザー作成・無効化はSCIM側を優先する

## 7. チャネル
- チャネルはワークスペース内のトピック単位の会話空間
- チャネル名は半角英数字とハイフンのみ、最大80文字
- チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ
- ユーザーは任意のパブリックチャネルに自由参加できる
- ユーザーは自分が参加中のチャネルから退出できる
  - ただし最後の管理者は退出前に他メンバーに管理権限を委譲する必要がある
- チャネルにはトピック(最大250文字)と説明(最大1500文字)を設定できる

**チャネル種別**
- **パブリック**: ワークスペース全員が閲覧・参加可能
- **プライベート**: 招待されたメンバーのみ閲覧可能

**アーカイブ**
- アーカイブ後は新規メッセージ投稿不可、ただし閲覧と検索は可能
- アーカイブはチャネル管理者またはワークスペース管理者が実行可能
- アーカイブされたチャネルはワークスペース管理者によって復元可能

**共有チャネル (Connect)**
- 他ワークスペースとチャネル共有可能(Enterpriseプラン)
- 共有先ワークスペースの参加者は自身のワークスペースのアカウントで投稿
- 共有チャネルのメッセージはどちらのワークスペースの保存期間ポリシーが適用されるか明示的に選択可能

## 8. メッセージ
- ユーザーはチャネル内にテキストメッセージを投稿できる
- メッセージは最大4000文字
- 投稿時刻と投稿者が記録される
- 投稿後も時系列で永続表示される
- Markdown記法(太字、斜体、コードブロック、リンク、引用、リスト)が利用可能

### 8.1 編集と削除
- 自分のメッセージを編集できる
- 編集されたメッセージには「編集済み」表示が付く
- 編集履歴は保存されないが、編集日時は記録される
- 自分のメッセージを削除できる
- チャネル管理者は他人のメッセージを削除できる(編集は不可)
- 削除されたメッセージは「このメッセージは削除されました」のプレースホルダで表示
- メッセージ削除はソフトデリート、削除ログは90日間保管

### 8.2 スレッド
- 任意のメッセージへの返信でスレッドを開始できる
- スレッド内のメッセージはチャネルのメインタイムラインには表示されない
- スレッド開始者および参加者は新規スレッド返信時に通知を受け取る
- スレッド返信は「チャネルにも投稿」オプションで親チャネルへ二重投稿可能
- スレッド内のメッセージにもメンション・編集・削除・リアクションが可能

### 8.3 リアクション
- 絵文字リアクションを付与できる
- 1メッセージあたり最大23種類の絵文字をリアクションとして表示
- 同一ユーザーが同一メッセージに対して同じ絵文字を複数回リアクションすることはできない
- カスタム絵文字はワークスペース管理者がアップロードでき、全ユーザーが使用可能
- カスタム絵文字は1ファイル128KB以下、PNG / JPEG / GIF形式

## 9. メンション
- メッセージ内で `@ユーザー名` でユーザーをメンション
- メンションされたユーザーには通知が届く(通知設定に従う)
- `@channel`: チャネルの全メンバー
- `@here`: チャネルでオンラインの全メンバー
- `@channel`および`@here`の使用はチャネル管理者によって制限可能
- ユーザーグループ(`@team-frontend`等)を作成可能
  - オーナーまたは管理者が作成・所属メンバー管理
  - メンションはグループ内全員へのメンションとして扱われる

## 10. ファイル共有
- メッセージにファイル添付可能
- 1ファイルあたり最大100MB(Enterpriseプランは最大5GB)
- 1メッセージあたり最大10ファイル添付可能
- 対応形式: 画像(PNG/JPEG/GIF/WebP)、PDF、動画(MP4/MOV)、各種ドキュメント、ZIPアーカイブ
- 画像とPDFはインラインプレビュー表示
- アクセス権を持つチャネル参加者のみ閲覧可能
- 投稿者またはチャネル管理者によって削除可能
- 削除されたファイルは即座にダウンロード不可、メタデータは90日間保持
- ウイルススキャン: アップロード時に自動スキャン、検知時は隔離

## 11. ダイレクトメッセージ (DM)
- 同一ワークスペース内の任意のユーザーにDM送信可能
- DMは1対1の会話のみをサポート
- マルチパーティDM (MPDM): 最大8名までのグループDM、チャネルとは異なり名前を持たない一時的な会話
- DM内のメッセージはチャネルメッセージと同じ仕様
- DMは送信者と受信者のみが閲覧でき、ワークスペース管理者でも他人のDMは閲覧できない
- DMは検索の対象だが、本人参加のDMのみが検索結果に含まれる

## 12. 検索
- メッセージ本文・チャネル名・ユーザー名・ファイル名を横断検索
- メッセージ・チャネル・ユーザー・ファイルのカテゴリ別表示
- 本人がアクセス権を持つチャネルとDMに限定
- 関連度順またはタイムスタンプ降順で並び替え可能
- 1クエリあたり最大1000件まで返却、それ以上はページング

**絞り込みフィルタ**
- `from:ユーザー名` — 投稿者で絞り込み
- `in:チャネル名` — チャネルで絞り込み
- `before:YYYY-MM-DD` / `after:YYYY-MM-DD` — 期間で絞り込み
- `has:link` / `has:file` / `has:reaction` — 添付要素で絞り込み

## 13. 通知

**チャネル通知レベル(3種類)**
- すべてのメッセージ
- メンション時のみ
- なし

- DMは常にメンション扱いで通知される
- 通知レベルはチャネル単位、およびワークスペース全体のデフォルトとして設定可能
- ワークスペースのデフォルトはチャネル個別設定で上書きされる
- 「通知を一時停止」: 30分・1時間・2時間・翌朝9時まで全通知を抑止
- 通知抑止中もアプリ内バッジ表示は継続
- キーワード通知: 自身のキーワードリスト設定可能
- 通知方法: ブラウザ通知、メール、モバイルプッシュ
- 優先順位: ブラウザ未開封時はメール、5分以内に既読されない場合のみメールを送る

## 14. ピン留めとブックマーク
- チャネル管理者またはチャネルメンバーがチャネル内で「ピン留め」可能
- ピン留めはチャネルあたり最大100件
- サイドパネルから一覧アクセス可能
- ユーザーは個人ブックマークとして任意のメッセージを保存可能

## 15. 下書きと送信予約
- 入力中の内容は自動的に下書き保存
- 下書きはチャネル / DM / スレッド単位で保持
- アプリ再起動後も復元
- 送信予約: 最大120日先までの任意の日時に予約可能
- 予約済みメッセージは送信前に編集または取り消しが可能

## 16. 既読管理
- 各メッセージは閲覧者単位で既読/未読状態を保持
- 未読のあるチャネルが太字で表示
- 「未読をすべて既読にする」アクション(チャネル単位またはワークスペース単位)
- 既読位置(最終既読メッセージID)はサーバー保存、デバイス間で同期

## 17. ワークフロービルダー
- ノーコードのワークフロー作成機能
- トリガー、ステップ、アクションの組み合わせで定義
- チャネル単位またはワークスペース単位で公開可能
- 作成・編集はオーナーまたは管理者のみ、メンバーはリクエスト送信のみ
- 各ワークフローの実行ログは過去30日間保存
- 実行回数: 1ワークスペースあたり月間10000回まで(Pro)、無制限(Enterprise)

**トリガー / ステップ種別**
- トリガー: 絵文字リアクション、特定キーワードのメッセージ投稿、フォーム送信、スケジュール(cron形式)、Webhook受信
- ステップ: フォーム入力、条件分岐(if/else)、メッセージ送信、外部API呼び出し、待機(最大7日)

(SVG図: トリガー → ステップ → アクション)

## 18. AIアシスト機能
- 「会話サマリー」: チャネルやスレッドのメッセージ要約
- 会話サマリーは手動実行、または長時間オフラインだったユーザーに自動生成して提示
- スマートリプライ: 文脈に応じた返信候補を3件まで提示
- メンション通知に対するクイックリアクション(絵文字提案、定型返信提案)
- 自然言語検索クエリ対応(例: 「先週マーケティングがリリースした件」)
- AI生成内容には必ず「AI生成」のラベルが付与される
- AI機能はワークスペース単位でオン/オフ切替可能、デフォルトはオフ
- AI機能利用時はメッセージ内容が外部AIプロバイダに送信されることをUI上で明示
- ユーザーはAI機能のオプトアウトを個人設定で選択可能(オプトアウト時は本人のメッセージはAI処理されない)
- AI翻訳: チャネル / スレッド単位、原文と訳文を併記表示
- AIアクションアイテム抽出: スレッド・ハドル文字起こしから「やること」「決定事項」「期限」を構造化抽出
- AIプロバイダ選択(Enterprise): 自社契約のOpenAI / Anthropic / 自社オンプレLLMから選択可能
- AIガードレール: 機密情報(社外秘ラベル)が付与されたチャネルのメッセージはAI処理を強制スキップ
- AI利用統計は管理コンソールで確認可能、月次の使用量とコストを表示

## 19. インテグレーションと外部連携
- 連携は「アプリ」と「Webhook」の2種類
- アプリストアからインストール型のアプリを追加可能
- インストールはオーナーまたは管理者のみ(メンバーはリクエスト送信のみ)
- 受信Webhook: 外部からチャネルへ投稿するURLを発行
  - 受信Webhookごとにレートリミットあり(60秒あたり最大60リクエスト)
- 送信Webhook: 特定キーワードがチャネルに投稿時に外部URLへPOST送信
- スラッシュコマンド: `/コマンド名`でアプリの機能を呼び出し
- すべての連携はOAuth 2.0で認可
- アプリスコープ: 細粒度のスコープ(channels:read, chat:write, files:write など30種以上)
- アプリ設定の変更履歴は監査ログに記録
- アプリ提出ガイドライン: アプリストア掲載前にセキュリティ審査(内製アプリは免除)
- カスタムインテグレーション: 単一ワークスペース内で利用するものは管理者承認後に即時利用可能
- 連携先サービス例: GitHub、Jira、Asana、Notion、Google Drive、Zoom、Salesforce、PagerDuty、Datadog

## 20. API
- REST APIおよびリアルタイムイベント受信用WebSocket APIを提供
- API認証はBotトークンまたはユーザートークン
- レートリミット: 1ワークスペースあたり毎秒60リクエスト(バースト120)
- レートリミット超過時はHTTP 429、`Retry-After`ヘッダで待機時間通知
- API経由で投稿されたメッセージにはBotユーザーが投稿者として記録
- イベントAPI: 新規メッセージ・編集・削除・リアクション追加など30種類以上のイベントをWebSocketで配信
- API SDK: Python / JavaScript / Goの3言語で公式提供

## 21. 監査と履歴

**監査ログで確認できるイベント**
- ユーザーのログイン / ログアウト(IPアドレス・User-Agent含む)
- ロール変更
- チャネル作成 / アーカイブ / 復元
- ユーザー招待 / 退会 / 無効化
- アプリのインストール / アンインストール
- 設定変更(SSO・SCIM・通知デフォルト等)

**保持期間**
- Pro: 90日間
- Enterprise: 7年間(法令対応)

- 通常メンバーは自身の操作履歴のみ過去30日分閲覧可能
- 監査ログは管理者向けUI、CSVエクスポート、SIEMフォワーディング(Enterprise)で取得可能

## 22. データ保存とエクスポート
- データ保存リージョン: 米国・EU・アジア太平洋から選択可能(Enterprise)
- メッセージ保存期間ポリシーをワークスペース全体およびチャネル単位で設定可能
  - 例: 30日 / 90日 / 1年 / 無期限
  - 期限超過のメッセージは自動削除
- DMには独自の保存ポリシーを設定可能

**データエクスポート**
- **標準エクスポート**: パブリックチャネルのメッセージとファイルをZIP出力(オーナー限定)
- **コンプライアンスエクスポート**: プライベートチャネル・DMを含む全データ出力(Enterprise限定、法的要請時のみ)
- エクスポート完了時にオーナーへ通知メール、ダウンロードリンクは7日間有効

## 23. モバイルアプリ
- iOS (iOS 16以降) およびAndroid (Android 10以降) のネイティブアプリを提供
- 本書で定義する全機能のうちAPI経由で提供可能なものをすべてサポート
- モバイル独自機能: プッシュ通知、生体認証(Face ID / Touch ID / 指紋認証)、デバイスごとのセッション管理
- モバイルからのSSO認証はインアプリブラウザで行う
- バージョンが古すぎる場合(リリースから365日以上経過)は強制アップデート要求
- オフライン時もキャッシュメッセージは閲覧可能、復帰時に未送信メッセージを自動再送
- モバイル端末紛失時: 管理者または本人がリモートからセッション失効可能、失効と同時にローカルキャッシュも削除
- プッシュ通知ペイロードはメッセージ本文を含めず件名のみ送信(情報漏洩対策)
- MDM: Microsoft Intune および Jamf による配布・ポリシー強制をサポート
- バックグラウンド更新は省電力モード時に自動的に頻度を下げる(Android Doze、iOS Low Power Mode)

- 開発者向けドキュメント: docs.linkr.app で公開、APIリファレンス・SDK使用例・チュートリアル
- ベータプログラム: 早期アクセス可能、フィードバック義務あり

## 34. 利用規約とプライバシー
※原文では最後に登場します
- ワークスペース作成時に利用規約・プライバシーポリシーへの同意を要求
- ポリシー変更時は30日前に全ワークスペース管理者へメール通知し、再同意を要求
- ユーザーデータの利用範囲はサービス提供と統計分析に限定し、第三者への販売は行わない
- 統計分析データは匿名化された後に利用
- 自身のデータをJSONまたはCSVでいつでもエクスポート可能(個人エクスポート)
- アカウント削除時には個人データを30日以内に完全消去(バックアップからの削除は90日以内)
- Cookie: ファーストパーティCookieのみ、サードパーティトラッカー不使用
- マーケティング通信: 個別にオプトイン/アウト可能
- 未成年: 13歳未満は利用不可、13-17歳は保護者同意ワークフロー通過時のみ
- 法執行機関からの開示要求: 透明性レポートを年2回公開
- CSAM: NCMEC等への自動通報および即座のアカウント停止
- バイオメトリクスデータ: サーバー送信せずデバイス内処理
- データ常駐: 米国・EU・日本・シンガポールから選択可能(Enterprise)
- リテンションロック: 管理者でも削除不可に設定可能
- プライバシー強化機能: 「自分の名前で検索可能か」をユーザーが個別制御
- 外部共有制限: 共有チャネル機能の利用をワークスペース/チャネル単位で禁止可能
- 退職者データ: 「読み取り専用ユーザー」として保持または即時無効化を選択可能
- データ持ち出し検知: 短時間の大量メッセージダウンロード/検索を管理者へ自動通知
- 透明性レポート: 政府からの情報開示要求件数、自動コンテンツ削除件数を半年ごとに公開

## 26. セキュリティ
- 通信: TLS 1.2以上で暗号化
- 保存時暗号化: AES-256でメッセージ・ファイルを暗号化

- セッション管理: 全デバイスで同時ログイン可能、セッション一覧から個別/全セッションを取り消し可能
- IPアドレス制限: 許可IPレンジに制限可能(Enterprise)
- ペネトレーションテスト: 年2回外部ベンダー実施、結果サマリーをエンタープライズ顧客に開示
- バグバウンティ: HackerOne経由、深刻度に応じて報奨金支払い
- 異常検知: 短時間の大量メッセージ削除、想定外の地理的アクセスパターンを検知して管理者アラート
- パスワードポリシー: 過去5世代の再利用禁止、90日ごとの強制変更(Enterpriseで設定可能)
- DDoS対策: AWS Shield Advanced および Cloudflare のレイヤー7保護を二重化

## 28. 技術スタック

### 28.1 フロントエンド
- 言語: TypeScript 5.x
- フレームワーク: React 18以上
- 状態管理: Zustand
- ビルドツール: Vite
- ルーティング: React Router 6
- スタイリング: Tailwind CSS + shadcn/ui コンポーネント

### 28.2 バックエンド

- ORM/SQLジェネレータ: sqlc
- 認証: JWT(RS256、鍵ローテーションは月次)
- リアルタイム: WebSocket(RFC 6455)、サーバー間ファンアウトはRedis Pub/Sub
- 非同期処理: Amazon SQS + Lambda(通知配信、監査ログ、エクスポート)

### 28.3 データストア
- PostgreSQL 16: 主データ。Aurora PostgreSQL Serverless v2、リードレプリカ最低3台
- Redis 7: セッション、Pub/Sub、レートリミット、ホットキャッシュ。ElastiCache for Redis(クラスタモード有効)
- Elasticsearch 8: 全文検索。OpenSearch Service
- S3: ファイル添付、署名付きURLでアクセス制御、ライフサイクルポリシーで保存期間ポリシー実装
- Glacier: 長期保存データ(Enterprise保存期間ポリシー)

### 28.4 インフラ
- クラウド: AWS、リージョンは契約に応じて選択(米国・EU・APAC)
- IaC: Terraform(terragruntでマルチ環境管理)
- コンテナオーケストレーション: ECS Fargate
- CDN: CloudFront
- WAF: AWS WAF + Cloudflareの二段構成
- DNS: Route 53
- 秘匿情報管理: AWS Secrets Manager + 顧客管理鍵(CMK)

### 28.5 CI/CD と開発フロー
- バージョン管理: GitHub
- CI/CD: GitHub Actions(PR毎にlint・test・E2E、mainマージ時にBlue/Greenデプロイ)
- ブランチ戦略: トランクベース開発 + Feature Flag
- フィーチャーフラグ: LaunchDarkly
- 依存脆弱性スキャン: Snyk + Dependabot
- コードレビュー: 全PRに最低2名レビュー必須、CODEOWNERSで自動アサイン
- リリース頻度: 週2回(火・木)、ホットフィックスは随時

### 28.6 監視と運用
- APM: Datadog
- ログ集約: Datadog Logs
- アラート: PagerDuty、オンコールローテーションは1週間交代
- SLO: メッセージ送信成功率99.95%、APIレスポンスp95 500ms以内
- インシデント対応: PagerDuty呼び出しから5分以内の一次応答、30分以内のステータスページ更新
- カオスエンジニアリング: 月1回 Gremlin で本番に近いステージング環境で障害注入
- パフォーマンステスト: 主要エンドポイントの k6 負荷テストをCIで毎晩実行
- リリースバジェット(Error Budget): SLO違反時はリリース自動停止、根本原因分析完了まで再開不可
- 設定変更履歴: Terraform Stateの差分をGitに記録、本番環境変更は2名承認制
- DBマイグレーション: pg-online-schema-changeによるロックフリーマイグレーション
- セキュリティパッチ: クリティカル24時間以内、ハイ7日以内、ミディアム30日以内に適用
- バックアップリストア演習: 四半期に1回、別環境で本番バックアップから復旧リハーサル
- カスタマーアドバイザリ: 重大インシデント発生時は1時間以内にステータスページ更新、24時間以内に詳細レポート公開

## 29. 音声・ビデオハドル
- チャネルおよびDM内で音声・ビデオハドル(短時間ミーティング)を開始可能
- 参加可能人数: 最大25名(Pro)/ 50名(Enterprise)
- 開始時にチャネル内に通知表示、ワンクリックで参加
- 画面共有: 全画面・特定ウィンドウ・特定ブラウザタブから選択可能
- 録画はオーナー権限を持つユーザーのみが開始可能
- 録画データは投稿チャネルにファイル自動添付、文字起こし付与
- 録画は保存ポリシーに従い、デフォルト90日間保管
- 音声品質指標(パケロス・ビットレート・遅延)はリアルタイムでホストに表示
- バックグラウンドノイズ除去はデフォルト有効、ユーザー設定で無効化可能

## 30. カスタムフォーム
- ワークフロー内またはメッセージから独立したフォームを作成可能
- フィールド種別: テキスト(短文/長文)、数値、選択(単一/複数)、日付、日時、ファイルアップロード、ユーザー選択、チャネル選択
- 必須項目・バリデーションルール(正規表現・最小最大値)を設定可能
- 回答結果は作成者または指定された通知先チャネルに投稿
- 回答はCSVエクスポート可能(フォーム作成者のみ)
- 公開リンク経由でワークスペース外のユーザーからも回答可能(Enterprise)
- 匿名回答オプション設定可能、匿名時は回答者情報をエクスポートに含めない

## 31. コンプライアンスとガバナンス
- リーガルホールド: 特定ユーザー/チャネルのデータを保存ポリシーに関わらず削除しない設定(Enterprise)
- リーガルホールドはコンプライアンス管理者ロールのみが設定可能、設定中は本人にも通知されない
- eDiscovery: 特定キーワード・期間・ユーザーで絞り込み検索可能
- データ主権: GDPR・CCPA・APPI等の各国法令対応
- 「忘れられる権利」リクエスト: 30日以内に該当データの匿名化または削除
- 監査ログのSIEMフォワーディング: Splunk・Datadog・QRadarへリアルタイム配信(Enterprise)
- 準拠: ISO 27001、SOC 2 Type II、ISMAP、HIPAA(Enterpriseの一部プランのみ)
- データ処理委託先一覧(Sub-processors)を公開ページで開示、変更時は事前通知
- カスタマー鍵管理(CKM): Enterprise契約者は自社KMSで暗号化鍵を管理可能
- カスタマーアクセスログ機能を提供(Enterprise)

## 32. オンボーディングと管理ツール
- 新規ワークスペース作成時にセットアップウィザード起動、初期チャネル・招待・通知設定をガイド
- 新規メンバー初回ログイン時に個人プロフィール設定・通知設定・チャネル参加のチュートリアル表示
- 管理コンソールから全ユーザーのアクティビティ統計(DAU/MAU・メッセージ数・チャネル別参加率)を確認可能
- 管理者は特定ユーザーへ「お試しチャネル」「会社からのお知らせ」等の必須参加チャネルを強制設定可能
- セキュリティアラート: 異常ログイン(新デバイス・異常な地域)を検知して本人と管理者へ通知
- 健全性チェック: 管理者向けに「使われていないチャネル」「長期間ログインしていないユーザー」を週次レポート
