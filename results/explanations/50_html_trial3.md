- ドキュメントタイトル: 「チャットツール「Linkr」要件定義書 (Core)」
- HTML 言語属性: `lang="ja"`
- 文字コード: `UTF-8`

- スタイルシート: `./styles.css` を外部参照

## 1. 概要

- 社内コミュニケーション用のチャットツールである。
- 本書はコア機能のみを定義する。

---

## 2. ユーザーとワークスペース

機能要件 (`data-category="functional"`):

- ユーザーは1つのメールアドレスにつき1アカウントを保持する。
- 各ユーザーは複数のワークスペースに所属できる。
- ワークスペースはチームの単位であり、URL サブドメインで識別される (例: `acme.linkr.app`)。
- ワークスペース内のユーザーは表示名とアバター画像を設定できる。

図(SVG ダイアグラム): 「ユーザー・ワークスペース・サブドメインの関係図」
- 図のタイトル(title要素): 「ユーザーは複数ワークスペースに所属し、ワークスペースはサブドメインで識別される」
- figcaption: 「ユーザーと複数ワークスペースの所属関係 (サブドメインで識別)」
- 図の構成要素: 「ユーザー」ノード、「ワークスペースA」、「ワークスペースB」、「acme.linkr.app」、「サブドメイン」ノードを線で接続

---

## 3. チャネル

機能要件 (チャネル全般):

- チャネルはワークスペース内のトピック単位の会話空間である。
- チャネルにはパブリックとプライベートの2種類がある。

### 3.1 チャネル種別

表「チャネル種別と閲覧・参加範囲」:

| 種別 | 閲覧・参加範囲 |
|---|---|
| パブリック | ワークスペース全員が閲覧・参加可能 |
| プライベート | 招待されたメンバーのみ閲覧可能 |

### 3.2 チャネル運用ルール

機能要件:
- チャネル名は半角英数字とハイフンのみ、最大80文字。
- チャネル作成者は自動的にメンバーとなり、初期管理権限を持つ。
- ユーザーは任意のパブリックチャネルに自由参加できる。
- ユーザーは自分が参加中のチャネルから退出できる。

警告事項 (`data-category="warning"`、退出ルールに対する例外):
- ただし最後の管理者は退出前に他メンバーに管理権限を委譲する必要がある。

---

## 4. メッセージ

機能要件:

- ユーザーはチャネル内にテキストメッセージを投稿できる。
- メッセージは最大4000文字。
- メッセージには投稿時刻と投稿者が記録される。
- メッセージは投稿後も時系列で永続表示される。
- メッセージ本文では Markdown 記法 (太字、斜体、コードブロック、リンク) が利用可能。

---

## 5. ダイレクトメッセージ (DM)

機能要件:

- ユーザーは同一ワークスペース内の任意のユーザーに DM を送信できる。
- DM は1対1の会話のみをサポートする (グループ DM は本書のスコープ外)。
- DM 内のメッセージはチャネルメッセージと同じ仕様 (4000文字上限・Markdown 対応)。

セキュリティ要件 (`data-category="security"`):

- DM は送信者と受信者のみが閲覧でき、ワークスペース管理者であっても他人の DM は閲覧できない。

---

## 6. 非機能要件

表「非機能要件一覧」(`data-category="nfr"`):

| 項目 | 要件 |
|---|---|
| 同時接続ユーザー数 | 1ワークスペースあたり最大500 |
| メッセージ送信レイテンシ | 送信から表示まで95%tile で500ms 以内 |
| データ永続性 | メッセージは無期限保存 (削除ポリシーは別途定義) |
| 対応ブラウザ | 最新版の Chrome / Safari / Firefox / Edge |

---

## 7. 技術スタック制約

定義リスト (`<dl>`、`data-category="tech"`):

- **フロントエンド**: TypeScript + React (バージョン18以上)
- **バックエンド**: Go (1.22以上)、HTTP フレームワークは chi
- **データストア**: PostgreSQL 16 (主データ) + Redis 7 (セッション・Pub/Sub)
- **リアルタイム通信**: WebSocket (RFC 6455準拠)
- **インフラ**: AWS 東京リージョン (ap-northeast-1) で稼働、IaC は Terraform で管理

## 付帯情報 (styles.css に明記されている内容)

styles.css の冒頭コメント: 「Linkr 要件定義書 共通スタイル」。

スタイル上、要件カテゴリは `data-category` 属性によるバッジ着色で区別される設計になっており、定義済みカテゴリは:
- `security` (赤系)
- `tech` (紫系)
- `nfr` (青緑系)
- `warning` (橙系)

その他、styles.css には `.cards` / `.card` (役割・プラン比較カード用) や `details/summary`、`aside` 等のスタイル定義もあるが、`requirements.html` 本体ではこれらの要素は使用されていない。
