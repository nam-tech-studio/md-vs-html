## 1. 概要
- 社内コミュニケーション用のチャットツール
- 本書はコア機能、コミュニケーション支援機能、拡張機能、エンタープライズ向けの管理・統合・コンプライアンス機能を定義する

## 2. ユーザーとアカウント
- 1メールアドレスにつき1アカウント
- 各ユーザーは複数のワークスペースに所属可能
- パスワード認証またはSSOで認証
- パスワード要件: 最低12文字、英大文字・小文字・数字・記号をそれぞれ1文字以上含む
- パスワードリセットは登録メールへのトークン送信、トークン有効期限60分
- 連続5回のログイン失敗で15分間ロック
- プレゼンス3種: オンライン / 退席中 / オフライン
- 10分間操作なしで自動的に「退席中」
- ステータスメッセージ: 絵文字＋最大100文字、有効期限選択肢は1時間 / 4時間 / 終日 / カスタム / 無期限
- 2要素認証: TOTP / SMS / WebAuthn
- 2要素認証はワークスペースオーナーが組織全体に強制可能

## 3. ワークスペース
- チームの単位、URLサブドメインで識別 (例: `acme.linkr.app`)
- 表示名・アバター画像を設定可能
- ワークスペース名・ロゴ・アイコンはオーナーが変更可能
- 契約プランが紐付き、プランで機能制限が異なる
- 削除実行から30日間は復元可能 (オーナーのみ)
- 30日経過後はメッセージ・ファイル・設定すべて完全消去

**契約プラン比較**
- **Free**: ユーザー50名まで、メッセージ履歴は直近10000件まで閲覧可能
- **Pro**: ユーザー無制限、メッセージ履歴無制限
- **Enterprise**: SSO、SCIM、データ常駐リージョン指定、エンタープライズキー管理、監査ログ拡張

## 4. ロールと権限
- **オーナー**: 全権限。請求設定変更、削除、他オーナー任命可
- **管理者**: ユーザー招待、チャネル管理、ロール付与（オーナー以外）
- **メンバー**: 通常利用権限のみ
- ロール変更はオーナーまたは管理者（オーナーへの昇格はオーナーのみ）
- 常に1名以上のオーナーが必要
- 退会するオーナーは事前にオーナー権限委譲が必要

**ゲストユーザー**
- 特定チャネルのみアクセス可能な制限ロール
- DM・検索・他チャネル閲覧不可
- オーナーまたは管理者のみが招待可能
- (図): オーナー > 管理者 > メンバー、ゲストは別ロールで特定チャネルのみ

## 5. 招待とユーザー管理
- 管理者・オーナーがメールで招待
- 招待リンク有効期限: 7日間
- 一度に最大100件まとめて招待可能
- ドメイン制限: 許可リストでメールドメインを制限可能
- ユーザーは無効化または完全削除が可能
  - **無効化**: ログイン不可だが過去メッセージ・名前は残る
  - **完全削除**: 投稿は「削除済みユーザー」として残るがアカウント情報は完全消去
- SCIM v2.0によるユーザープロビジョニング (Enterpriseプランのみ)

## 6. SSO
- Enterpriseプランでワークスペース全体で有効化可能
- 対応プロトコル: SAML 2.0、OIDC
- SSO有効時はパスワード認証を無効化可能 (オーナー設定)
- SSO属性 (名前、メール、ユーザーグループ) は自動反映
- JIT (ジャストインタイム) プロビジョニング: SSO初回ログイン時に自動アカウント作成可能
- SCIMと併用時はユーザー作成・無効化はSCIM側を優先

## 7. チャネル
- ワークスペース内のトピック単位の会話空間
- チャネル名: 半角英数字とハイフンのみ、最大80文字
- 作成者は自動メンバー、初期管理権限を持つ
- パブリックチャネルへは自由参加可能
- 退出可能、ただし最後の管理者は管理権限委譲が必要
- トピック最大250文字、説明最大1500文字

**種別**
- **パブリック**: ワークスペース全員が閲覧・参加可
- **プライベート**: 招待されたメンバーのみ閲覧可

**アーカイブ**
- アーカイブ可能、新規投稿不可だが閲覧・検索は可能
- チャネル管理者またはワークスペース管理者が実行
- 復元はワークスペース管理者が可能

**共有チャネル (Connect)**
- 他ワークスペースとの共有可 (Enterpriseプラン)
- 共有先参加者は自身のアカウントで投稿
- どちらのワークスペースの保存期間ポリシーを適用するか明示的に選択可能

## 8. メッセージ
- テキストメッセージ、最大4000文字
- 投稿時刻と投稿者を記録
- 時系列で永続表示
- Markdown記法可（太字、斜体、コードブロック、リンク、引用、リスト）

### 8.1 編集と削除
- 自分のメッセージは編集可
- 編集後は「編集済み」表示
- 編集履歴は保存しないが、編集日時は記録
- 自分のメッセージは削除可
- チャネル管理者は他人のメッセージを削除可（編集不可）
- 削除メッセージは「このメッセージは削除されました」プレースホルダ表示
- ソフトデリート、削除ログは90日間保管

### 8.2 スレッド
- 任意メッセージへの返信でスレッド開始
- スレッド内メッセージはメインタイムラインに表示されない
- スレッド開始者・参加者は新規返信時に通知
- 「チャネルにも投稿」オプションで二重投稿可
- スレッド内でもメンション・編集・削除・リアクション可

### 8.3 リアクション
- 絵文字リアクション付与可
- 1メッセージあたり最大23種類の絵文字を表示
- 同一ユーザーが同一メッセージに同じ絵文字を複数回付与不可
- カスタム絵文字: ワークスペース管理者がアップロード、全ユーザーが使用可
- カスタム絵文字: 1ファイル128KB以下、PNG / JPEG / GIF

## 9. メンション
- `@ユーザー名` でメンション
- メンションされたユーザーへ通知（通知設定に従う）
- `@channel`: 同一チャネル全メンバー
- `@here`: 同一チャネルのオンライン全メンバー
- `@channel`/`@here`の使用はチャネル管理者が制限可能
- ユーザーグループ (`@team-frontend`等) 作成可
  - オーナー/管理者が作成・所属管理
  - メンションはグループ内全員へのメンション扱い

## 10. ファイル共有
- メッセージにファイル添付可
- 1ファイル最大100MB（Enterpriseは最大5GB）
- 1メッセージあたり最大10ファイル
- 対応形式: 画像 (PNG/JPEG/GIF/WebP)、PDF、動画 (MP4/MOV)、各種ドキュメント、ZIP
- 画像とPDFはインラインプレビュー
- アクセス権を持つチャネル参加者のみ閲覧可
- 投稿者またはチャネル管理者が削除可
- 削除後は即座にダウンロード不可、メタデータは90日間保持
- ウイルススキャン: アップロード時に自動スキャン、検知時は隔離

## 11. ダイレクトメッセージ (DM)
- 同一ワークスペース内ユーザーへDM送信可
- DMは1対1
- マルチパーティDM (MPDM): 最大8名、名前を持たない一時的グループDM
- 仕様はチャネルメッセージと同じ
- 送信者・受信者のみ閲覧、ワークスペース管理者でも他人のDMは閲覧不可
- 検索対象だが、本人参加のDMのみ検索結果に含まれる

## 12. 検索
- メッセージ本文・チャネル名・ユーザー名・ファイル名を横断検索
- カテゴリ別表示（メッセージ・チャネル・ユーザー・ファイル）
- 本人がアクセス権を持つチャネルとDMに限定
- 関連度順またはタイムスタンプ降順の並び替え
- 1クエリ最大1000件、超過分はページング

**絞り込みフィルタ**
- `from:ユーザー名`: 投稿者
- `in:チャネル名`: チャネル
- `before:YYYY-MM-DD` / `after:YYYY-MM-DD`: 期間
- `has:link` / `has:file` / `has:reaction`: 添付要素

## 13. 通知
**チャネル通知レベル3種**
- すべてのメッセージ
- メンション時のみ
- なし

- DMは常にメンション扱いで通知
- チャネル単位、およびワークスペース全体のデフォルト設定可
- ワークスペースのデフォルトはチャネル個別設定で上書き
- 「通知を一時停止」: 30分・1時間・2時間・翌朝9時まで全通知抑止
- 抑止中もアプリ内バッジ表示は継続
- キーワード通知: 個人キーワードリスト設定可
- 通知方法: ブラウザ通知、メール、モバイルプッシュ
- 優先順位: ブラウザ未開封時はメール、5分以内に既読されない場合のみメール送信

## 14. ピン留めとブックマーク
- チャネル管理者またはチャネルメンバーがメッセージをピン留め可
- チャネルあたり最大100件
- サイドパネルから一覧アクセス
- 個人ブックマークとして任意メッセージ保存可

## 15. 下書きと送信予約
- 入力内容は自動的に下書き保存
- チャネル / DM / スレッド単位で保持
- アプリ再起動後も復元
- 送信予約: 最大120日先まで
- 予約済みメッセージは送信前に編集・取り消し可

## 16. 既読管理
- 閲覧者単位で既読/未読保持
- 未読のあるチャネルは太字表示
- 「未読をすべて既読にする」アクション（チャネル単位 / ワークスペース単位）
- 既読位置（最終既読メッセージID）はサーバー保存、デバイス間同期

## 17. ワークフロービルダー
- ノーコードのワークフロー作成
- トリガー、ステップ、アクションの組み合わせ
- チャネル単位またはワークスペース単位で公開可
- 作成・編集はオーナー/管理者のみ、メンバーはリクエスト送信のみ
- 実行ログは過去30日間保存
- 実行回数: 月間10000回まで (Pro)、無制限 (Enterprise)

**トリガー / ステップ種別**
- トリガー: 絵文字リアクション、特定キーワードのメッセージ投稿、フォーム送信、スケジュール (cron形式)、Webhook受信
- ステップ: フォーム入力、条件分岐 (if/else)、メッセージ送信、外部API呼び出し、待機 (最大7日)
- (図): トリガー → ステップ → アクション

## 18. AIアシスト機能
- 「会話サマリー」（チャネル・スレッドの要約）
- 手動実行、または長時間オフラインだったユーザーに自動生成・提示
- スマートリプライ: 返信候補3件まで提示
- メンション通知へのクイックリアクション（絵文字提案、定型返信提案）
- 検索クエリの自然言語入力対応（例: 「先週マーケティングがリリースした件」）
- AI生成内容には「AI生成」ラベル必須
- ワークスペース単位でオン/オフ切替可、デフォルトはオフ
- AI機能利用時のメッセージが外部AIプロバイダに送信されることをUIで明示
- 個人設定でオプトアウト可（オプトアウト時は本人メッセージはAI処理されない）
- AI翻訳: チャネル/スレッド単位、原文と訳文を併記
- AIアクションアイテム抽出: 「やること」「決定事項」「期限」を構造化抽出
- AIプロバイダ選択 (Enterprise): 自社契約のOpenAI / Anthropic / 自社オンプレLLMから選択可
- AIガードレール: 機密情報（社外秘ラベル）チャネルのメッセージはAI処理を強制スキップ
- AI利用統計: 管理コンソールで月次の使用量・コスト確認可

## 19. インテグレーションと外部連携
- 「アプリ」と「Webhook」2種
- アプリストアからインストール型アプリ追加可
- アプリインストールはオーナー/管理者のみ、メンバーはリクエストのみ
- 受信Webhook: 外部からチャネルへ投稿するURL発行可、レートリミット60秒あたり最大60リクエスト
- 送信Webhook: 特定キーワード投稿時に外部URLへPOST
- スラッシュコマンド: `/コマンド名` で呼び出し
- 連携はOAuth 2.0で認可
- アプリスコープ: 細粒度スコープ (channels:read, chat:write, files:write など30種以上)
- アプリ設定変更履歴は監査ログに記録
- アプリ提出ガイドライン: 掲載前にセキュリティ審査（内製アプリは免除）
- カスタムインテグレーション: 単一ワークスペース内利用は管理者承認後に即時利用可
- 連携先サービス例: GitHub、Jira、Asana、Notion、Google Drive、Zoom、Salesforce、PagerDuty、Datadog

## 20. API
- REST APIおよびWebSocket API
- 認証: Botトークンまたはユーザートークン
- レートリミット: 1ワークスペースあたり毎秒60リクエスト（バースト120）
- 超過時はHTTP 429、`Retry-After`ヘッダで待機時間通知
- API投稿はBotユーザーが投稿者として記録
- イベントAPI: 新規メッセージ・編集・削除・リアクション追加など30種類以上のイベント
- 公式SDK: Python / JavaScript / Go の3言語

## 21. 監査と履歴
**監査対象イベント**
- ログイン/ログアウト（IPアドレス・User-Agent含む）
- ロール変更
- チャネル作成/アーカイブ/復元
- ユーザー招待/退会/無効化
- アプリのインストール/アンインストール
- 設定変更（SSO・SCIM・通知デフォルト等）

**保持期間**
- Pro: 90日間
- Enterprise: 7年間（法令対応）

- 通常メンバーは自身の操作履歴のみ過去30日分閲覧可
- 取得手段: 管理者向けUI、CSVエクスポート、SIEMフォワーディング (Enterprise)

## 22. データ保存とエクスポート
- データ保存リージョン: 米国・EU・アジア太平洋から選択 (Enterprise)
- メッセージ保存期間ポリシー: ワークスペース全体およびチャネル単位で設定可
  - 例: 30日 / 90日 / 1年 / 無期限
  - 期限超過は自動削除
- DMには独自保存ポリシー設定可

**エクスポート**
- 標準エクスポート: パブリックチャネルのメッセージとファイルをZIPで出力 (オーナー限定)
- コンプライアンスエクスポート: プライベートチャネル・DM含む全データ (Enterprise限定、法的要請時のみ)
- 完了時にオーナーへ通知メール、ダウンロードリンクは7日間有効

## 23. モバイルアプリ
- iOS (iOS 16以降) / Android (Android 10以降) ネイティブアプリ
- API経由で提供可能な全機能をサポート
- モバイル独自機能: プッシュ通知、生体認証 (Face ID / Touch ID / 指紋認証)、デバイスごとのセッション管理
- モバイルからのSSOはインアプリブラウザ
- リリースから365日以上経過した古いバージョンは強制アップデート
- オフライン時もキャッシュ閲覧可、復帰時に未送信メッセージ自動再送
- 紛失時: 管理者または本人がリモートからセッション失効可、失効と同時にローカルキャッシュ削除
- プッシュ通知ペイロードはメッセージ本文を含めず件名のみ（情報漏洩対策）
- MDM: Microsoft Intune および Jamf による配布・ポリシー強制
- バックグラウンド更新は省電力モード (Android Doze、iOS Low Power Mode) で自動的に頻度低下

## 33. ヘルプとサポート
- アプリ内ヘルプセンター: 機能別ヘルプ記事をワークスペース言語で表示
- 右下のヘルプアイコンから検索
- お問い合わせフォームからサポートチケット起票可
- サポート対応時間: Pro 平日9-18時 (日本時間)、Enterprise 24/7
- ステータスページ: 全ユーザーに公開、過去90日のインシデント掲載
- フィードバック機能: 機能要望投稿、Vote付き
- リリースノート: アプリ内通知バーで告知、詳細は公式ブログ
- 開発者ドキュメント: docs.linkr.app（APIリファレンス、SDK使用例、チュートリアル）
- ベータプログラム: 早期アクセス、フィードバック義務あり

## 34. 利用規約とプライバシー
- ワークスペース作成時に利用規約・プライバシーポリシーへの同意要求
- ポリシー変更時は30日前にメール通知し再同意要求
- データ利用範囲: サービス提供と統計分析に限定、第三者販売なし
- 統計分析データは匿名化後に利用
- 個人エクスポート: JSONまたはCSV形式でユーザー自身がいつでも可
- アカウント削除時: 個人データを30日以内に完全消去（バックアップからは90日以内）
- Cookie: ファーストパーティCookieのみ、サードパーティトラッカーなし
- マーケティング通信: 個別オプトイン/アウト
- 13歳未満は利用不可、13-17歳は保護者同意ワークフロー必須
- 透明性レポート: 法執行機関からの開示要求件数と種別を年2回公開
- CSAM: NCMEC等への自動通報および即座のアカウント停止
- バイオメトリクスデータ: サーバー送信せずデバイス内処理
- データ常駐: 米国・EU・日本・シンガポールから選択 (Enterprise)
- リテンションロック: 削除を防ぐべきデータは管理者でも削除不可に設定可
- プライバシー強化: 「自分の名前で検索可能か」を個別制御
- 外部共有制限: 共有チャネル機能をワークスペース/チャネル単位で禁止可
- 退職者データ: 「読み取り専用ユーザー」保持または即時無効化を選択
- データ持ち出し検知: 短時間に大量ダウンロード/検索発生時は管理者へ自動通知
- 透明性レポート: 政府開示要求件数、自動コンテンツ削除件数等を半年ごとに公開

## 24. アクセシビリティ
- WCAG 2.1 Level AA 準拠
- スクリーンリーダー対応: VoiceOver / NVDA / JAWS
- キーボードのみで全機能アクセス可
- カラーコントラスト比 4.5:1 以上（通常テキスト）
- フォーカスインジケータは常に可視

## 25. 国際化
- UI翻訳7言語: 日本語、英語、簡体字中国語、韓国語、ドイツ語、フランス語、スペイン語
- ユーザーごとに表示言語選択可
- ワークスペース管理者がデフォルト言語設定可
- メッセージ本文は翻訳されない（ユーザー入力）
- 日付・時刻はユーザータイムゾーンに従う
- 文字コード: UTF-8

## 26. セキュリティ
- 通信: TLS 1.2以上で暗号化
- 保存時暗号化: AES-256
- EKM (エンタープライズキー管理): CMK (顧客管理鍵) によるキーラップ (Enterprise)
- DLP (データ漏洩防止): カスタムパターン (クレジットカード番号等) を検出、ブロックまたは警告
- セッション管理: 全デバイス同時ログイン可、セッション一覧から個別/全セッション取り消し可
- IPアドレス制限: 許可IPレンジで制限可 (Enterprise)
- ペネトレーションテスト: 年2回外部ベンダー、結果サマリーをエンタープライズ顧客に開示
- バグバウンティ: HackerOne経由で受付、深刻度に応じて報奨金
- 異常検知: 短時間の大量メッセージ削除、想定外の地理的アクセスを検知し管理者アラート
- パスワードポリシー: 過去5世代再利用禁止、90日ごとの強制変更 (Enterpriseで設定可能)
- DDoS対策: AWS Shield Advanced + Cloudflare レイヤー7保護を二重化

## 27. 非機能要件
- 同時接続ユーザー数: 1ワークスペースあたり最大10000 (Enterprise)
- メッセージ送信レイテンシ: 95パーセンタイルで500ms以内
- 検索応答時間: 95パーセンタイルで1秒以内 (1000万メッセージ規模)
- ファイルアップロード: 100MBファイルが30秒以内 (回線100Mbps想定)
- 可用性: SLA 99.9% (Pro) / 99.99% (Enterprise)
- データ永続性: 年間データ損失率 < 0.0001%
- バックアップ: 毎日、地理的に分離した3拠点保管 (RPO 24時間 / RTO 4時間)
- 対応ブラウザ: 最新版のChrome / Safari / Firefox / Edge

### 28.5 CI/CD と開発フロー

### 28.6 監視と運用
- APM: Datadog
- ログ集約: Datadog Logs
- アラート: PagerDuty連携、オンコールローテーションは1週間交代
- SLO: メッセージ送信成功率99.95%、APIレスポンスp95 500ms以内

- 設定変更履歴: Terraform State 差分を Git に記録、本番変更は2名承認制
- DBマイグレーション: pg-online-schema-change によるロックフリー
- セキュリティパッチ: クリティカル24時間以内、ハイ7日以内、ミディアム30日以内
- バックアップリストア演習: 四半期に1回、別環境で復旧リハーサル
- カスタマーアドバイザリ: 重大インシデント発生時1時間以内にステータスページ更新、24時間以内に詳細レポート公開

## 29. 音声・ビデオハドル
- チャネル・DM内で開始可能（短時間ミーティング）
- 参加上限: Pro 25名 / Enterprise 50名
- ハドル開始時にチャネル内通知、ワンクリックで参加
- 画面共有: 全画面・特定ウィンドウ・特定ブラウザタブから選択

- 録画はデフォルト90日間保管

## 30. カスタムフォーム
- ワークフロー内またはメッセージから独立したフォーム作成可
- フィールド種別: テキスト (短文/長文)、数値、選択 (単一/複数)、日付、日時、ファイルアップロード、ユーザー選択、チャネル選択
- 必須項目・バリデーションルール (正規表現・最小最大値) 設定可
- 回答結果は作成者または通知先チャネルに投稿
- 回答はCSVエクスポート可（フォーム作成者のみ）
- 公開リンク経由でワークスペース外からも回答可 (Enterprise)
- 匿名回答オプション、匿名時は回答者情報をエクスポート除外

## 31. コンプライアンスとガバナンス
- リーガルホールド: 保存ポリシーに関わらず削除しない (Enterprise)
- リーガルホールドはコンプライアンス管理者ロールのみ設定可、本人にも通知されない
- eDiscovery: コンプライアンスエクスポートに加え、キーワード・期間・ユーザーで絞り込み検索
- データ主権: GDPR・CCPA・APPI等に対応するアクセス・削除リクエストフロー
- 「忘れられる権利」: 要求から30日以内に匿名化または削除
- 監査ログSIEMフォワーディング: Splunk・Datadog・QRadar (Enterprise)
- 準拠認証: ISO 27001、SOC 2 Type II、ISMAP、HIPAA (Enterpriseの一部プランのみ)
- データ処理委託先一覧 (Sub-processors): 公開ページで開示、変更時は事前通知
- カスタマー鍵管理 (CKM): Enterprise契約者は自社KMSで鍵管理可
- カスタマーアクセスログ: 顧客自身が取得可 (Enterprise)

## 32. オンボーディングと管理ツール
- 新規ワークスペース作成時にセットアップウィザード（初期チャネル・招待・通知設定をガイド）
- 新規メンバー初回ログイン時に個人プロフィール・通知設定・チャネル参加のチュートリアル
- 管理コンソールから全ユーザーアクティビティ統計 (DAU/MAU・メッセージ数・チャネル別参加率) 確認可
- 必須参加チャネル（「お試しチャネル」「会社からのお知らせ」等）を強制設定可
- セキュリティアラート: 異常なログイン (新デバイス・異常な地域) を検知し本人と管理者に通知
- 健全性チェック: 「使われていないチャネル」「長期間ログインしていないユーザー」を週次レポート
