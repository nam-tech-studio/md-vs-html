## タイトル
- 文書名: **チャットツール「Linkr」要件定義書 (Core)**

## 1. 概要
- 社内コミュニケーション用のチャットツールである
- 本書は**コア機能のみ**を定義する

## 2. ユーザーとワークスペース
- ユーザーは1つのメールアドレスにつき**1アカウント**を保持する
- 各ユーザーは**複数のワークスペースに所属できる**
- ワークスペースは**チームの単位**であり、**URLサブドメインで識別される**
  - 例: `acme.linkr.app`
- ワークスペース内のユーザーは**表示名**と**アバター画像**を設定できる

## 3. チャネル
- チャネルはワークスペース内の**トピック単位の会話空間**である
- チャネルには**パブリック**と**プライベート**の2種類がある
  - **パブリック**: ワークスペース全員が閲覧・参加可能
  - **プライベート**: 招待されたメンバーのみ閲覧可能
- チャネル名は**半角英数字とハイフンのみ**、**最大80文字**
- チャネル作成者は**自動的にメンバー**となり、**初期管理権限**を持つ
- ユーザーは任意のパブリックチャネルに**自由参加できる**
- ユーザーは自分が参加中のチャネルから**退出できる**
  - ただし**最後の管理者**は退出前に**他メンバーに管理権限を委譲する必要がある**

## 4. メッセージ
- ユーザーはチャネル内に**テキストメッセージを投稿できる**
- メッセージは**最大4000文字**
- メッセージには**投稿時刻**と**投稿者**が記録される
- メッセージは投稿後も**時系列で永続表示される**
- メッセージ本文では**Markdown記法**が利用可能
  - 利用可能な要素: **太字、斜体、コードブロック、リンク**

## 5. ダイレクトメッセージ (DM)
- ユーザーは**同一ワークスペース内**の任意のユーザーにDMを送信できる
- DMは**1対1の会話のみ**をサポートする
  - **グループDMは本書のスコープ外**
- DM内のメッセージはチャネルメッセージと同じ仕様
  - **4000文字上限**
  - **Markdown対応**
- DMは**送信者と受信者のみ**が閲覧できる
  - **ワークスペース管理者であっても他人のDMは閲覧できない**

## 6. 非機能要件
- **同時接続ユーザー数**: 1ワークスペースあたり**最大500**
- **メッセージ送信レイテンシ**: 送信から表示まで**95%tileで500ms以内**
- **データ永続性**: メッセージは**無期限保存**
  - 削除ポリシーは**別途定義**
- **対応ブラウザ**: 最新版の **Chrome / Safari / Firefox / Edge**

## 7. 技術スタック制約
- **フロントエンド**: TypeScript + React（**バージョン18以上**）
- **バックエンド**: Go（**1.22以上**）、HTTPフレームワークは **chi**
- **データストア**:
  - PostgreSQL 16（**主データ**）
  - Redis 7（**セッション・Pub/Sub**）
- **リアルタイム通信**: WebSocket（**RFC 6455準拠**）
- **インフラ**:
  - **AWS 東京リージョン (ap-northeast-1)** で稼働
  - **IaC は Terraform** で管理

## ビジネスルール／エッジケース（原文記載分）
- 1メールアドレス = 1アカウント（重複アカウント不可）
- チャネル名のバリデーション: 半角英数字とハイフンのみ／最大80文字
- 退出時の管理者委譲ルール: 最後の管理者は委譲なしには退出できない
- DMの閲覧権限: 管理者であっても他人のDMにアクセス不可（管理者特権なし）
- パブリックチャネルへの参加は自由、プライベートチャネルは招待制
